В случае использования лицензирования на основе Виртуализованных Мощностей Лицензиат должен приобрести разрешения в объеме, достаточном для охвата всех активированных процессорных ядер, доступных для Программы или управляемых ею, в соответствии с Правилами Подсчета числа Лицензий для Виртуализованных Мощностей на Web-странице по адресу http://www.ibm.com/software/lotus/passportadvantage/Counting_Software_licenses_using_specific_virtualization_technologies.html.

* Активированное процессорное ядро ? это процессорное ядро, доступное для использования на физическом или виртуальном сервере, даже в том случае, если мощность процессорных ядер может быть ограничена или ограничивается с помощью технологий виртуализации, команд операционной системы, параметров настройки BIOS или аналогичных ограничений.

Особые Положения для Программы

Определения

"Производное Приложение" ? это компьютерная программа с уникальным именем, которая создана Лицензиатом или для Лицензиата с использованием Программы и которая включает в себя, требует наличия и/или выполняет Программное Обеспечение для Внедрения,  отдельно устанавливаемое на вычислительном устройстве. 

"Программное Обеспечение для Внедрения" ? это части Программы, которые интегрированы с другими программами, инструментами и кодом и встроены в Производное Приложение, либо установлены отдельно от Производного Приложения, но запускаются или вызываются Производным Приложением либо требуются для работы Производного Приложения.

Лицензия на разработку

Если Программа имеет обозначение "Developer Edition" или "Analyst Studio", Лицензиат имеет право использовать Программу только для внутренних целей разработки, включающих проектирование, разработку, тестирование, выполнение и/или поддержку Приложения (Приложений) Лицензиата. Кроме того, Лицензиат может запускать Программу посредством интегрированной среды разработки (IDE) только для внутренних деловых операций.

Single User Edition (Версия для одного пользователя)

Если Программа имеет обозначение "Single User Edition", Лицензиат имеет право использовать Программное Обеспечение для Внедрения, связанное с Программой, для одного Производного Приложения исключительно в целях непосредственной поддержки внутренних деловых операций Лицензиата. Разрешения нельзя использовать совместно для запуска нескольких Производных Приложений и можно переназначать только в случае безвозвратной передачи для запуска другого Производного Приложения.

На Клиентском Устройстве, на котором работает Программа, для Программы должно быть доступно не более 6 процессорных ядер.

Лицензия на внедрение

Для приобретенных Лицензиатом разрешений на Программное Обеспечение для Внедрения, в Свидетельстве о Правах на которое есть пометка "Limited Use", "Desktop Edition" или "Server Edition" в Названии Программы, Лицензиат имеет право использовать Программное Обеспечение для Внедрения, связанное с Программой, для одного Производного Приложения Лицензиата исключительно в целях непосредственной поддержки внутренних бизнес-операций Лицензиата.

Для всех других приобретенных Лицензиатом разрешений на Программное Обеспечение для Внедрения с пометкой "Deployment Edition" Лицензиат имеет право использовать Программное Обеспечение для Внедрения, связанное с Программой, для любого Производного Приложения Лицензиата исключительно в целях непосредственной поддержки внутренних бизнес-операций Лицензиата.

Для приобретенных Лицензиатом разрешений на Программное Обеспечение для Внедрения, в Свидетельстве о Правах на которое есть пометка "CPLEX Optimizer LP" или "CPLEX LP" в Названии Программы, Лицензиат имеет право использовать Программное Обеспечение для Внедрения, связанное с Программой, только для решения непрерывных задач линейного программирования (Linear Programming, LP) и квадратичного программирования (Quadratic Programming, QP).

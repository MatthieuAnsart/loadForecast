CPLEX remote object for distributed parallel optimization

Announces new routines and methods to implement your own distributed algorithms with the CPLEX remote object.

CPLEX V12.5 offers a remote object to facilitate your own implementation of distributed parallel algorithms. Though CPLEX itself does not implement a distributed parallel algorithm as such, the CPLEX remote object greatly simplifies your development of your own application implementing a distributed parallel algorithm. For an introduction to this new feature, see the topic Remote object for distributed parallel optimization in the CPLEX User's Manual. That introduction includes sample code contrasting a local application of CPLEX with an application making use of this remote object in a distributed application.

CPLEX V12.5 also includes new examples, demonstrating this new feature in applications. The new example parmipopt.c shows how to run distributed MIP optimizations concurrently in parallel. The new example parbenders.c takes a Benders decomposition of a model to run in parallel in a distributed environment. There are also additional generic and platform-specific make files along with sample projects in CPLEX 12.5 to assist you in executing those examples, as well as in developing and deploying your own distributed applications using the CPLEX remote object on all plaforms supported by CPLEX V12.5.

For further detail about the routines supporting the CPLEX remote object, see especially the documentation of the new routine in the Callable Library (C API) CPXXopenCPLEXremote.

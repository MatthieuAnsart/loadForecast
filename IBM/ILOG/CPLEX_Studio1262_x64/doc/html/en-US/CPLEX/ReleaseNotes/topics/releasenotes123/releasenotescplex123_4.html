Changes in existing parameters

Describes changes in existing parameters of interest to all users.

The table Table 1 lists parameters that have changed in behavior or acceptable values since the previous release. Navigational links in the table take you to the documentation of the changed parameter in the  CPLEX Parameters Reference Manual.

Presolve parameters interact with the new lazy constraint callbacks. If your application uses the previous cut callbacks or the new lazy constraint callbacks, see the topic Presolve parameters and lazy constraints in these release notes for information about this interaction.

Certain parameters, such as limitations on the number of nodes searched or the number of iterations performed during optimization, formerly accepted int. Now these parameters can accept symbols denoting much larger values. For a list of these parameters, see the tables Table 2 and Table 3 in this release note.

See also the topic New parameters in these release notes.

Table 1. Changed parameters
Concert APIs Callable Library (C API) Python API Comment
Threads CPX_PARAM_THREADS threads The default behavior changed with respect to parallel mode, opportunistic search, deterministic search. See the topic Simplified threading and parallel search in these release notes for an introduction to the changes.
Reduce CPX_PARAM_REDUCE preprocessing.reduce In the presence of lazy constraints, CPLEX turns off dual reductions in presolve. See the topic Presolve parameters and lazy constraints in these release notes.
PreLinear CPX_PARAM_PRELINEAR preprocessing.linear In the presence of lazy constraints, CPLEX turns off nonlinear reductions in presolve. See the topic Presolve parameters and lazy constraints in these release notes.
ParallelMode CPX_PARAM_PARALLELMODE parallel The default behavior changed with respect to number of threads used. See the topic Simplified threading and parallel search in these release notes for an introduction to the changes.
CoeRedInd CPX_PARAM_COEREDIND preprocessing. coeffreduce There are changes in the coefficient reduction setting. The new default value is -1. An additional value 3 supports new, more aggressive reductions with tilting.
Tip:

Tilting means to replace one row of a model with another such that an integer vector is feasible in the new row if and only if the integer vector was feasible in the original row. In contrast to a dominating row, the LP region for a tilted row can contain fractional points that are excluded by the LP region of the original row and the bounds of the variables. But, the aim is to tilt the row in such a way that it gets tighter in those areas of the LP polyhedron that are not already covered by other constraints.

Parameters extended to type long

The parameters in the table Table 2 have been extended on all platforms (that is, all combinations of supported hardware and operating system) to accept a long integer as a value. For some of these parameters, the default value is now much larger. The documentation of each of these parameters in the CPLEX Parameters Reference Manual reflects these changes in type, in default, and in possible values. (Navigational links from the table take you to the online documentation of each parameter.) See also the topic Possible changes in CPLEX extended parameters in these release notes for details of this change in the API that interests you.

Port-dependent changes in parameters

Port-dependent parameters are LONG if you use a port (that is, the format of CPLEX suitable to your combination of compiler and operating system) for a 64-bit platform. For such ports, the symbol CPXNNZ is defined as a 64-bit wide integer. For 32-bit ports, CPXNNZ is a 32-bit integer, and the port-dependent parameters are of type INT. See the topic Possible changes in CPLEX extended parameters in these release notes for details of this change in the API of CPLEX that interests you.

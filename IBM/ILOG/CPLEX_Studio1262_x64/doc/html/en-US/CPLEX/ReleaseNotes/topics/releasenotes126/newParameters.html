New parameters for all users

Documents new parameters of CPLEX V12.6.0.

These notes highlight new parameters available in CPLEX V12.6.0.

New parameters to manage ramp-up phase in parallel distributed MIP

As explained in the topic Solving a MIP with distributed parallel optimization in the CPLEX User's Manual, there is a ramp-up phase when you apply this new feature to solve a problem. The following new parameters enable you to customize this ramp-up phase for your model.
  • To set the duration of the ramp-up phase, use the ramp up duration parameter, CPXPARAM_DistMIP_Rampup_Duration.
  • To set a deterministic time limit on the ramp-up phase, use the deterministic time spent in ramp up during distributed parallel optimization parameter, CPXPARAM_DistMIP_Rampup_DetTimeLimit.
  • To set a wall-clock time limit in seconds on the ramp-up phase, use the time spent in ramp up during distributed parallel optimization parameter, CPXPARAM_DistMIP_Rampup_TimeLimit.

New names of existing parameters

CPLEX V12.6.0 introduces new names of all existing parameters of CPLEX.

CPLEX offers a variety of parameters for you to tune its performance to the demands of your particular problem. These numerous parameters involve a variety of algorithms and many different aspects of performance, sometimes making it difficult for a user to identify those parameters of most interest in the user's task. Furthermore, in some cases, the name of a given parameter in a given application programming interface (API) diverges significantly from its name in other components of CPLEX or other APIs; this divergence in names can make a change between components unnecessarily challenging for CPLEX users, (for example, when users switch from one API to another, or switch between the Interactive Optimizer for testing or debugging and an API for production). Also, some APIs and some components of CPLEX already organize parameters into a hierarchy of feature-related topics, whereas others do not offer such logical, task-oriented organization.

The new names of existing parameters address some of these vagaries in names and organization with the aim of making the application of parameters uniformly easier for users.

For a list of the new task-oriented categories organizing the new parameter names, see the topic Hierarchy of new parameter names in these release notes. For a table showing the new names and their correspondence with the deprecated existing names of parameters, see New names of CPLEX parameters also in these release notes.

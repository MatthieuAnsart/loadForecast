Conversion notes for users of the Callable Library (C API)

Changes in the Callable Library (C API) can impact existing applications.

These conversion notes highlight changes in CPLEX V12.5.1 that can impact existing applications of the Callable Library (C API).

  • Callbacks, threads, and parallel execution
    Behavior of parallel execution in the presence of multiple threads and callbacks now conforms to documentation.
  • Symbolic constants: numbering changed
    Symbolic constants changed in the Callable Library (C API).
  • Deprecated features removed:solution pool filtering
    Features deprecated in a previous version have been removed from CPLEX V12.5.1.
  • Documentation of CPXqpindefcertificate and CPXXqpindefcertificate changed
    Error corrected in the documentation of Callable Library (C API) routines CPXqpindefcertificate and CPXXqpindefcertificate
  • Invalid asynchronous routines for the remote object
    Features inadvertently documented in a previous version have been removed from CPLEX V12.5.1.
  • Changes in asynchronous and join routines for the remote object
    Arguments of asynchronous and join routines corrected in CPLEX V12.5.1.
  • Changes in MPI for the remote object worker
    MPI semantics changed for remote object worker in CPLEX V12.5.1.

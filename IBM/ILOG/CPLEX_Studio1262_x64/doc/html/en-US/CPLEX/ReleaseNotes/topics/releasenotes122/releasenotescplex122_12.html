Conversion note for users of power32_sles10_4.1

Informs users of a particular port of CPLEX about an issue in the IBM JRE version 5.

Because of an issue in the IBM Java Runtime Environment (JRE) version 5, the CPLEX port named power32_sles10_4.1 produces unpredictable behavior when used in the following conditions:

  • POWER or Power PC

  • 32-bit mode

  • GNU/Linux operating system

In other words, if all those criteria combine, so that you attempt to use the CPLEX port named power32_sles10_4.1 on a Power PC in 32-bit mode on the GNU/Linux operating system with IBM JRE V5, you are likely to observe unpredictable behavior, such as a segmentation fault.

Work around: use IBM JRE V6 instead in those conditions.

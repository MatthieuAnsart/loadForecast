Changes in the Callable Library (C API)

Previously deprecated routines CPXbinsolwrite and CPXwritesol have now been removed.

The routines CPXbinsolwrite and CPXwritesol (deprecated in CPLEX V10) have been removed from the Callable Library (C API) of CPLEX V12.4. Instead, use the XML solution format supported by routines such as CPXsolwrite. The reference manual File formats supported by CPLEX describes the XML solution format.

Changes in the Python API: features formerly in MIPCallback now in MIPInfoCallback

The method get_num_cuts and the class cut_type moved from MIPCallback to MIPInfoCallback. Changes for MATLAB users

Describes a change in behavior of interest to users of the CPLEX connector for MATLAB toolbox.

In the CPLEX connector for MATLAB, the default behavior of the function cplexoptimset() has changed. The default setting of the built-in MATLAB toolbox field LargeScale is now 'on', meaning that CPLEX solves a linear program (LP) by means of the barrier algorithm without crossover and returns the barrier iteration count.

To take advantage of this new behavior, consider these options:

  • [Default] LargeScale == 'on' and Simplex == 'off'

    CPLEX uses barrier without crossover by default in cplexoptimset() and returns the barrier iteration count.

  • LargeScale == 'on' and Simplex == 'on'

    CPLEX uses barrier followed by crossover and returns the simplex iteration count.

  • LargeScale == 'off' and Simplex == 'off'

    CPLEX disables barrier, but uses the default LP method and returns the simplex iteration count.

  • LargeScale == 'off' and Simplex == 'on'

    CPLEX uses primal simplex and returns the simplex iteration count.

To reproduce previous behavior in your MATLAB applications, turn off this option by setting it to 'off'.

Quadratic expression interface

Introduces an improved interface to accommodate quadratic expressions.

CPLEX V12.4 introduces a full-featured interface to support quadratic expressions in all APIs. The new interface accommodates quadratic expressions in the objective function and in constraints. The new interface makes it easier to query, modify, and manipulate models representing quadratic programs (QP) and quadratically constrained programs (QCP). It also accommodates second order cone programs (SOCP).

For example, when your application reads a QP, QCP, or SOCP, your application can now make use of iterators and accessors specifically designed for quadratic expressions, whether the expressions are in the objective function or in the constraints of the model. That is, when you import a model, the new interface facilitates queries and modifications of quadratic expressions in the model, and continues to support linear terms as well. The new interface also offers methods to check whether a constraint is a quadratic constraint or not.

For examples of this new interface in use, see also these samples distributed with the product. These examples show how to create a QP model, how to modify the coefficient of the quadratic terms in the objective function, and how to iterate over both the linear and quadratic terms in an expression.

  • In the C++ API, see iloqpex3.cpp.

  • In the Java API, see QPex3.java.

  • In the .NET API, see QPex3.cs and QPex3.vb

  • In the Python API, see socpex1.py.

  • In the MATLAB connector, see socpex1.m.

See also the topic New features in Concert Technology for more information about improved facilities for managing quadratic expressions.

Deterministic time limit

Introduces a deterministic time limit

CPLEX V12.4, in addition to the conventional time limits supported in earlier versions, now offers a deterministic time limit, available in all APIs and connectors. You manage this deterministic time limit by means of the parameter CPX_PARAM_DETTILIM, DetTiLim, documented in the CPLEX Parameter Reference Manual.

This deterministic time limit implies additional solution status codes:

  • CPX_STAT_ABORT_DETTIME_LIM

    CplexStatus.AbortDetTimeLim

    25 Optimization terminated because of the deterministic time limit.

  • CPX_STAT_CONFLICT_ABORT_DETTIME_LIM

    CplexStatus.ConflictAbortDetTimeLim

    39 The conflict refiner terminated because of the deterministic time limit.

  • CPXMIP_DETTIME_LIM_FEAS

    CplexStatus.DetTimeLimFeas

    131 An integer feasible solution exists, but the deterministic time limit has been reached.

  • CPXMIP_DETTIME_LIM_INFEAS

    CplexStatus.DetTimeLimInfeas

    132 The deterministic time limit has been reached, but no integer feasible solution exists yet.

You can also set a deterministic time limit for tuning a model or set of models. You do so by means of the symbol CPX_TUNE_DETTILIM, TuningDetTimeLim, 3 in the routine or method that invokes tuning.

New file format

Introduces a new file format: annotated linear program (ALP).

CPLEX V12.4 offers a new file format, annotated linear program (ALP). ALP files are LP files that contain the generic name of each variable, annotated by the type of each variable and bounds on each variable. Like the LP file format, ALP file format is algebraic, allowing entry of a model and data as an algebraic linear programming formulation. The new file format is supported by all APIs of CPLEX (Callable Library C API, Concert Technology C++ API, .NET API, Java API; Python API, Interactive Optimizer, CPLEX for MATLAB, CPLEX for Excel).

New features in Concert Technology

Highlights new features in C++, Java, .NET.

CPLEX V12.4 offers methods in Concert Technology for manipulating quadratic expressions when such terms occur in the objective function or constraints of your model. Technically, in this context, a quadratic expression represents a sum of quadratic terms of the form qexp = sum q_ij * x_i * x_j.

For names of the new methods, classes, or interfaces supporting quadratic expressions, see the topics in these release notes about the API of interest to you.

C++ API

New iterators to traverse both linear and quadratic expressions are available in the C++ API of Concert Technology. These iterators are available for ranges (that is, instances of the class IloRange) and for objectives (that is, instances of the class IloObjective). These iterators are shortcuts for the following lines of code:

IloExpr e = obj.getExpr();
e.getLinearIterator();

For more detail about these iterators, see their documentation in the reference manual of the C++ API.

  • IloObjective::getLinearIterator

  • IloObjective::getQuadIterator

  • IloRange::getLinearIterator

  • IloRange::getQuadIterator

New methods set the coefficients of the quadratic part of an expression (which may appear in a constraint or range of constraints) or a quadratic term in the objective function:

  • IloExpr::setQuadCoef(const IloNumVar var1, const IloNumVar var2, IloNum value);
  • IloObjective::setQuadCoef(const IloNumVar var1, const IloNumVar var2, IloNum value);

C++ API: new enumeration to support large CPLEX integer parameters

CPLEX V12.3 offers a new enumeration to support very large values in integer-valued parameters: IloCplex::LongParam. See the reference manual of the C++ API of CPLEX for more detail about this new enumeration. See also the topic Possible changes in CPLEX extended parameters as well as the topic C++ API: changes for large integer parameters (both in these release notes) for an explanation of this new enumeration.

Java API: method IloCplex.not accepts constraints

The Java method IloCplex.not has been improved so that it accepts as its argument any constraint that CPLEX can extract. Formerly, this method in the Java API accepted only instances of IloRange as its argument. For more information about which constraints CPLEX can extract, see the reference manual of the Java API of CPLEX, especially documentation of the interface IloConstraint.

Java API: notification about user data at nodes

CPLEX V12.3 adds a new interface to accommodate a user object associated with a node by means of a branch callback. The new interface is named IloCplex.BranchCallback.NodeData. It offers the virtual method delete. If you implement this method of this new interface in your branch callback function, then CPLEX notifies your application when that user object is no longer needed by CPLEX, so that your application can then delete the user object attached to the node and also delete any data associated with it.

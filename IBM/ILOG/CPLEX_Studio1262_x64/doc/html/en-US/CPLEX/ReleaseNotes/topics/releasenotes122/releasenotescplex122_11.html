Conversion notes for MATLAB users

Describes changes in the CPLEX connector for The MathWorks MATLAB.

See also the new functions, announced in MATLAB new features.

Least-square functions changed

For users of The MathWorks MATLAB toolbox, the existing least-square functions now accept linear inequality and equality constraints as arguments.

Previous signature New signatures
x = cplexlsqnonnegmilp(C,d) x = cplexlsqnonnegmilp(C,d)
x = cplexlsqnonnegmilp(C,d,Aineq,bineq)
x = cplexlsqnonnegmilp(C,d,Aineq,bineq,Aeq,beq)
x = cplexlsqnonnegmilp(C,d,sostype,sosind,soswt) x = cplexlsqnonnegmilp(C,d,Aineq,bineq,Aeq,beq,sostype,sosind,soswt)
x = cplexlsqnonnegmilp(C,d,sostype,sosind,soswt,ctype) x = cplexlsqnonnegmilp(C,d,Aineq,bineq,Aeq,beq,sostype,sosind,soswt,ctype)
x = cplexlsqnonnegmilp(C,d,sostype,sosind,soswt,ctype,x0) x = cplexlsqnonnegmilp(C,d,Aineq,bineq,Aeq,beq,sostype,sosind,soswt,ctype,x0)
x = cplexlsqnonnegmilp(C,d,sostype,sosind,soswt,ctype,x0,options) x = cplexlsqnonnegmilp(C,d,Aineq,bineq,Aeq,beq,sostype,sosind,soswt,ctype,x0,options)

With these least-square functions, a problem statement that formerly looked like this:

Description
Finds the minimum of a problem specified by 
   min      norm(C*x-d)^2
   st.      x >= 0
            x belongs to BICSN

now can look like this:

Description
Finds the minimum of a problem specified by 
   min      norm(C*x-d)^2
   st.      Aineq*x <= bineq 
            Aeq*x    = beq
            x >= 0
            x belongs to BICSN

If you have been using the existing least-square functions in your MATLAB application or sessions, you may need to modify the call to these functions when you upgrade to this release in order to take into account these additional arguments.

Parameter classes removed from CPLEX for MATLAB

The classes CpxIntParam, CpxDoubleParam, CpxStrParam and CpxBoolParam have been removed from the toolbox of CPLEX for MATLAB. Instead, use the property cplex.Param now to set and modify parameters. The use of parameter-related methods, such as tuneParam, readParam, getChgParam remains the same.

Data-consistency checking in CPLEX for MATLAB

The datacheck parameter now controls consistency checking in CPLEX for MATLAB. When datacheck is off, performance may be better in the case of valid data in a model. If datacheck is on, then CPLEX produces a detailed error message in the case of invalid data.

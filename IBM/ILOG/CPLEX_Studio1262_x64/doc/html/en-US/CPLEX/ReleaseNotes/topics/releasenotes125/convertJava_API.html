Changes in the Java API

Describes changes in the Java API of CPLEX and suggests replacements.

Adding cuts

The methods for adding cuts, both from the IloCplex object and from a callback, have changed. The type of argument that such methods accept is now IloRange; the argument is no longer IloConstraint. These methods have changed:

  • IloCplex.addCut

  • IloCplex.addCuts

  • IloCplex.addUserCut

  • IloCplex.addUserCuts

  • IloCplex.addUserCuts

  • IloCplex.addLazyConstraint

  • IloCplex.addLazyConstraints

  • IloCplex.addLazyConstraints

  • IloCplex.UserCutCallback.addLocal

  • IloCplex.LazyConstraintCallback.addLocal

Change in callback interface

The NodeData interface is now part of the MIPCallback class. (Formerly, it was part of the BranchCallback class.) If your Java application of CPLEX implemented this interface, you need to edit your application and recompile to take into account this change.

Deprecated

These methods of the Java API of CPLEX V12.5 have been deprecated and will be removed in a future release:

  • IloCplex.setVectors
  • IloCplex.SolveCallback.setVectors

Use these new or existing methods instead of those deprecated methods to add one or more starting solution candidates:

  • IloCplex.setStart
  • IloCplex.addMIPStart
  • IloCplex.SolveCallback.setStart

Changed in Callable Library (C API)

Describes change in behavior of C routines accessing parameters.

The behavior of two routines of the Callable Library (C API) that access parameters of CPLEX V12.3 has changed to return an error when an argument (that should be a string) is null.

  • CPXinfostrparam now returns the error CPXERR_NULL_POINTER if its final argument defvalue_str (the string denoting the default value of the parameter that the routine accesses) is null.

  • CPXgetparamname now returns the error CPXERR_NULL_POINTER if its final argument name_str (the string denoting the name of the parameter that the routine accesses) is null.

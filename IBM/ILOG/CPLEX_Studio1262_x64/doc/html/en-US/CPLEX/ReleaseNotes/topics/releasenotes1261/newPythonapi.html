New features of the Python API

Introduces new features of the Python API in CPLEX V12.6.1.

These notes highlight new features of interest to users of the Python API in CPLEX V12.6.1.

Multiple versions of Python supported by CPLEX

Formerly, CPLEX offered support for only one version of Python in a given installation. Now, however, CPLEX V12.6.1 supports multiple versions of Python, in particular, Python V2.6, V2.7 (depending on your operating system), and V3.4 in this release.

For more detail about how this additional support can impact paths to your CPLEX-Python modules, see the conversion note in these release notes.

For step-by-step advice on converting an earlier version of your CPLEX-Python application to the most recent supported version of Python (such as from Python 2.5 or 2.6 to 3.0), consult the detailed Python documentation at Python Software Foundation.

Source of the solution and a lazy constraint callback

CPLEX V12.6.1. offers a new method to query the source of a solution to determine whether a lazy constraint callback was invoked because of a solution found during a heuristic or because of a solution found at a node in the search tree. For more detail, see the documentation of this new method and the existing enumeration:

  • cplex.callbacks.LazyConstraintCallback.get_solution_source
  • cplex.callbacks.IncumbentCallback.solution_source

Encoding names

In CPLEX V12.6.1, you can use the wildcard * (also known as the asterisk) to specify an encoding name by means of the API encoding parameter, parameters.read.apiencoding. In that case, CPLEX uses the default converter according to the protocols and standards of International Components for Unicode (ICU). In other words, CPLEX calls ICU with a NULL encoding. When ICU is called with a NULL encoding (as in this case), ICU takes the name of the encoding from the terminal where the application started.

For more information about parameters.read.apiencoding, particularly for special considerations about Python, see the documentation of the API string encoding switch in the CPLEX Parameters Reference Manual.

Symbolic constants: numbering changed

Symbolic constants changed in the Callable Library (C API).

To accommodate new symbolic constants, the identifying number of the following symbolic constants has changed in CPLEX V12.5.1:

  • CPX_CUT_USER used to be 14 and is now 15
  • CPX_CUT_TABLE used to be 15 and is now 16
  • CPX_CUT_SOLNPOOL used to be 16 and is now 17
  • CPX_CUT_NUM_TYPES used to be 17 and is now 18

Remedy

If your application used the identifying number rather than the symbolic name of the constant, change the number to the new value, or (better) change the old, incorrect number to the symbolic name; then recompile your application.

These symbolic values are documented in the reference manual of the Callable Library (C API).

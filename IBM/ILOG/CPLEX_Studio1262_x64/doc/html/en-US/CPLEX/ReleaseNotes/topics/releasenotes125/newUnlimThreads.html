Unlimited threads and getting the number of cores

Introduces new routines and methods to access number of cores; removes limit on number of threads.

The number of threads for CPLEX to use can now be set to any positive integer number, even if this number exceeds the number of cores on the machine. The default behavior of CPLEX with respect to the number of threads and cores remains unchanged. That is, by default, CPLEX continues to use as many threads as there are cores.

Earlier versions of CPLEX licensed for a given number of cores are still limited to the licensed number.

CAUTION:
Setting the number of threads to a number greater than the number of cores on the machine typically results in degradation of the performance of CPLEX optimizers.

Nevertheless, in some situations it is useful to exploit a larger number of threads than the number of available cores. For example, some operating systems report a smaller number of cores than the number actually available in the computing hardware. In those situations, where you know surely that the number of cores is actually larger than the number reported by the operating system, then setting the number of threads available to CPLEX can be advantageous.

Furthermore, when you must accurately reproduce results on machines with differing numbers of cores, you can improve reproducibility of results by setting the number of threads available to CPLEX accordingly.

To assist you in calculating the number of threads that CPLEX uses by default (that is, the number of cores) on your platform, CPLEX offers these new routines and methods:

  • In the Callable Library (C API), see the routine CPXXgetnumcores.
  • In the C++ API, see the method IloCplex::getNumCores.
  • In the Java API, see the method IloCplex.getNumCores.
  • In the .NET API, see the property Cplex.NumCores.
  • In the Python API, see the method cplex.get_num_cores.

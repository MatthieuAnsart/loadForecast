Deprecated in the C++ API

Lists deprecated macros and classes of the C++ API and recommends replacements.

The macros ILOCUTCALLBACK0, ILOCUTCALLBACK1, through ILOCUTCALLBACK7 were formerly available to create an implementation class for a user-defined callback and a function that creates an instance of this class and returns a handle to it. These macros are deprecated in CPLEX V12.3. Use the new classes IloCplex::LazyConstraintCallbackI and IloCplex::UserCutCallbackI instead, or use the new macros, ILOUSERCUTCALLBACKn and ILOLAZYCONSTRAINTCALLBACKn instead.

The class CutCallbackI has been removed. The class LazyConstraintCallbackI now inherits directly from the class ControlCallbackI. Likewise, the class UserCutCallbackI now inherits directly from the class ControlCallbackI. In these release notes, the topic C++ API: changes in user cut callbacks and in lazy constraint callbacks offers more information about this change in inheritance.

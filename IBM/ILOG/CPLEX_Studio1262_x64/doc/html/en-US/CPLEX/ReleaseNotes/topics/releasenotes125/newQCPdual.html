QCP dual values and reduced costs

Announces new routines and methods to access the dual values and reduced costs in solutions of QCPs.

Perhaps you recall that CPLEX V12.4 showed how to express a second order cone program (SOCP) in Lagrangian terms and introduced new routines and methods in each API to access the dual values and reduced costs of the SOCP model in those terms. (In the CPLEX User's Manual, the topic Accessing dual values and reduced costs of SOCP solutions discusses access to dual values in the Lagrangian representation of SOCP models.) Now, CPLEX V12.5 generalizes those features for quadratically constrained programs (QCPs). For an introduction to this new feature, see the topic Accessing dual values and reduced costs of QCP solutions in the CPLEX User's Manual. That new topic in the user's manual reviews Lagrangian terms specific to QCPs and demonstrates a procedure to access dual values and reduced costs.

CPLEX V12.5 also offers examples of these new routines and methods in use for each API.

  • In the Callable Library (C API), see the examples xqcpdual.c and qcpdual.c.
  • In the C++ API, see the example iloqcpdual.cpp.
  • In the Java API, see the example QCPDual.java.
  • In the C#.NET API, see the example QCPDual.cs.
  • In the Python API, see the example qcpdual.py.
  • In the MATLAB connector, see the example qcpdual.m.

To access dual values and reduced costs (according to the procedure outlined in Accessing dual values and reduced costs of QCP solutions) in the solution of a quadratically constrained program (QCP) in your application, CPLEX offers these new routines and methods, documented in the reference manuals of the APIs:

  • Callable Library (C API)
    • CPXXgetqconstrdslack and CPXgetqconstrdslack
  • C++ API
    • IloCplex::getQCDSlack
  • Java API
    • IloCplex.getQCDSlack
  • .NET API
    • Cplex.GetQCDSlack
  • Python API
    • cplex.solution.get_quadratic_dualslack
  • MATLAB connector, a new field in the Cplex.Solution structure
    • qcpdslack

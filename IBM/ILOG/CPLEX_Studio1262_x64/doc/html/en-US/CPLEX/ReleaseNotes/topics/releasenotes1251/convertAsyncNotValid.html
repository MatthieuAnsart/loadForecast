Invalid asynchronous routines for the remote object

Features inadvertently documented in a previous version have been removed from CPLEX V12.5.1.

These inadvertently documented asynchronous and join routines have been removed from CPLEX V12.5.1.

  • CPXXgetmethod_async
  • CPXXgetnummipstarts_async
  • CPXXgetsolnpoolnumsolutions_async
  • CPXXgetsolnpoolnumreplaced_async
  • CPXXgetsolnpoolnumfilters_async
  • CPXXgetmethod_join
  • CPXXgetnummipstarts_join
  • CPXXgetsolnpoolnumsolutions_join
  • CPXXgetsolnpoolnumreplaced_join
  • CPXXgetsolnpoolnumfilters_join
  • CPXgetmethod_async
  • CPXgetnummipstarts_async
  • CPXgetsolnpoolnumsolutions_async
  • CPXgetsolnpoolnumreplaced_async
  • CPXgetsolnpoolnumfilters_async
  • CPXgetmethod_join
  • CPXgetnummipstarts_join
  • CPXgetsolnpoolnumsolutions_join
  • CPXgetsolnpoolnumreplaced_join
  • CPXgetsolnpoolnumfilters_join

Context

The following routines of the CPLEX Callable Library (C API) return an integer (for example, an integer to specify the number of MIP starts available, an integer to specify the number of solutions in the solution pool, and so forth). In other words, the integer that they return does not specify the status of the routine. They are not suitable for use asynchronously with the remote object.

  • CPXXgetmethod
  • CPXXgetnummipstarts
  • CPXXgetsolnpoolnumsolns
  • CPXXgetsolnpoolnumreplaced
  • CPXXgetsolnpoolnumfilters
  • CPXgetmethod
  • CPXgetnummipstarts
  • CPXgetsolnpoolnumsolns
  • CPXgetsolnpoolnumreplaced
  • CPXgetsolnpoolnumfilters

Based on those routines, the nonexistent asynchronous and join routines cited in this note were inadvertently documented in CPLEX V12.5.0.0 as asynchronous and join routines for the remote object. The cited routines are not available in CPLEX V12.5.1.

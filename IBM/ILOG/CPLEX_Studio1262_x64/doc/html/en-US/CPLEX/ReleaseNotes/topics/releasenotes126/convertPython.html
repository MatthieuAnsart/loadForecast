Conversion notes for Python users of CPLEX V12.6.0

Names of these parameters have changed for users of Python.

CPLEX V12.6.0 offers new names for all existing parameters. These new names are organized hierarchically, by task-oriented categories for greater ease of use. To discover the new name of a given parameter, consult the reference manual Parameters of CPLEX where you will find documentation citing the new name and previous name for each parameter in each application programming interface (API) and component, such as the Interactive Optimizer.

For quick reference, these release notes offer a table of these new names associated with their unchanged four-digit identifier and their former name in the Callable Library (C API) New names of CPLEX parameters.

CPLEX V12.6.0 continues to support the previous names of parameters generally, but migration to the new names is strongly recommended. In particular, for users of the Python API of CPLEX V12.6.0, the name of five parameters must be changed in existing Python API applications of CPLEX V12.6.0. In each of these cases, the term tuning in the former name must be changed to tune in the new name.

  • parameters.tuning.dettimelimit must change to parameters.tune.dettimelimit
  • parameters.tuning.display must change to parameters.tune.display
  • parameters.tuning.measure must change to parameters.tune.measure
  • parameters.tuning.repeat must change to parameters.tune.repeat
  • parameters.tuning.timelimit must change to parameters.tune.timelimit

Error-message format-string changed

Documents changes in the strings that format error messages.

The value returned by the routine CPXgeterrorstring has changed in certain cases: in situations where CPXgeterrorstring used to return the formatting character %d, it now returns %lld to accommodate the type CPXLONG, that is, very large values. This symbol represents a 64-bit integer and is wide enough to hold the information returned by the routine. If your application relied on the formatting character %d (for example, in a printf statement displaying error messages returned by CPXgeterrorstring), then you need to change the type to accommodate such very large values. To learn where the formatting character occurred, see the documentation of error codes in the reference manual of the Callable Library (C API).

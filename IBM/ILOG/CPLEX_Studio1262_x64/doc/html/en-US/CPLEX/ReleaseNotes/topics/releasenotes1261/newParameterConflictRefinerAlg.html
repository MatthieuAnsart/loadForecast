Conflict refiner algorithm

Announces a new parameter of CPLEX V12.6.1 to choose the algorithm CPLEX uses in the conflict refiner.

A new parameter lets you choose the algorithm that CPLEX uses in the conflict refiner to find minimal conflicts in an infeasible model.

By default, CPLEX automatically chooses the algorithm in the conflict refiner according to the way that CPLEX determined the infeasibility of the original problem.

To see which algorithm that CPLEX chose by default, use the parameter to manage conflict information display.

The algorithm selected by default by CPLEX ordinarily yields the fastest possible performance. However, in certain models, a nondefault selection can work better. In particular, settings 1 (one), 2, or 3 entail preprocessing or propagation algorithms that are very fast, but those same algorithms can be inefficient at discarding large numbers of constraints quickly. In such a case, explicitly setting this parameter to 5 for a limited solve by the conflict refiner or 6 for a full solve can improve performance.

Setting 4 of this parameter invokes the conflict refiner to detect an irreducibly inconsistent set (IIS) in a linear program (LP). You can apply this setting anyway to models that are not strictly LP; in that case, CPLEX tries to detect a conflict using only the linear constraints of the model.

This new parameter also introduces new symbols and enumerations to express its values symbolically in the application programming interfaces (APIs):

  • In the Callable Library (C API)

    0  CPX_CONFLICTALG_AUTO
1  CPX_CONFLICTALG_FAST
2  CPX_CONFLICTALG_PROPAGATE
3  CPX_CONFLICTALG_PRESOLVE
4  CPX_CONFLICTALG_IIS
5  CPX_CONFLICTALG_LIMITSOLVE
6  CPX_CONFLICTALG_SOLVE

Changes in the .NET API

Describes changes in the .NET API of CPLEX and suggests replacements.

Adding cuts

The methods for adding cuts, both from a Cplex object and from a callback, have changed. The class of argument that such methods accept is now Range; the class of argument is no longer Constraint.

Change in callback interface

The NodeData interface is now part of the MIPCallback class. (Formerly, it was part of the BranchCallback class.) If your .NET application of CPLEX implemented this interface or accessed the NodeData property, you need to edit your application and recompile to take into account this change.

Conversion notes for users of the Python API

Explains changes introduced in the Python API by CPLEX V12.6.1.

These notes explain changes introduced in CPLEX V12.6.1 for users of the Python API.

Multiple versions of Python

Formerly, a given installation of CPLEX offered only one version of Python modules, set-up scripts, and sample files, all located in the directory yourCPLEXhome/python/PLATFORM, (or in the folder yourCPLEXhome\python\PLATFORM) where yourCPLEXhome specified the location where CPLEX was installed as part of IBM ILOG CPLEX Optimization Studio, and PLATFORM stood for your combination of operating system and compiler.

Now, in CPLEX V12.6.1, however, the path in the CPLEX installation to the files of interest to Python users includes a designation of the Python version, like this: yourCPLEXhome/python/VERSION/PLATFORM where VERSION designates a version of Python compatible with CPLEX. For example, the relative path to CPLEX-Python modules on a Linux platform where Python 3 is available looks similar to this: 
yourCPLEXhome/python/3.4/x86-64_linux/

As a consequence of this change in the path to CPLEX files of special interest to Python users, you can now use CPLEX V12.6.1 with more than one version of Python installed on your local platform. In particular, each such directory (or folder) contains set up files specific to that designated version of Python.

On Linux and AIX operating systems, the CPLEX-Python module for Python V2.6 also works with Python V2.7. In other words, even though the directory (or folder) is named 2.6, the modules and examples also work with Python V2.7 on Linux and AIX operating systems.

See also the announcement in these release notes about support in CPLEX V12.6.1 for multiple versions of Python.

Node ID query

Previously, the method get_node_ID was defined only for the class BranchCallback. That is, unfortunately, the method was not available in a user cut callback nor in a lazy constraint callback, rendering it difficult to write such instructions to CPLEX as "Create no more than N cuts at the current node."

Now, however, in CPLEX V12.6.1, the method get_node_ID is defined in the class ControlCallback, thus making the method available in all derived subclasses for greater ease of use.

Methods return boolean now (not int)

Some methods in the class cplex.callbacks.HSCallback (the parent class of both HeuristicCallback and SolveCallback) now return a boolean value, rather than the integer values formerly returned. The methods are:

  • is_primal_feasible
  • is_dual_feasible

Writing files in Python

Formerly, the method Cplex.write assumed the MPS file format by default. This defect has been corrected, so that Cplex.write correctly interprets the file format as you designate it in CPLEX V12.6.1.

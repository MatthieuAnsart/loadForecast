New features of the Callable Library (C API)

Introduces new features of the Callable Library (C API) in CPLEX V12.5.1.

These notes highlight new features of interest to users of the Callable Library (C API) in CPLEX V12.5.1.

New routine to support the remote object

The Callable Library (C API) of CPLEX V12.5.1 offers this new routine to support the remote object. CPXXenvgroupstatus accesses the status of the most recent previous multicast operation and returns the status of the environment at the specified index.

New symbolic constant to support lift and project cuts

A new symbolic constant, CPX_CUT_LANDP with the identifier 14, has been added to the Callable Library (C API) to support lift and project cuts in CPLEX V12.5.1. See also the announcement New Parameters for all users in these release notes about the new lift and project parameter.

New symbolic constants to support callbacks

The following symbolic constants have been added to support detailed information available from callbacks in the Callable Library (C API) of CPLEX V12.5.1.

  • CPX_CALLBACK_INFO_LANDPCUT_COUNT 133 to count the number of lift and project cuts.
  • CPX_CALLBACK_INFO_USERCUT_COUNT 134 to count the number of user-added cuts.
  • CPX_CALLBACK_INFO_TABLECUT_COUNT 135 to count the number of active lazy constraints.
  • CPX_CALLBACK_INFO_SOLNPOOLCUT_COUNT 136 to count the number of active solution pool cuts.

New symbolic constants to support calculating dual values of QCP

To support the new parameter, calculate QCP dual values, CPLEX V12.5.1 introduces symbolic constants for the possible values of the parameter CPX_PARAM_CALCQCPDUALS.
  • CPX_QCPDUALS_NO Do not calculate dual values for the QCP.
  • CPX_QCPDUALS_IFPOSSIBLE Calculate dual values for the QCP as long as the calculations do not interfere with presolve reductions. default
  • CPX_QCPDUALS_FORCE Calculate dual values and disable any presolve reductions that interfere with these calculations.

New file formats available from CPXNETwriteprob

The routines CPXXNETwriteprob and CPXNETwriteprob of the Callable Library (C API) now support these additional file formats:
  • .rlp file extension for the RLP file format of LP formulations with generic names of rows and columns.
  • .alp file extension for the ALP file format of LP formulations with generic names of rows and columns, plus types and bounds.

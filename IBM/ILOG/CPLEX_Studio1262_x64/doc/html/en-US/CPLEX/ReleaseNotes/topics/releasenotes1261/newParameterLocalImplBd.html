Locally valid implied bound cuts

Announces a new parameter of CPLEX V12.6.1 to switch on or off generation of locally valid implied bound cuts in a MIP.

A new parameter enables you to control whether CPLEX generates locally valid cuts from implied bounds in your MIP model: MIP locally valid implied bound cuts switch

This new parameter differs from the existing MIP globally valid implied bound cuts switch, which controls globally valid cuts generated from implied bounds.

A new enumeration MIPCutsLocalImplied offers values for this new parameter.

Table 1. Values
Value Meaning
-1 Do not generate locally valid implied bound cuts
0 Automatic: let CPLEX choose; default
1 Generate locally valid implied bound cuts moderately
2 Generate locally valid implied bound cuts aggressively
3 Generate locally valid implied bound cuts very aggressively

Microsoft Windows 32-bit users: removal of undecorated function names

This change does not affect those who use Microsoft Visual Studio C++; it can affect those who are using unsupported compilers or languages, such as Fortran.

In previous releases of CPLEX, cplex.DLL made all function call symbols available with undecorated names (that is, without the suffix @N) even though the functions were, in fact, _stdcall functions for Microsoft Windows users.

This convention was deprecated in CPLEX V12.5.1. The convention has been removed from CPLEX V12.6.0.

Now functions are exported from cplex.DLL only once: either with the suffix @N if they are _stdcall functions or without that suffix if they are not _stdcall functions.

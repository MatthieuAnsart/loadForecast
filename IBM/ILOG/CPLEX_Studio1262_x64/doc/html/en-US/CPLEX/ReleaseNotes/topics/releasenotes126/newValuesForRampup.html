New values for ramp up duration in distributed parallel optimization

Documents new values in CPLEX V12.6.0 for ramp up duration.

As announced in Distributed parallel MIP optimization in these release notes, CPLEX V12.6.0 can solve mixed integer programming (MIP) models in parallel in a distributed environment. In doing so, CPLEX can execute a ramp up phase, managed by a new parameter ramp up duration. CPLEX introduces new enumerations and symbolic values in each application programming interface ( API) to represent possible values of this parameter.

  • In the Callable Library (C API), the values are defined as:
    • CPX_RAMPUP_DISABLED (-1) to turn off ramp up
    • CPX_RAMPUP_AUTO (0) to allow CPLEX to decide (default)
    • CPX_RAMPUP_DYNAMIC (1) to switch dynamically to distributed tree search
    • CPX_RAMPUP_INFINITE (2) to set an infinite horizon for ramp up (also known as concurrent MIP optimization)
  • In C++, see the enum DistMIPRampupDuration for these new values.
  • In Java, see the static public class DistMIPRampupDuration for these new values.
  • In .NET, see the class DistMIPRampupDuration for these new values.
  • In Python, see the class parameters.distmip.rampup.values for these new values.
  • In MATLAB (Cplex class API), see the class Cplex.Param.distmip.rampup.values.

The value of the new parameter ramp up duration interacts with values of the new ramp-up timing parameters time spent in ramp up during distributed parallel optimization and deterministic time spent in ramp up during distributed parallel optimization. For more detail about that interaction, see the documentation of all three parameters in the reference manual Parameters of CPLEX.

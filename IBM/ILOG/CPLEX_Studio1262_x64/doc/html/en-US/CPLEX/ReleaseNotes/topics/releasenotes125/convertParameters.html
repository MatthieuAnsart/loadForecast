Changed behavior of parameters

Notes changed behavior of existing parameters, with links to further documentation.

This release note lists existing parameters for which the behavior has changed in CPLEX V12.5. Two groups of parameters are of interest:

  • MIP display
  • Tuning and its effect on other parameters

Change in the behavior of the MIP display parameter

The behavior of the MIP node log display information parameter (CPX_PARAM_MIPDISPLAY, MIPDisplay) has changed with respect to the value 3. Now that setting of the MIP display parameter shows additional information for each new incumbent solution, such as the elapsed time in seconds and the deterministic time in ticks since beginning of optimization. This additional information about elapsed time since the beginning of optimization is also now available for the values 4 and 5 of this parameter as well.

Tuning and its effect on other parameters

Formerly, the tuning feature of CPLEX reset the value of certain parameters to the default value (if the user had set these parameters to other values) at the end of a tuning session. Now, the tuning feature leaves these parameters at the value set by the user.

More detail about these parameters is available in the IBM ILOG CPLEX Parameters Reference Manual.

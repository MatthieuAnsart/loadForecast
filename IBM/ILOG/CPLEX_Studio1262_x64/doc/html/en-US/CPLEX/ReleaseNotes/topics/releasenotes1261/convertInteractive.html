Conversion notes for users of the Interactive Optimizer

Explains changes introduced in the Interactive Optimizer by CPLEX V12.6.1.

These notes explain changes introduced in CPLEX V12.6.1 for users of the Interactive Optimizer.

512 byte limit on input lines

When the CPLEX Interactive Optimizer encounters an input command of more than 512 bytes, it now displays the error message CPXERR_LINE_TOO_LONG. (In the past, the Interactive Optimizer silently failed to parse such a command line correctly.)

For example, if the path to a working directory is extremely long in a command such as

set workdir /path_of_more_than_499_bytes/to/a/working/directory

CPLEX Interactive Optimizer now displays an error message to inform the user that it did not execute the command as expected.

Space in string parameters related to file system

The Interactive Optimizer in CPLEX V12.6.1 now accepts space in a string used as the value of a parameter related to the file system. Specifically, commands such as set workdir can now accept a path name that contains spaces. This change to accept space in a string applies both to interactive input (such as a user typing during a session) and to file input (such as the contents of a PRM file).

CPLEX V12.6.1 offers two alternative techniques for specifying space in a string in such a case:

  • Enclose the entire string in double quotation marks. For example, "Program Files" is an acceptable string containing a space.
  • In place of the space, use a percent sign with the code for space, according to your encoding (such as ISO-8859-1 or UTF-8). For example, Program%20Files is an acceptable string containing a space.

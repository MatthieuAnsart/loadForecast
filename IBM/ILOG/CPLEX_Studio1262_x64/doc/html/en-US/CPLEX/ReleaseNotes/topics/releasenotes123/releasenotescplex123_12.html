Deprecated in the Callable Library (C API)

Lists deprecated routines of the Callable Library and recommends replacements.

Ancient string-handling routines

These routines of the Callable Library (C API) are deprecated in CPLEX V12.3. They will be removed in a future release. These CPLEX-specific routines were introduced in very early versions of CPLEX to support portable code and programming languages with limitations in string-handling. However, progress in systems applications has made them obsolete. Replace the deprecated routines by conventional system calls.

  • CPXmalloc

  • CPXfree

  • CPXrealloc

  • CPXmemcpy

  • CPXstrlen

  • CPXmsgstr

  • CPXstrcpy

Previous cut callback interface

These routines of the Callable Library (C API) have been made obsolete by improvements in the user cut callback interface and lazy constraint callback interface. These routines are deprecated in CPLEX V12.3. They will be removed in a future release. Replace them by new routines in the user cut callback interface and lazy constraint callback interface.

  • CPXsetcutcallbackfunc

    If your application needs a user cut callback function, replace this deprecated routine by CPXsetusercutcallbackfunc.

    If your application needs a lazy constraint callback function, replace this deprecated routine by CPXsetlazyconstraintcallbackfunc.

  • CPXgetcutcallbackfunc

    To access a user cut callback function, replace this deprecated routine by CPXgetusercutcallbackfunc.

    To access a lazy constraint callback function, replace this deprecated routine by CPXgetlazyconstraintcallbackfunc.

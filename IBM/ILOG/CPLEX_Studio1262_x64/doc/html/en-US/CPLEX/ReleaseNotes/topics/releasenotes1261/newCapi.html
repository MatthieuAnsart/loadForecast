New features of the Callable Library (C API)

Introduces new features of the Callable Library (C API) in CPLEX V12.6.1.

These notes highlight new features of interest to users of the Callable Library (C API) in CPLEX V12.6.1.

New routines to support the remote object

The Callable Library (C API) of CPLEX V12.6.1 offers these new routines to support the remote object. These routines allow a user to manage a transport type (such as MPI or TCP/IP) at a low level of control.
  • CPXXtransportctrldbl
  • CPXXtransportctrlint
  • CPXXtransportctrllong

Lazy constraint callback source

New symbolic values in CPLEX V12.6.1 specify the source (a node or a heuristic) of a lazy constraint callback when your application invokes CPXXgetcallbacknodeinfo and CPXgetcallbacknodeinfo. The new values are:

  • CPX_LAZYCONSTRAINTCALLBACK_NODE specifying that the lazy constraint callback was invoked from a node.
  • CPX_LAZYCONSTRAINTCALLBACK_HEUR specifying that the lazy constraint callback was invoked from a heuristic.

Encoding names

In CPLEX V12.6.1, you can use the wildcard * (also known as the asterisk) to specify an encoding name by means of the API encoding parameter, CPXPARAM_Read_APIEncoding. In that case, CPLEX uses the default converter according to the protocols and standards of International Components for Unicode (ICU). In other words, CPLEX calls ICU with a NULL encoding. When ICU is called with a NULL encoding (as in this case), ICU takes the name of the encoding from the terminal where the application started.

For more information about CPXPARAM_Read_APIEncoding, see the documentation of the API string encoding switch in the CPLEX Parameters Reference Manual.

New example: Benders decomposition using lazy constraint and user cut callbacks

Announces new example demonstrating Benders decomposition

CPLEX V12.3 offers a new example, available in all APIs, of a Benders decomposition to solve a model. The problem, known as an asymmetric traveling salesperson problem (ATSP), is represented as a mixed integer linear program (MILP) decomposed into a master ILP and a slave LP. To solve the master, the example adds Benders cuts based on solutions found in the slave. The example adds cuts by means of a lazy constraint callback function to separate integer feasible solutions; it also adds cuts by means of a user cut callback function to separate fractional infeasible solutions. In other words, the example highlights the new distinction in CPLEX V12.3 between user cut callbacks and lazy constraint callbacks, as announced in the topic User cut callbacks distinguished from lazy constraint callbacks in these release notes.

  • In the Callable Library (C API), see the examples bendersATSP.c and xbendersATSP.c (64-bit). That is, distinct examples illustrate how to use the new 64-bit interface announced in the topic New features in the Callable Library: 64-bit integer counters in these release notes.

  • In the C++ API, see the example ilobendersatsp.cpp.

  • In the Java API, see the example BendersATSP.java.

  • In the .NET API, see the example BendersATSP.cs.

  • In the Python API, see the example bendersatsp.py.

Getting the number of cores

Documents impact of new methods to get the number of cores.

The number of threads for CPLEX to use can now be set to any positive integer number, even if this number exceeds the number of cores on the machine, as explained in the announcement Unlimited threads and getting the number of cores.

If you previously used a method such as IloCplex::getMax(IloCplex::IntParam::Threads) or IloCplex.getMax(IloCplex.IntParam.Threads) as a way to access the number of cores on your platform, you must change your application because that practice now produces incorrect results. To avoid such incorrect results, use one of the following methods instead.

  • In the Callable Library (C API), use the routine CPXXgetnumcores.
  • In the C++ API, use the method IloCplex::getNumCores.
  • In the Java API, use the method IloCplex.getNumCores.
  • In the .NET API, use the property Cplex.NumCores.
  • In the Python API, use the method cplex.get_num_cores.

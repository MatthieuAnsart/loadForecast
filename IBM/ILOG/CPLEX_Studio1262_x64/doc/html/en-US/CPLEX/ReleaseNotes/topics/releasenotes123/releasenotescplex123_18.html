Quality report: kappa for QP

Introduces kappa in the report about the quality of a solution in QP models.

CPLEX V12.3 improves the solution quality report to show the kappa condition number of models solved by the simplex algorithm, even for models with a quadratic term in the objective function (QP problem type).

To see the kappa condition number of the basis in the solution a QP model solved by the simplex optimizer, use one of the following techniques.

  • In Concert Technology, use one of the following methods:

    • In the C++ API, use the method IloCplex::getQuality(IloCplex::Kappa).

    • In the Java API, use the method IloCplex.getQuality(IloCplex.QualityType.Kappa).

    • In the .NET API, use the method Cplex.GetQuality(Cplex.QualityType.Kappa).

  • In the Python API, use the method Cplex.solution.get_float_quality(Cplex.solution.quality_metric.kappa)

  • In the Callable Library (C API), use the routine CPXgetdblquality to access the kappa condition number in the double-precision variable dvalue, like this:

    status = CPXgetdblquality(env, lp, &dvalue, CPX_KAPPA);

  • In the Interactive Optimizer, use the command: display solution kappa

  • In the CPLEX connector for MATLAB, see the topic Quality report: kappa for QP in MATLAB in these release notes.

For more information about the kappa condition number as a means of detecting ill-conditioned models generally, see the topic Measuring problem sensitivity with basis condition number kappa in the User's Manual for CPLEX (CPLEX User’s Manual > Continuous optimization > Solving LPs: simplex optimizers > Diagnosing performance problems > Numeric difficulties > Measuring problem sensitivity with basis condition number).

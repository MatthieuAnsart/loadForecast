Conversion notes for Excel connector users

Describes changes of interest to customers who use the CPLEX connector for Microsoft Excel.

CPXsolve in the CPLEX connector for Microsoft Excel now accepts an additional, optional argument, the Boolean value NoDialog. If the value of the optional argument cannot be converted to a Boolean value, the value is ignored. If the optional argument is False, then the function solves the problem and shows the dialog afterward that allows you to reset solution values or to show reports. If the argument is True, then the function solves the problem and shows no dialog at all. Instead, it writes the solution to the current spreadsheet and produces no reports. The advantage of the new optional argument is that you can solve a problem with it non-interactively.

The new signature of the macro looks like this:

CPXsolve()  |equivalent to CPXsolve(False) or
CPXsolve(True)

See cplexcplexvba.bas (among the samples distributed with the product) for an example of this new optional argument in use.

Changes in existing parameters

Describes changes in existing parameters of interest to all customers.

The table titled Table 1 lists parameters that have changed in behavior or possible values since the previous release. Navigational links in the table take you to the documentation of the changed parameter in the CPLEX Parameters Reference Manual.

Table 1. Changed parameters
Concert Parameter Callable Library Interactive Optimizer Link to Reference Manual
ClockType CPX_PARAM_CLOCKTYPE clocktype clock type for computation time
MIPInterval CPX_PARAM_MIPINTERVAL mip interval MIP node log interval

The default value of the ClockType parameter has changed from 0=automatic: let CPLEX choose to 2=wallclock time.

The parameter MIP node log interval (CPX_PARAM_MIPINTERVAL, MIPInterval) now accepts either a positive or negative integer value, making it easier for you to adjust the amount of information logged for your problem.

  • By default, CPLEX V12.2 displays log information during a MIP solve at relatively high frequency during the early stages of the solve, and displays log lines at progressively longer intervals as the algorithm continues. In other words, CPLEX V12.2 displays information frequently in the beginning and progressively less often as it works.

  • A positive value n specifies the number of nodes CPLEX processes before it displays a new line in the node log. That is, CPLEX displays a new line regularly every n nodes.

    CPLEX continues to log new incumbents as they arrive.

  • A negative value n determines how much processing CPLEX does before it displays a new line in the node log. A negative value close to zero means that CPLEX displays new lines in the log frequently. A negative value far from zero means that CPLEX displays new lines in the log less frequently. In other words, a negative value of this parameter contracts or dilates the interval at which CPLEX displays information in the node log.

    CPLEX continues to log new incumbents as they arrive.

  • To reproduce the former default behavior, set this parameter at 100.

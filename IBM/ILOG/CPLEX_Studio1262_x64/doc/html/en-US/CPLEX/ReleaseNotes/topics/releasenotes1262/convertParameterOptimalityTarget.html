Optimality target: new name for former solution target parameter

Announces a change in the name of a parameter of CPLEX V12.6.2.

The name of the former solution target parameter has changed from CPX_PARAM_SOLUTIONTARGET, SolutionTarget to CPXPARAM_OptimalityTarget, OptimalityTarget. This parameter is now known as the optimality target parameter. The behavior of this parameter remains the same: it specifies the type of solution CPLEX attempts to compute when CPLEX solves a continuous or mixed integer quadratic model. See the documentation of optimality target in the Parameters Reference Manual.

The former name CPX_PARAM_SOLUTIONTARGET, SolutionTarget has been deprecated.

The former identifier 1131 of this parameter has also been deprecated. Use its new identifier 1148 instead.

For control over the type of solution (whether basic or non basic) that CPLEX searches for, see the documentation of the new parameter, solution type for LP and QP, in the Parameters Reference Manual.

Tip: For users of the Interactive Optimizer, the deprecation of the former name SolutionTarget and the introduction of the new parameter SolutionType can cause problems in commands that used the abbreviation sol for the parameter formerly known as SolutionTarget but now known as OptimalityTarget. For example, if you have a script of Interactive Optimizer commands that includes a command such as: 
set sol 2
you need to change that command to use the full, unambiguous name, like this: 
set solutiontarget 2
or, better, change the command to use the new name, like this: 
set optimalitytarget 2
That change will avoid ambiguity of the deprecated abbreviated name with an abbreviation of the new parameter SolutionType in a command such as the following one: 
set sol 2
which causes CPLEX V12.6.2 to raise an error about the ambiguous command.

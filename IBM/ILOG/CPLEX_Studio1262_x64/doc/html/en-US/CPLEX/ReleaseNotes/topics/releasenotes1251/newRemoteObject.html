New features of the remote object

Introduces new features of the remote object in CPLEX V12.5.1.

CPLEX V12.5.1 offers additional features of interest to users of the remote object. Introduced in the Callable Library (C API) of CPLEX V12.5, the remote object is now also available to users of the C++ API and the Java API of CPLEX V12.5.1.

For more about the remote object in the C++ API, see also the release note New features of the Concert C++ API.

For more about the remote object in the Java API, see also the release note New features of the Concert Java API.

The message passing interface (MPI) as a transport type for the remote object is now available to users on Solaris Ultra Sparc. For details about supported versions of the operating system and compiler, see the Detailed System Requirements (DSR) on the IBM Support Portal at:

Detailed System Requirements (DSR)

Furthermore, for users of the message passing interface as a transport type in applications of the remote object, CPLEX V12.5.1 offers support for an additional implementation: MPICH2.

(This new option is in addition to support for Open MPI already available in CPLEX V12.5.)

This additional transport type is available to users of the CPLEX remote object in applications of the Callable Library (C API), C++ API, and Java API on platforms where MPICH2 has been installed.

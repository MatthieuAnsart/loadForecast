New CPLEX parameters

Announces new parameters of CPLEX V12.6.2.

These notes announce new parameters available in CPLEX V12.6.2.

Boolean Quadric Polytope (BQP) cuts

A new parameter, BQP cuts, controls the generation of Boolean Quadric Polytope (BQP) cuts when CPLEX solves a nonconvex quadratic program (QP) or nonconvex mixed integer quadratic program (MIQP) to global optimality. For documentation of this new parameter, see Boolean Quadric Polytope cuts in the Parameters Reference Manual. See also the definition of BQP cuts and their bibliographic background in Boolean Quadric Polytope (BQP) cuts in the CPLEX User's Manual.

Collaboration of Oktay Gunluk and Jeff Linderoth in the Mathematical Programming group at the Thomas J. Watson Research Center with Pierre Bonami and Christian Bliek of the CPLEX development team made implementation of this new parameter possible.

CPU mask for binding threads to processors

Tip: This feature is available to users of AIX and GNU/Linux on x86, x86-64, Power, and Z architectures.

A new parameter introduced in CPLEX V12.6.2 enables you to specify whether and how CPLEX binds multiple threads to multiple processors (or cores) in your multithreaded applications on multiprocessor platforms. See the documentation of CPU mask and thread-binding switch for more detail about this innovation.

Solution type: basic or non basic

A new parameter, the solution type, specifies whether CPLEX searches for a basic or non basic solution of the current model. For documentation of this new parameter, including acceptable values and their effects, see solution type for LP and QP in the Parameters Reference Manual.

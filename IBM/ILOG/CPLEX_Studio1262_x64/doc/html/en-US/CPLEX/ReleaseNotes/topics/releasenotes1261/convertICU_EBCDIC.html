Conversion note for ICU and EBCDIC users

Explains changes introduced by CPLEX V12.6.1 that apply to users of International Components for Unicode (ICU) with EBCDIC encoding.

This conversion note applies to users of ICU with EBCDIC encodings in CPLEX applications.

CPLEX uses ICU, the International Components for Unicode, in its interpretation of encodings, such as EBCDIC, the Extended Binary Coded Decimal Interchange Code, an 8-bit character encoding used mainly on IBM computer operating systems.

Because of a difference in the way that the IBM Java Virtual Machine and Runtime Environment interprets newline characters, and the way ICU interprets newline characters, CPLEX users can encounter unexpected results with respect to character encodings that include a newline character when the user's application relies on EBCDIC encoding.

To avoid this anomaly, the user can specify: 
 "IBM1047,swaplfnl"
After that specification by the user, ICU behaves as does IBM VM and JRE with respect to newline characters in EBCDIC encoding in CPLEX applications.

For example, when the user creates a file encoded in EBCDIC, the user can explicitly state the full name of the encoding used, or the user can append the suffix ",swaplfnl" to the name.

Distributed parallel MIP optimization

Announces distributed parallel MIP optimization

As the most recent CPLEX offering for solving certain difficult mixed integer programs, distributed parallel MIP optimization operates in two phases: first, a ramp-up phase, in which each worker applies different parameter settings to the same problem as the other workers; then, in the remainder of the solve (the second phase), each worker works in one part of a common MIP tree. Each worker communicates what it finds to the (unique) master node, which acts as the conductor or coordinator for the whole process.

Distributed MIP is based on the CPLEX remote object for distributed parallel optimization introduced in CPLEX V12.5.0 for users of the Callable Library (C API) and extended to other application programming interfaces (APIs) in CPLEX V12.5.1. Like the remote object, distributed MIP can potentially use any of the transport protocols available in your computing environment. In order to specify to CPLEX which machines to use in addition to the master, you provide a file, written in XML and known as a Virtual Machine Configuration (VMC), to declare the transport protocol to use, the addresses of the target machines, and other configuration settings. You inform CPLEX of this VMC file and load it into your CPLEX environment.

As explained in the topic about distributed parallel MIP optimization in the CPLEX User's Manual, there is a ramp-up phase when you apply this new feature to solve a problem. The following new parameters enable you to customize this ramp-up phase for your model.

  • To manage the duration and character of the ramp-up phase, use CPXPARAM_DistMIP_Rampup_Duration.
  • To set a deterministic time limit on the ramp-up phase, use CPXPARAM_DistMIP_Rampup_DetTimeLimit.
  • To set a wall-clock time limit in seconds on the ramp-up phase, use CPXPARAM_DistMIP_Rampup_TimeLimit.

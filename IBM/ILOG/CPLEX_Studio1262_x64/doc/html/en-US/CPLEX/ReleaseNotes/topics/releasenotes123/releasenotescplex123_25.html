New features for users of MATLAB

Describes new features available to users of the CPLEX connector for The MathWorks MATLAB.

64-bit MATLAB connector

In CPLEX V12.3, the CPLEX connector for The MathWorks MATLAB is compatible with installations of MATLAB that support 64-bits, such as Mac OS. (Previous versions of the CPLEX connector for MATLAB were already compatible with 64-bit Microsoft Windows and 64-bit GNU/Linux operating systems.)

Priority orders in MATLAB

In CPLEX V12.3, the CPLEX connector for MATLAB supports priority orders for branching. For general information about priority orders for branching, see the topic Issuing priority orders in the CPLEX User’s Manual. For details specific to priority orders in the CPLEX connector for MATLAB, see the online help available in a MATLAB session.

Tolerances in MATLAB

In the CPLEX connector for MATLAB, the Toolbox parameter TolFun has been refined to distinguish the value used in LP models from the value used in MIP models. In other words, you can now set different tolerances for your LP and MIP models. In an LP model, the Toolbox parameter TolFun corresponds to cplex.Param.simplex.tolerances.optimality in the class API. In a MIP model, the Toolbox parameter TolFun corresponds to the method cplex.Param.mip.tolerances.absmipgap in the class API.

Named files for exporting models

In the CPLEX connector for MATLAB, the user can now specify the name of a file to export a model. In the Toolbox, the user specifies a file name with a new option of ExportModel. (In the class API, the user specifies a file name with the existing method writeModel.)

Sign of lambda (Lagrange multipliers)

In CPLEX V12.3, the CPLEX connector for MATLAB now conforms to conventions in MATLAB with respect to the sign (positive or negative) of lambda in the methods lambda.ineqlin and lambda.eqlin. The lambda structure is documented in the reference manual of the CPLEX connector for MATLAB.

In this context, the fields of a lambda structure contain the Lagrange multipliers for the solution returned by an optimization. In previous versions, CPLEX conventions about characteristics of a solution were not consistent with MATLAB conventions; in other words, the sign of lambda appeared reversed to MATLAB users. This inconsistency in conventions between the two products has been reconciled in favor of the MATLAB convention with respect to lambda and Lagrange multipliers in these two methods lambda.ineqlin and lambda.eqlin.

New parameters in MATLAB

Parameters formerly available only in other APIs of CPLEX are now available as well to users of the CPLEX connector for MATLAB in CPLEX V12.3. For further information about these parameters, see the online documentation about CPLEX available in a MATLAB session.

  • cplex.Param.output.mpslong specifies the degree of precision for CPLEX to write numbers to a file in MPS format.

  • cplex.Param.output.writelevel specifies the level of detail for CPLEX to write a solution to a file in SOL format or a MIP start to a file in MST format.

  • cplex.Param.output.intsolfileprefix turns on or off CPLEX output to write the current MIP incumbent integer solution to a file and sets a prefix for the name of that file.

  • cplex.Param.output.clonelog (the new parameter to clone the logs of nodes during parallel optimization) is also available in CPLEX V12.3 connector for MATLAB. Its default value 1 (one) clones the logs of nodes in parallel optimization. Use the value 0 (zero) to turn off this feature.

Problem structure as input to class constructor

The Cplex class constructor can now take a problem structure as input. Each of the toolbox functions defines “problem structure” in its reference entry. In other words, this new constructor makes it easier to switch between Toolbox functions and the class API.

Quality report: kappa for QP in MATLAB

In CPLEX V12.3, the connector for MATLAB produces a solution quality report to show the kappa condition number of models solved by the simplex algorithm, even for models with a quadratic term in the objective function (QP problem type). For instructions about producing a solution quality report in the Cplex Class API, see the online documentation about CPLEX available in a MATLAB session.

Note: There was an error in earlier versions of these release notes which implied that the solution quality report is available through the CPLEX for MATLAB Toolbox functions. In fact, the solution quality report is available from only the Cplex Class API. For more information, see Technote: Accessing CPLEX solution quality metrics in MATLAB (http://www-01.ibm.com/support/docview.wss?uid=swg21573656).

Interrupting CPLEX in MATLAB

Earlier versions of the CPLEX connector for MATLAB recognized control-C to interrupt the solution process. Now, you can interrupt certain other processes as well by means of control-C, in the class API:

  • Tuning CPLEX parameters with the method tuneparam

  • Using the method feasopt

  • Refining conflicts with the method refineconflict

  • Populating the solution pool with the method populate

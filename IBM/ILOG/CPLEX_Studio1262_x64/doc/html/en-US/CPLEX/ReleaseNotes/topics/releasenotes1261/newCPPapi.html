New features of the C++ API

Introduces new features of the C++ API in CPLEX V12.6.1.

These notes highlight new features of interest to users of the C++ API in CPLEX V12.6.1.

New methods to support the remote object

The C++ API of CPLEX V12.6.1 offers these new methods to support the remote object. These methods manage transport control.
  • IloCplex::transportctrl(int ctrl, double &data)
  • IloCplex::transportctrl(int ctrl, CPXINT &data)
  • IloCplex::transportctrl(int ctrl, CPXLONG &data)

Source of the solution and a lazy constraint callback

CPLEX V12.6.1. offers a new method to query the source of a solution to determine whether a lazy constraint callback was invoked because of a solution found during a heuristic or because of a solution found at a node in the search tree. For more detail, see the documentation of this new method and the existing enumeration:

  • IloCplex::LazyConstraintCallbackI::getSolutionSource
  • IloCplex::IncumbentCallbackI::SolutionSource

Output operator for node identifier

CPLEX V12.6.1 now includes an output operator << for a node identifier IloCplex::MIPCallbackI::NodeId. In other words, in your C++ applications, you can now write a node identifier from a branch callback, for example, like this:

std::cout << getNodeId() << std::endl;

Extracted constraint exception

CPLEX V12.6.1 raises a new exception, IloCplex::ExtractedConstraintException, when an attempt occurs to add a constraint that has already been extracted to the model. For example, the method IloCplex::addLazyConstraints raises this exception if the lazy constraints in question have already been extracted to the model.

Encoding names

In CPLEX V12.6.1, you can use the wildcard * (also known as the asterisk) to specify an encoding name by means of the API encoding parameter, IloCplex::Param::Read::APIEncoding. In that case, CPLEX uses the default converter according to the protocols and standards of International Components for Unicode (ICU). In other words, CPLEX calls ICU with a NULL encoding. When ICU is called with a NULL encoding (as in this case), ICU takes the name of the encoding from the terminal where the application started.

For more information about IloCplex::Param::Read::APIEncoding, see the documentation of the API string encoding switch in the CPLEX Parameters Reference Manual.

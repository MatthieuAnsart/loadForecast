Cuts factor parameter changed behavior

Announces changes in the behavior of the cuts factor parameter of CPLEX V12.6.2.

The default value, the range of acceptable values, and the behavior of the cuts factor parameter have changed. For complete documentation, see cut factor row-multiplier limit in the Parameters Reference Manual.

Default value

Previously, the default value of the cuts factor parameter was 4.0. Now, the default value is -1.0 (negative one). The default value -1.0 means that CPLEX dynamically adjusts the limit on the number of cuts added to the model.

Range of acceptable values

The range of acceptable values of this parameter is now [-1.0, BIGREAL] (where the symbol BIGREAL depends on your platform).

Behavior

For values between zero and one inclusive (that is, in the range [0.0, 1.0], CPLEX generates no cuts.

For values strictly greater than 1.0 (one), CPLEX limits the number of rows in the model with cuts added. The limit on this total is the product of CutsFactor times the original number of rows.

If CPLEX has presolved the model, the "original number of rows" is the number of rows in the presolved model. (This behavior with respect to a presolved model is unchanged.)

CPLEX regards negative values of this parameter as equivalent to the default value -1.0. That is, a negative value specifies no particular limit on the number of cuts. CPLEX computes and dynamically adjusts such a limit automatically.

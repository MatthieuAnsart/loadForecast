New reporting features in Python API of CPLEX

Announces new features in the Python API to report problem statistics.

New features in the Python API of CPLEX report problem statistics previously available in the Interactive Optimizer of CPLEX. Going beyond the display available in the Interactive Optimizer, the new features of the Python API produce formatted reports available interactively in a Python session. The same raw data of the reports is also available programmatically in a Python application. Additionally, the new features generate histograms of the nonzero counts of the columns or rows of the linear constraint matrix of a model.

In addition to problem statistics, these new methods of the Python API also report metrics of the quality of a solution. For example, these lines create a model from the LP file named problem.lp, solve the problem, and generate a formatted report about the quality of the incumbent solution:

c = cplex.Cplex("problem.lp")
c.solve()
print c.solution.get_quality_metrics()

Similarly, this line shows how to generate a quality report about a solution in the solution pool, where soln can be either an index or a name that identifies a member of the solution pool:

print c.solution.pool.get_quality_metrics(soln)

For additional detail, see the documentation of these methods in the QualityMetrics class in the CPLEX Python API Reference Manual. See these same samples, along with more general information, in the topic Querying solution information in the Python API in the CPLEX User’s Manual. Also see the topic Example: generating a histogram in the CPLEX User’s Manual.

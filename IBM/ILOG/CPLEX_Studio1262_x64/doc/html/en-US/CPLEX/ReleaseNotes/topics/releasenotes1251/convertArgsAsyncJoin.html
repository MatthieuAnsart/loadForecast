Changes in asynchronous and join routines for the remote object

Arguments of asynchronous and join routines corrected in CPLEX V12.5.1.

Context

These routines of the CPLEX Callable Library (C API) return the name of an element in a buffer.

  • CPXXgetcolname
  • CPXXgetrowname
  • CPXXgetmipstartname
  • CPXXgetsosname
  • CPXXNETgetnodename
  • CPXXNETgetarcname
  • CPXgetcolname
  • CPXgetrowname
  • CPXgetmipstartname
  • CPXgetsosname
  • CPXNETgetnodename
  • CPXNETgetarcname

For use with the remote object, CPLEX offers an asynchronous version and a join for each of those routines.

Correction

The signature of the asynchronous implementation of those routines for use with the remote object was incorrectly documented in a previous version of CPLEX. The incorrect signature of each included an erroneous buffer argument. The reference manual of the Callable Library (C API) in CPLEX V12.5.1 now documents the correct signature of each of these routines (without the erroneous argument).

  • CPXXgetcolname_async
  • CPXXgetrowname_async
  • CPXXgetmipstartname_async
  • CPXXgetsosname_async
  • CPXXNETgetnodename_async
  • CPXXNETgetarcname_async
  • CPXgetcolname_async
  • CPXgetrowname_async
  • CPXgetmipstartname_async
  • CPXgetsosname_async
  • CPXNETgetnodename_async
  • CPXNETgetarcname_async

The signature of the join routine for use with the remote object was incorrectly documented in a previous version of CPLEX for each of those routines. The incorrect signature of each omitted a necessary argument specifying a buffer. The reference manual of the Callable Library (C API) in CPLEX V12.5.1 now documents the correct signature of each of these routines (including the required buffer argument).

  • CPXXgetcolname_join
  • CPXXgetrowname_join
  • CPXXgetmipstartname_join
  • CPXXgetsosname_join
  • CPXXNETgetnodename_join
  • CPXXNETgetarcname_join
  • CPXgetcolname_join
  • CPXgetrowname_join
  • CPXgetmipstartname_join
  • CPXgetsosname_join
  • CPXNETgetnodename_join
  • CPXNETgetarcname_join

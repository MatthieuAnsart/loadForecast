Names as arguments

All Callable Library (C API) routines that accept a list of names now accept a null pointer as a name. Previously, CPLEX considered passing a null pointer as a name to be a bug; if data checking by means of DATACHECK were turned on, CPLEX used to return the error CPXERR_NULL_NAME (1224). Now this error code is no longer produced by CPLEX. The error code remains in the CPLEX header file to avoid breaking existing user-written code that referenced it, but the error code is deprecated in future applications.

Deleting names

Formerly, the Callable Library (C API) routines CPXXdelnames and CPXdelnames deleted only names of rows and names of columns. Now both these routines delete all names, including names of indicator constraints and names of special ordered sets (SOSs), when these routines are invoked.

Deprecated

These items in the Callable Library (C API) were deprecated in a previous release. They have been removed from CPLEX V12.5.

  • The routine CPXmstwrite has been deprecated and removed. Use the routine CPXwritemipstarts instead.
  • The routine CPXmstwritesolnpolnall has been deprecated and removed. Use the routine CPXwritemipstarts instead.
  • The routine CPXmstsolnpoolwrite has been deprecated and removed. Use the routine CPXwritemipstarts instead.
  • The routine CPXgetsolnpoolnummipstarts has been deprecated and removed. Use the routine CPXgetnummipstarts instead.
  • The routine CPXchgmipstart has been deprecated and removed. Use the routine CPXchgmipstarts instead.
  • The routine CPXgetmipstart has been deprecated and removed. Use the routine CPXgetmipstarts instead.
  • The routine CPXiiswrite has been deprecated and removed. Use the routine CPXclpwrite instead.
  • The routine CPXdisplayiis has been deprecated and removed. Use the routine CPXclpwrite instead.
  • The symbolic constant CPXIIS_TERSE has been deprecated and removed. Because the routines that could invoke this value have been removed, there is no need for a replacement.
  • The symbolic constant CPXIIS_VERBOSE has been deprecated and removed. Because the routines that could invoke this value have been removed, there is no need for a replacement.

These routines in the Callable Library (C API) were deprecated in a previous release.

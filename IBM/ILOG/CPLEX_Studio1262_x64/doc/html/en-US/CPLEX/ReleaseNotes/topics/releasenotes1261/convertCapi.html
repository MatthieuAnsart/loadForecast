Conversion notes for users of the Callable Library (C API)

Explains changes introduced in the Callable Library (C API) by CPLEX V12.6.1.

These notes explain changes introduced in CPLEX V12.6.1 for users of the Callable Library (C API).

CPXXpreslvwrite: changed behavior

In CPLEX V12.6.1, the behavior of CPXXpreslvwrite and CPXpreslvwrite changed with respect to lazy constraints and lazy constraint callbacks. If a lazy constraint or lazy constraint callback is present in the model, CPLEX V12.6.1 now turns off both dual reductions and nonlinear reductions. The new behavior is consistent with previous behavior of presolve. For more detail about dual reductions during presolve, see the documentation of primal and dual reduction type in the reference manual of CPLEX Parameters. Likewise, for more detail about nonlinear reductions during presolve, see the documentation of the linear reduction switch.

Change in error message 1465

The wording of the message in CPXERR_LINE_TOO_LONG, number 1465, has changed from specifying the error in number of characters. The error message now specifies the error in number of bytes.

previous message:
Line %lld: Line longer than limit of %d characters
new message:
Line %lld: Line longer than limit of %d bytes

Identifying candidate constraints for lazy constraint pool

Distinguishes user cuts from lazy constraints.

Here is a procedure for identifying which constraints in your model are good candidates to be lazy constraints. The procedure entails changing the problem type more than once.

  1. Begin with your mixed integer programming model (MIP).
  2. Change its problem type to a linear program (LP).
  3. Solve that LP.
  4. Identify constraints for which the dual value is 0 (zero). These are the constraints to make lazy after another change of problem type.
  5. Change the LP problem type to a mixed integer linear program problem type (MILP) so that the identified constraints can be added as lazy constraints.
  6. Solve the problem again, this time as a MILP.
Tip: Lazy constraints can be added only to problems of the type MILP; lazy constraints cannot be added to a problem of the type LP.

Deploying an application of the CPLEX remote object

Outlines steps to deploy an application of the remote object in CPLEX.

To execute an application of the CPLEX remote object, you must perform these steps:

  1. Build the master process.
  2. Build the object that implements the dynamically loadable user function.
  3. Deploy both the user-function object and the cplex binary file to the remote machine.
  4. Execute that code.

For simplicity, the following amplification of those steps executes everything on the same local machine, rather than on a real remote machine. However, later topics highlight the differences between local and remote execution. Furthermore, the make files provided with the product explicitly address those differences with machine-specific settings.

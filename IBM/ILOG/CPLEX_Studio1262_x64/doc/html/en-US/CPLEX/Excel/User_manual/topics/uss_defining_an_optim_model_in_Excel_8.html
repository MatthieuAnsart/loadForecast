Constraints Section

Describes the Constraints Section of an optimization model spreadsheet.

Constraints are restrictions you place on certain cells in the Decision Variables Section.

Any combination of constraints and decision variable cells can be entered into the Constraints Section. For a given constraint one cell contains the lower bound, one cell contains the upper bound, and one cell contains the constraint formula. For ease of use, you can create a table structure that presents these cells in a row; you can also group similar constraints by locating their bounds and formulas in a series of columns.

For example, in the diet.xls problem, column B (labeled Minimum) contains the lower bound of each constraint, column C (Provided) contains the constraint formula, and column D (Maximum) contains the upper bound of each of the same constraints. Each constraint is expressed in a single row.

Note:

It is not necessary to provide both a lower and an upper bound to a constraint. If nothing is specified, the missing bound is taken to be minus infinity for a lower bound or infinity for an upper bound. Constraints with quadratic terms must have only one finite bound.

The constraint formula and the formulas and functions used to compose the constraint may only have terms which are linear or quadratic in the decision variables.

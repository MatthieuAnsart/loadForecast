Callable Library examples using cplexx.h

These examples in C use the Callable Library and illustrate how to include cplexx.h.

The following examples are delivered with CPLEX in IBM ILOG CPLEX Optimization Studio. They differ from other examples in the Callable Library (C API) to show how to use features included by cplexx.h.

xbendersatsp.c demonstrates lazy constraint callbacks to separate integer feasible LP solutions and user cut callbacks to separate fractional infeasible LP solutions in a Benders decomposition of an asymmetric traveling salesperson problem.
xdiet.c is a linear program that can be generated by columns (add foods to the diet) or by rows (add requirements to the diet).
xfoodmanu.c implements a solution to the food production planning problem, well known from the modeling textbook of H. P. Williams; this implementation uses indicator constraints and semi-continuous variables.
xtuneset.c analyzes a set of models and suggests nondefault parameter settings to improve performance.
xlpex1.c is a basic linear programming example, illustrating data population techniques.
xlpex2.c is a basic linear programming example with data coming from files.
xlpex3.c demonstrates how to add rows to a problem and re-optimize.
xlpex4.c illustrates CPLEX callback functions; it is derived from lpex1.c.
xlpex5.c illustrates CPLEX message handler; it is derived from lpex1.c.
xlpex6.c illustrates loading of bases; it is derived from lpex1.c.
xlpex7.c shows how to get column names; it is derived from lpex2.c.
xlpex8.c shows another way to build a problem; it is derived from lpex1.c.
xmipex1.c is a basic mixed integer programming example, derived from lpex8.c.
xmipex2.c is a basic mixed integer programming example, derived from lpex2.c.
xmipex3.c illustrates SOS sets and priority orders; it derives from lpex8.c.
xmipex4.c shows how to create and use an informational callback for logging, limiting number of nodes, and interrupting optimization; also shows how to terminate optimization; also shows how to use time stamps.
xpopulate.c shows how to generate and keep multiple solutions in the solution pool.
xfixnet.c uses indicator constraints to avoid numeric difficulty in a fixed-charge network flow problem. (It does not use the conventional CPXNET interface.)
xnetex1.c is a basic network example.
xnetex2.c is a basic network example, with data coming from files.
xqpex1.c is a basic quadratic programming example, derived from lpex8.c.
xqpex2.c is a quadratic programming example; it derives from lpex2.c.
xglobalmiqpex1.c shows how to read a convex or nonconvex mixed integer quadratic programming model (MIQP) and solve it for a global optimum.
xglobalqpex1.c shows how to read a convex or nonconvex quadratic programming model (QP) and solve it for a first order or global optimum.
xmiqpex1.c is a mixed integer quadratic programming example; it derives from lpex8.c.
xindefqpex1.c shows how to solve an indefinite quadratic program (nonPSD QP), whether convex or nonconvex, to a local optimum.
xadpreex1.c uses advanced presolve to re-optimize with a secondary objective.
xadmipex1.c uses node and branch callbacks to optimize a MIP.
xadmipex2.c uses a heuristic callback to optimize a MIP.
xadmipex3.c uses a branch callback on a MIP with special ordered sets (SOSs).
xadmipex4.c uses a user-specified cut table for a MIP.
xadmipex5.c adds cuts through the cut callback for a MIP.
xadmipex6.c starts a MIP optimization from an LP solution.
xadmipex7.c uses a solve callback using barrier and dual optimizers.
xsocpex1.c shows how to access dual values and reduced costs in SOCP models.
xqcpdual.c shows how to access dual values and reduced costs in quadratically constrained programming models (QCP).
xsteel.c is based on the model steelT.mod from the AMPL modeling book by Fourer, Gay, and Kernighan.

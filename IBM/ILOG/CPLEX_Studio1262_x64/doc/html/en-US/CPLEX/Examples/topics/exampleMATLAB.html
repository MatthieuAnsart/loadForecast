MATLAB examples

These examples use the CPLEX connector for MathWorks MATLAB.

The following examples are delivered with CPLEX in IBM ILOG CPLEX Optimization Studio.

blend.m is a blending problem to produce a requested quantity of alloy from four sources (pure metal, raw material, scrap, ingots) each with a cost.
cplexplot.m shows how to use the built-in MATLAB function plot.
cutstock.m is a cutting-stock problem illustrating column generation; this kind of model is also known as a knapsack problem with reduced cost in the objective function.
diet.m implements a variation of George Dantzig's familiar diet problem; the model can be generated by columns (add foods to the diet) or by rows (add requirements to the diet).
etsp.m is a scheduling problem with costs for earliness and for tardiness.
facility.m is a warehouse-location problem.
fixcost1.m is a production planning problem with fixed and variable costs.
globalqpex1.m shows how to read a convex or nonconvex quadratic programming model (QP) and solve it for a first order or global optimum.
indefqpex1.m enters and solves an indefinite quadratic program (nonPSD QP) to a local optimum.
inout1.m is a production model with trade-offs between buying externally produced items to meet demand and producing equivalent items internally.
inout3.m solves a production model with an objective to minimize external production given product demand, a cost constraint, and minimum internal production constraints.
lpex1.m shows how to populate data in a model either by rows or by columns.
lpex2.m shows how to add data from a file.
lpex3.m shows how to add rows to a model and re-optimize.
lpex6.m shows how to load a basis from a file.
lpex7.m shows how to access names of columns to display them in the solution.
mipex1.m solves a mixed integer programming problem (MIP); it creates the model by rows, by columns, or by coefficients.
mipex2.m solves a MIP after reading data from files.
mipex3.m demonstrates special ordered sets (SOSs) and priorities in a MIP.
mipex4.m solves a MIP and demonstrates callbacks.
miqpex1.m solves a MIP with a quadratic objective (MIQP).
mixblend.m solves a blending problem with integer variables (MIP); that is, some of the resources can be added to the alloy only in integer quantities.
populate.m shows how to generate multiple solutions to a MIP and store them in the solution pool.
qcpex1.m solves a quadratically constrained problem (QCP).
qpex1.m solves a model with a quadratic objective (QP).
qpex2.m solves a model with a quadratic objective (QP) and data read from a file.
rates.m is a planning problem for a power plant with multiple generators; the model uses semi-continuous variables.
socpex1.m shows how to access dual values and reduced costs in SOCP models.
qcpdual.m shows how to access dual values and reduced costs in quadratically constrained programming models (QCP).
steel.m solves the model known as steelT.mod in the AMPL book by Fourer, Gay, and Kernighan; it is a multi-period production problem.
transport.m uses a piecewise linear cost function.

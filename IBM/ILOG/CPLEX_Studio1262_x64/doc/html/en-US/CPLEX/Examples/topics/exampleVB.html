VB.NET examples

These examples in VB.NET use Concert Technology.

The following examples are delivered with CPLEX in IBM ILOG CPLEX Optimization Studio.

BendersATSP.vb demonstrates lazy constraint callbacks to separate integer feasible LP solutions and user cut callbacks to separate fractional infeasible LP solutions in a Benders decomposition of an asymmetric traveling salesperson problem.
Blend.vb solves a blending problem.
CplexServer.vb shows how to enter a problem with the class CplexModeler and how to serialize the model for solving.
CutStock.vb is a cutting stock model, illustrating column generation.
Diet.vb is an implementation of the well known diet problem; this linear program can be generated by columns (add foods to the diet) or by rows (add requirements to the diet).
Etsp.vb demonstrates a scheduling problem with costs for earliness and tardiness.
Facility.vb is a warehouse-location problem.
FixCost1.vb is a production planning problem with fixed costs.
FoodManufact.vb implements a solution to the food production planning problem, well known from the modeling textbook of H. P. Williams.
GlobalQPex1.vb shows how to read a convex or nonconvex quadratic programming model (QP) or mixed integer quadratic programming model (MIQP) and solve it for a first order or global optimum.
Goalex1.vb uses the goal API for branching.
Goalex2.vb uses the goal API to add cuts.
Goalex3.vb uses the goal API to add node evaluators.
IndefQPex1.vb solves an indefinite quadratic program (nonPSD QP).
LPex1.vb is a basic linear programming model illustrating data population techniques.
LPex2.vb is a basic linear programming model with data coming from files.
LPex3.vb shows how to add rows to a model and re-optimize.
LPex4.vb illustrates callbacks.
LPex6.vb shows how to load a basis.
LPex7.vb shows how to access names of columns.
MIPex1.vb is a basic MIP model.
MIPex2.vb is another basic MIP model.
MIPex3.vb includes special ordered sets (SOSs) in a MIP model.
MIPex4.vb shows how to use an informational callback for logging, limiting number of nodes, and interrupting optimization; also shows how to terminate optimization; also shows how to use time stamps.
MIQPex1.vb is a mixed integer quadratic programming model; it includes a quadratic objective in a MIP model.
MixBlend.vb is a MIP blending model.
QPex1.vb includes a quadratic objective in an LP.
QPex2.vb is another model with a quadratic objective.
QPex3.vb uses the Concert Technology interface for quadratic expressions to enter and to modify a model with quadratic expressions in the objective function.
QCPex1.vb is a quadratically constrained problem.
Populate.vb shows how to generate and keep multiple solutions in the solution pool.
Rates.vb is a planning problem with semi-continuous variables.
SocpEx1.vb shows how to access dual values and reduced costs in SOCP models.
Steel.vb is a production planning problem illustrating ways to build a model; based on steelT.mod from the AMPL modeling book by Fourer, Gay, and Kernighan.
Transport.vb uses a piecewise linear cost function.
TuneSet.vb analyzes a set of models and suggests nondefault parameter settings for better performance.
Warehouse.vb is a warehouse-location problem; it uses goals.
AdMIPex1.vb uses node and branch callbacks to optimize a MIP.
AdMIPex2.vb uses a heuristic callback to optimize a MIP.
AdMIPex3.vb uses a branch callback on a MIP with special ordered sets (SOSs).
AdMIPex4.vb uses a user-specified cut table for a MIP.
AdMIPex5.vb adds cuts through the cut callback of a MIP.
AdMIPex6.vb starts a MIP optimization from an LP solution.

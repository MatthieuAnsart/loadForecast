Overview of the APIs of CPLEX

Provides links to reference manuals of the C, C++, and Java application programming interfaces of CPLEX

This overview of the APIs provides a convenient way to locate information about routines and methods of CPLEX, categorized by purpose. Information about the C, C++, and Java APIs appears in the tables of this overview, along with links to features documented in the reference manuals of the APIs. These navigational links are categorized by purpose for common tasks, such as creating or modifying a model, analyzing a solution, reading or writing files, and similar operations.

  • Introduction
    The Overview of the APIs follows these naming conventions in programming languages.
  • Creating and deleting problems
    These methods and routines handle the first and last steps in an optimization application: creating and populating the problem object and deleting the problem object.
  • Modifying problems
    These problem modification routines change a problem object after it has been created.
  • Optimizing problems
    The optimizers available for different problem types appear in this table.
  • Generating and keeping a pool of solutions
    These methods and routines manage the solution pool, which stores multiple solutions to a mixed integer programming (MIP) model.
  • Accessing results for continuous models
    These solution query methods and routines access information about the results of applying an optimization method to a problem object.
  • Accessing results for discrete models
    These solution query methods and routines access information about the results of applying an optimization method to a problem object.
  • Accessing general problem data
    These problem query routines and methods access information about a problem object after it has been created.
  • Accessing MIP problem data
    These MIP problem query routines and methods access information about a problem object after it has been created.
  • Accessing QP problem data
    These QP problem query routines and methods access information about a problem object after it has been created.
  • Accessing QCP problem data
    These routines and methods access information about quadratically constrained problems (QCPs).
  • Using MIP starts
    These routines and methods manage MIP starts, also known as advanced starts or warm starts.
  • Using callbacks
    These callback routines interrupt and resume optimization; they also access progress information while the optimization is in process.
  • Using logical constraints
    These logical constraints are available in Concert Technology for representing certain nonlinear elements in a model.
  • General utilities
    These general utilities initialize and close the CPLEX environment.
  • Reading files
    These file-reading routines and methods read problems from system files.
  • Writing files
    These file-writing routines and methods write information to system files.
  • Using messages
    These message-handling utilities control messages sent to the screen and to files.
  • Recognizing parameters that differ
    Access procedures differ between the C, C++, and Java APIs in these parameters.
  • Setting parameters
    These methods and routines access and modify parameters that control various aspects of CPLEX behavior.

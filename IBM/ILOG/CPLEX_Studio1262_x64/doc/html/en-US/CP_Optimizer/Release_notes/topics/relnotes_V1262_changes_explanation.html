New feature: Explanation log

With this version of CP Optimizer, an explanation log has been added.

This release of CP Optimizer provides a new debugging feature that explains failures during search. This feature computes and displays a minimal conflicting set of constraints and decisions explaining why a failure occurred at a search branch. These conflict sets can help with understanding the reasons that a model is difficult to solve. These sets can help to track down modeling errors and weak formulations. The conflicting set that explains a failure is computed using the conflict refiner.

To activate the explanation log, the parameter IloCP::LogSearchTags must be set to IloCP::On and the parameter IloCP::SearchStrategy must be set to IloCP::DepthFirst. This will display failure tags in the engine log when solving the model. To specify the failures to explain, the member function IloCP::explainFailure(IloInt failureTag) needs to be called with the tag of a failure. Several failures tags can be added. The member function IloCP::clearExplanations() is used to clear the set of failure tags to be explained.

The failure explanation feature is not available for models using interval variables, such as scheduling models.

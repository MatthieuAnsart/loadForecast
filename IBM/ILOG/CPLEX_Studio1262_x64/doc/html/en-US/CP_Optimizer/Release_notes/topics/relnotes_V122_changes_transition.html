Transition based scheduling

Transition based scheduling has been introduced in this version of CP Optimizer.

CP Optimizer V12.2 provides new features (integer expressions and constraints) allowing for easier modeling of problems that involve constraints and costs related with the transitions between activities executed on a unary resource. For instance, it is now possible to model transition costs or transition times that hold between immediate successors in a sequence. Furthermore, constraints between sequence variables, such as in permutation flow-shop problems, are now easier to handle.

For more information on the new transition based scheduling API, please refer to the concept “Interval variables sequencing” in the CP Optimizer C++ API Reference Manual and to the sections Modeling sequence-dependent setup times and Modeling classical scheduling costs in the CP Optimizer User’s Manual.

CP Optimizer V12.2 comes with 3 illustrative examples for transition-based modeling:

  • sched_setup: transition times between immediate successors and forbidden transitions in a sequence,

  • sched_tcost: minimization of total transition cost, and

  • sched_pflowshop: permutation flow-shop problem.

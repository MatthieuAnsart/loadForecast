New feature: New value and default for the workers parameter

Describes a new value and a new default for the parameter Workers.

In the C++ API, the IloCP::Auto value has been added to the set of possible values of the parameter IloCP::Workers. In the Java™ API, this value is IloCP.ParameterValues.Auto, and in the .NET languages API it is CP.ParameterValues.Auto. When the workers parameter is set to this value, the search uses as many workers as there are core CPUs available on the machine. The default value of the workers parameter is now Auto. When CP Optimizer is run on a multi-core machine, the search for a solution will run in parallel.

When performing complete exploration of the search space and collecting every unique solution as is done in the CrewScheduling example, it is recommended that you set the IloCP::Workers parameter to 1. Otherwise you may experience an increased number of duplicate solutions.

When solving large problems, the choice of IloCP::Workers parameter must take into account the amount of RAM available to the workstation. As a general rule, assume that for N workers, the memory usage of the search is multiplied by N.

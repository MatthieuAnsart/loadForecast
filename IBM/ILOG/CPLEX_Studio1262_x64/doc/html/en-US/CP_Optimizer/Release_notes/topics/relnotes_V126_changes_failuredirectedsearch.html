New feature: Failure-directed search

A new parameter controls usage of failure directed search.

Failure-directed search assumes that there is no (better) solution or that such solution is very hard to find. Therefore it focus on systematic exploration of search space, eliminating first assignments that are most likely to fail. CP Optimizer starts failure-directed search when other search strategies are (no longer) successful.

Failure-directed search is used only for scheduling problems (i.e. models containing interval variables) and only when the search type parameter is set to restart.

In the C++ API of CP Optimizer, failure-directed search is controlled by new parameters IloCP::FailureDirectedSearch and IloCP::FailureDirectedSearchMaxMemory.

In the Java™ API of CP Optimizer, failure-directed search is controlled by new parameters IloCP.IntParam.FailureDirectedSearch and IloCP.IntParam.FailureDirectedSearchMaxMemory.

Likewise, in the C# API of CP Optimizer, failure-directed search is controlled by new parameters CP.IntParam.FailureDirectedSearch and CP.IntParam.FailureDirectedSearchMaxMemory.

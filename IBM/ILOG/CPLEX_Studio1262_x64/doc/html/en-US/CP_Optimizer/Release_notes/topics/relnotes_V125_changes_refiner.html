New feature: Conflict refiner

A conflict refiner has been introduced in this version of CP Optimizer.

Given an infeasible model, the conflict refiner can identify conflicting constraints and variable domains within the model to help you identify the causes of the infeasibility. In this context, a conflict is a subset of the constraints and variable domains of the model which are mutually contradictory. The conflict refiner first examines the full infeasible model to identify portions of the conflict that it can remove. By this process of refinement, the conflict refiner arrives at a minimal conflict. A minimal conflict is usually smaller than the full infeasible model and thus makes infeasibility analysis easier.

Note that the CP Optimizer conflict refiner behaves much the same as the CPLEX conflict refiner with a similar API.

For more information on the conflict refiner, please refer to the concept “Conflict refiner” in the CP Optimizer C++ API Reference Manual.

Provided in the distribution is a new example that illustrates the use of the conflict refiner to find minimal conflicts in a model. In the C++ API, the new example is sched_conflict.cpp. In the Java™ API, the new example is SchedConflict.java . In the C# API, the new example is SchedConflict.cs.

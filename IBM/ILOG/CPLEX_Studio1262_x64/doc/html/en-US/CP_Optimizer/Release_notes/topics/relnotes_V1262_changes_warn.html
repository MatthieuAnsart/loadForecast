New feature: Warnings

A new feature for producing warnings about model constructs is available.

When a solve is launched, potentially suspicious modeling constructs are reported via the CP Optimizer "warning" stream (see IloCP::warning(), IloCP.warning(), or CP.Warning()). The level of warning can be controlled by a parameter. This parameter takes values from 0 to 3. 0 will turn off all warnings whereas 3 will report all possible warnings. The default value is 2.

In the C++ API of CP Optimizer, the parameter is set using IloCP::setParameter(IloCP::WarningLevel, ...).

In the Java™ API of CP Optimizer, the parameter is set using IloCP.setParameter(IloCP.IntParam.WarningLevel, ...).

Likewise, in the C# API of CP Optimizer, the parameter is set using CP.SetParameter(CP.IntParam.WarningLevel, ...).

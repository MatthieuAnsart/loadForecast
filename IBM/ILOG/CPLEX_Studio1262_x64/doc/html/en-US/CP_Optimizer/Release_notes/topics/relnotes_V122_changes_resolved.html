Table 1. CP Optimizer resolved issues
Issue
Possible crash when using a disjunctive cumul function with alwaysIn constraints.
Possible crash when producing a solution on a cumul function where all intervals contribute with a pulse and all present intervals have zero-length.
Possible crash when using an objective of the form x/y with some sub-expressions of x or y related with interval variables.
Bug in aggregations of difference constraints (for instance x!=y+1).
Possibly wrong propagation in case an optional interval last in a sequence becomes absent.
Depending on how the function f is created, expression f(x) where x is a numerical expression may compute wrong values

Expressions IloHeightAtStart/End do not correctly implement the semantics described in the documentation. These expressions return an erroneous value in the following cases:

  • IloHeightAtStart(a,f) in case of null length intervals with pulses and IloStepAtStart and

  • IloHeightAtEnd(a,f) in case of pulses.

For state functions, the triangular inequality on transition distance is not checked whereas state functions require this property.
In the .NET Languages API, there is no way to free the objects that are created to build search phases.

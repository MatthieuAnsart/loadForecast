intVar

Creates a new integer variable.

Note that there is also the type intVar with the same name.

Syntax

The function intVar is using special syntax. The potential values for the variable (its domain) can be given as a comma separated list of integers and ranges:

To simplify code generation, a trailing comma after a non-empty list of possible values is allowed.

Parameters

In any order, arbitrary number of:

  • single integer constants,
  • ranges min..max for intervals of integers [min, max].

Description

The function intVar creates a new integer decision variable with the specified set of potential values (domain). Decision variables represent the unknowns that needs to be found. That is, during solution process CP Optimizer must assign each variable a value from its domain in such a way that all specified constraints are satisfied.

Example

// Integer variable with a domain given as a range:
a = intVar(0..100);

// Enumerated domain:
b = intVar(1, 2, 3, 7, 10, 11, 12);

// Variable with the same domain as variable b but using ranges.
// Note the trailing comma at the end.
c = intVar(1..3, 7, 10..12);

// The variables above must take consecutive values:
a < b;
b < c;

Requirements

Integer variable can take only integer value within range intmin..intmax, therefore the specified values of the domain must also respect those limits.

Notes

  • Possible values of the variable can be specified in any order, they can repeat and the ranges can overlap.
  • Empty ranges such as 10..8 are also allowed. Of course, they do not contribute to the domain of the interval variable.
  • If the supplied list of possible values is empty, then there is no way to assign the variable and the whole problem is therefore insoluble (CP Optimizer reports that there is no solution).
  • CP Optimizer is obliged to assign a value to every decision variable when it searches for a solution, even if the variable is not used in any constraint or expression. Such variables have no purpose and slow down the search.

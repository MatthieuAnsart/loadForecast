Identifiers

A variable or expression can be referenced later by the identifier associated with it. As such, the identifier must be unique.

There are two kinds of identifiers: simple and generic identifiers.

Simple identifiers

A simple identifier is a name that most programming languages allow as an identifier. The first character must be an ASCII letter or underscore, while the remaining characters could also be ASCII digits. For example foo, _x, foo2, and fooFoo are all simple identifiers. The following words are reserved and cannot be used as simple identifiers: inf, intmax, intmin, intervalmax, intervalmin, div, mod, present, absent and optional.

Generic identifiers

A generic identifier does not have to start with a letter, and it can contain any UTF-8 characters (although some of them must be escaped). Unlike simple identifiers, generic identifiers must be enclosed in quotation marks. For example, "foo foo", "x[1]", "x+y" are valid generic identifiers. Note that enclosing quotation marks are not part of the identifier. Even reserved words, when enclosed in quotation marks, become identifiers, e.g. "inf" is also valid identifier. Recognized escape sequences are:

  • \" is a quotation mark,
  • \\ is a single backslash,
  • \n, \r, \t, \f and \b have the same meaning as in the C programming language: new line, carriage return, tab, form feed and backspace.

Note that simple identifiers can be also enclosed in quotation marks. There is no difference between foo and "foo"; they can be used interchangeably.

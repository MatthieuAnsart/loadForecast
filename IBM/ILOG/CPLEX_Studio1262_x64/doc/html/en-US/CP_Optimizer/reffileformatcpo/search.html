The search section

A CPO file may contain an optional search section which specifies a variable fixing strategy that CP Optimizer should use while searching for solutions. The strategy is described by search phases.

  • Introduction
    Some models have structures that are particularly well-understood by the user. It is possible to help CP Optimizer exploit these structures during the search and achieve much better solving times. In these cases, it is possible to adjust the automatic search in CP Optimizer using search phases in order to use the problem structures.
  • Selectors and evaluators
    A search phase can specify not only a group of variables to fix but also in what order they should be fixed. Additionally, a search phase can specify what values should be tried first for those variables. The fixing order can be specified using selectors and evaluators.
  • Syntax of search section
    The search section starts by search { and ends by }. Inside the section it is possible to declare search phases and any other objects necessary for them (i.e. evaluators, selectors etc.). The section can also be empty.
  • Types in search section
    All statements in the search section must be related to the fixing strategy. For example, it is not possible to declare new variables in the search section. Therefore there is a limitation in respect of types that can be used within search section.

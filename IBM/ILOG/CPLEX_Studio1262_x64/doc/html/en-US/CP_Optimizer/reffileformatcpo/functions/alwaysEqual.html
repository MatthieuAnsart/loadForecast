alwaysEqual

This constraint fixes a given state for a state function during a variable or fixed interval.

Syntax

constraint alwaysEqual(stateFunction f, intervalVar itv, uint val, bit isStartAligned = 0, bit isEndAligned = 0)
constraint alwaysEqual(stateFunction f, sint start, sint end, uint val, bit isStartAligned = 0, bit isEndAligned = 0)

Parameters

  • f: constrained state function.
  • itv: interval variable during which function f is constrained.
  • start, end: fixed interval [start,end) during which function f is constrained.
  • val: value of f during the interval.
  • isStartAligned, isEndAligned: these Boolean flags state whether the interval is start or end aligned on the state function f (default: no alignment).

Description

This function returns a constraint that ensures that state function f is defined everywhere on the interval (either interval variable itv when it is present or fixed interval [start,end)) and remains equal to value val over this interval.

Generally speaking, the optional Boolean values isStartAligned and isEndAligned allow synchronization of start and end with the intervals of the state function:

  • When isStartAligned is true, it means that start must be the start of an interval of the state function.
  • When isEndAligned is true, it means that end must be the end of an interval of the state function.

For more information on state functions, see the concept State functions in CP Optimizer.

Example

The example below models a set of activities requiring different temperatures of an oven. Activities requiring the same temperature can be executed in parallel.

a1 = intervalVar(size=5);
a2 = intervalVar(size=6);
a3 = intervalVar(size=8);
a4 = intervalVar(size=4);
a5 = intervalVar(size=7);
a6 = intervalVar(size=7);
temperature = stateFunction();
// Required temperatures for the activities
alwaysEqual(temperature, a1, 0);
alwaysEqual(temperature, a2, 0);
alwaysEqual(temperature, a3, 0);
alwaysEqual(temperature, a4, 1);
alwaysEqual(temperature, a5, 1);
alwaysEqual(temperature, a6, 2);
// Limited capacity of oven: no more than 2 activities in parallel
pulse(a1,1)+pulse(a2,1)+pulse(a3,1)+pulse(a4,1)+pulse(a5,1)+pulse(a6,1)<=2;

A possible solution to this problem is the following assignment of interval variables: a1=[0,5), a2=[0,6), a3=[5,13), a4=[13,17), a5=[13,20), a6=[20,27)`.

Example

Alignments are often used to model some synchronization of start and end times for a set of activities belonging to the same batch. The example below is a variation of the previous example where activities have variable duration and activities performed in parallel (same batch) must start and end at the same time.

a1 = intervalVar(size=2..5);
a2 = intervalVar(size=6..7);
a3 = intervalVar(size=6..10);
a4 = intervalVar(size=4..10);
a5 = intervalVar(size=2..7);
a6 = intervalVar(size=5..10);
temperature = stateFunction();
// Required temperatures for the activities. Activities in the same batch are synchronized.
alwaysEqual(temperature, a1, 0, 1, 1);
alwaysEqual(temperature, a2, 0, 1, 1);
alwaysEqual(temperature, a3, 0, 1, 1);
alwaysEqual(temperature, a4, 1, 1, 1);
alwaysEqual(temperature, a5, 1, 1, 1);
alwaysEqual(temperature, a6, 2, 1, 1);
// Limited capacity of oven: no more than 2 activities in parallel
pulse(a1,1)+pulse(a2,1)+pulse(a3,1)+pulse(a4,1)+pulse(a5,1)+pulse(a6,1)<=2;

A possible solution to this problem is the following assignment of interval variables: a1=[0,2), a2=[2,8), a3=[2,8), a4=[8,12), a5=[8,12), a6=[12,17)`.

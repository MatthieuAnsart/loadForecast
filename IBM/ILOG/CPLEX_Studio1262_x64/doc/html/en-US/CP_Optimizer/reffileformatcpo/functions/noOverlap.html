noOverlap

Constrains a set of interval variables not to overlap each others.

Syntax

constraint noOverlap(intervalVarArray itvs)
constraint noOverlap(sequenceVar seq, transitionMatrix dist = null, bit isDirect = 0)

Parameters

  • itvs: an array of interval variables.
  • seq: a sequence variable.
  • dist: an optional transition matrix defining the transition distance between consecutive interval variables.
  • isDirect: a Boolean flag stating whether the distance specified in the transition matrix dist holds between direct successors (isDirect=1) or also between indirect successors (isDirect=0, default).

Description

This function returns a constraint over a set of interval variables {a1,...,an} that states that all the present intervals in the set are pairwise non-overlapping. It means that whenever both interval variables ai and aj, i!=j are present, ai is constrained to end before the start of aj or aj is constrained to end before the start of ai.

If the no-overlap constraint has been built on an interval sequence variable seq, it means that the no-overlap constraint works on the set of interval variables {a1,...,an} of the sequence and that the order of interval variables of the sequence will describe the order of the non-overlapping intervals. That is, if ai and aj, i!=j are both present and if ai appears before aj in the sequence value, then ai is constrained to end before the start of aj. If a transition matrix dist is specified and if tpi and tpj respectively denote the types of interval variables ai and aj in the sequence seq, it means that a minimal distance dist[tpi,tpj] is to be maintained between the end of ai and the start of aj. If Boolean flag isDirect is true, the transition distance holds between an interval and its immediate successor in the sequence otherwise, if isDirect is false (default), the transition distance holds between an interval and all its successors in the sequence.

Example

a = intervalVar(size=5);
b = intervalVar(size=6);
c = intervalVar(optional, size=7);
// A sequence variable defined on {a,b,c}
seq = sequenceVar([a,b,c]);
// Interval variables in the sequence do not overlap
noOverlap(seq);

A possible value for the interval variables in this problem is a=[0,5), b=[5,11), c=[11,18). Another one is b=[0,6)', c=[6,13), a=[13,18). Another solution with c absent is a=[0,5), b=[5,11), c=absent`.

Example

a = intervalVar(size=5);
b = intervalVar(size=6);
c = intervalVar(optional, size=7);
noOverlap([a,b,c]);

This example shows the same problem as above using the constructor with an array of interval variables to shortcut the manual construction of the sequence variable seq.

Example

a = intervalVar(size=5);
b = intervalVar(size=6);
c = intervalVar(optional, size=7);
// A sequence variable defined on {a,b,c} with types {0,1,2}
seq = sequenceVar([a,b,c],[0,1,2]);
// A transition matrix between types [0,1,2]
dist = transitionMatrix(
           0,  5, 20, 
          14, 10,  7, 
           5,  8,  0,);
// Interval variables in the sequence do not overlap and must satisfy the transition distance
noOverlap(seq,dist,1);

A possible value for the interval variables in this problem is a=[0,5), b=[10,16), c=[23,30).

Example

a = intervalVar(size=5);
b = intervalVar(size=6);
c = intervalVar(optional, size=7);
// A sequence variable defined on {a,b,c} with types {0,1,2}
seq = sequenceVar([a,b,c],[0,1,2]);
// A transition matrix between types [0,1,2]
dist = transitionMatrix(
           0,  5, 20, 
          14, 10,  7, 
           5,  8,  0,);
// Interval variables in the sequence do not overlap and must satisfy the transition distance
noOverlap(seq,dist);

The difference between this example and the previous one is that the noOverlap constraint specifies that the transition distance should also hold between indirect successors (isDirect=0 by default). A possible value for the interval variables in this problem is a=[0,5), b=[10,16), c=[25,32). Note that the solution of the previous problem a=[0,5), b=[10,16), c=[23,30) is not feasible here because c (of type 2) is a (indirect) successor of a (of type 0) and the transition distance dist[0,2]=20 between the end value of a (5) and the start value of c (23) is not respected.

Requirements

  • All types of interval variables in sequence seq should be possible indexes of matrix dist. Stated otherwise, the size of matrix dist should be strictly larger than any interval type in sequence seq.

Notes

  • The Boolean flag isDirect makes a difference only in case the transition matrix does not satisfy the triangle inequality.
  • The function that takes an array of interval variables as argument is a shortcut to avoid the manual creation of a sequence variable in case this sequence is used in a unique noOverlap constraint without transition matrix.

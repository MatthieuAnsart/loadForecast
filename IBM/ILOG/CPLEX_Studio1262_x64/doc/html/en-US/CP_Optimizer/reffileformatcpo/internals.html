The internals section

The section internals contains internal data of CP Optimizer produced by the model dump function: IloCP::dumpModel in the C++ API, IloCP.dumpModel in the Java™ API, CP.DumpModel in C# API or oplrun -d for OPL models.

The first purpose of internal data is to guarantee the same behavior of CP Optimizer on the model loaded from the file. The second purpose is to include additional information regarding the version of CP Optimizer that produced the file. Therefore it is best to use model dumps, not exports, when contacting IBM support.

The content of this section is internal and should not be modified by the user. If you want to modify a file that contains an internals section then delete this section first; otherwise, CP Optimizer may refuse to read the file. Also do not include an internals section in your own CPO files.

For the reasons above, the syntax of the section internals is not documented.

Example

A model dumped using the C++ function IloCP::dumpModel contains the following section:

internals {
  version(12.6.1.0);
  architecture("x86-64_linux/static_pic", 64);
  ids(0, 1, 3, 5, 7, 9, 11, 12, 14, 16, 18, 20, 22, 23);
}

This data states that the model was dumped by CP Optimizer V12.6.1.0 on a 64-bit x86 Linux platform. The ids at the end of the section allows reordering of modeling objects in order to achieve the same behavior during solve as the original model. This line can be very long.

When this section is deleted then the behavior of CP Optimizer may change because it can process variables, expressions and constraints in a different order. It is still the same model though. The effect is similar to reordering statements in the file or changing the parameter RandomSeed.

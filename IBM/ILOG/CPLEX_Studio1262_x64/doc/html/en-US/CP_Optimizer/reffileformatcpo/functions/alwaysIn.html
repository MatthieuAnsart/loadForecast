alwaysIn

These constraints restrict the possible values of a cumulExpr or stateFunction to a particular range during a variable or fixed interval.

Syntax

constraint alwaysIn(cumulExpr f, intervalVar itv, uint min, uint max)
constraint alwaysIn(cumulExpr f, sint start, sint end, uint min, uint max)
constraint alwaysIn(stateFunction f, intervalVar, uint min, uint max)
constraint alwaysIn(stateFunction f, sint start, sint end, uint min, uint max)

Parameters

  • f: constrained cumul expression or state function.
  • itv: interval variable during which function f is constrained.
  • start, end: fixed interval [start,end) during which function f is constrained.
  • min, max: allowed range for values of f during the interval.

Description

These functions return a constraints that restricts the possible values of a cumul expression or state function f to a particular range [min,max] during an interval variable itv or a fixed interval [start,end). In the case of an interval variable itv, this constraint is active only when the interval variable is present, if the interval is absent the constraint is always satisfied, regardless of the value of f. When the constraint is posted on a state function, the range constraint holds only on the segments where the state function is defined.

Example

The following example models a discrete resource with an availability profile over time.

a1 = intervalVar(size=8);
a2 = intervalVar(size=5);
a3 = intervalVar(size=6);
a4 = intervalVar(size=7);
resourceUse = pulse(a1,2) + pulse(a2,3) + pulse(a3,3) + pulse(a4,2);
// Resource availability profile: 3 on [0,4), 4 on [4,10), 3 on [10,intervalmax)
alwaysIn(resourceUse, 0,  4, 0, 3);
alwaysIn(resourceUse, 4, 10, 0, 4);
alwaysIn(resourceUse, 10, intervalmax, 0, 3);

A possible solution to this problem is the following assignment of interval variables: a1=[0,8), a2=[16,21), a3=[10,16), a4=[4,10).

Example

The following example models a discrete resource together with a maintenance activity that can be performed only when no activity is executed on the resource.

a1 = intervalVar(size=8);
a2 = intervalVar(size=5);
a3 = intervalVar(size=6);
a4 = intervalVar(size=7);
// Maintenance activity to be scheduled to start in [10,15]
maintenance = intervalVar(size=3,start=10..15);
resourceUse = pulse(a1,2) + pulse(a2,3) + pulse(a3,3) + pulse(a4,2);
// Resource max capacity is 4
resourceUse <= 4;
// Maintenance activity requires the resource not to be used (max level 0).
alwaysIn(resourceUse, maintenance, 0, 0);

A possible solution to this problem is the following assignment of interval variables: a1=[0,8), a2=[8,13), a3=[16,22), a4=[0,7), maintenance=[13,16).

Example

The following example models the temperature of an oven as a state function. Each activity requires the temperature of the oven to belong to a particular range. Activities requiring incompatible temperature cannot be executed in parallel.

a1 = intervalVar(size=8);
a2 = intervalVar(size=5);
a3 = intervalVar(size=6);
a4 = intervalVar(size=7);
temperature = stateFunction();
// Each activity specifies a particular range of possible temperature
alwaysIn(temperature, a1, 10, 20);
alwaysIn(temperature, a2, 15, 25);
alwaysIn(temperature, a3, 5, 10);
alwaysIn(temperature, a4, 5, 15);
// The temperature should always be defined and remain constant during each activity
alwaysConstant(temperature, a1);
alwaysConstant(temperature, a2);
alwaysConstant(temperature, a3);
alwaysConstant(temperature, a4);

A possible solution to this problem is the following assignment of interval variables: a1=[0,8), a2=[8,13), a3=[0,6), a4=[0,7).

Requirements

  • For the function using a constant interval, end should be greater than start.
  • max should be greater than min.

Notes

Note that unlike constraints alwaysConstant and alwaysEqual, the constraint alwaysIn does not require the state function f to be defined over the entire interval.

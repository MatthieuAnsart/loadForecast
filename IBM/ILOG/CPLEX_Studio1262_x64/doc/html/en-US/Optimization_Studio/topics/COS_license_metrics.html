License metrics

IBM® License Metric Tool helps Passport Advantage® clients determine their full and sub-capacity PVU licensing requirements.

Learn more: http://www.ibm.com/software/products/en/license-metric-tool/.

The IBM ILOG® CPLEX® Optimization Studio installer adds a product-specific tag file for the Developer Edition Authorized User metric. The IBM ILOG CPLEX Enterprise Server installer adds a product-specific tag file for the Server Edition. The IBM ILOG CPLEX Optimizer Deployment Entry Edition adds a product-specific tag for the Deployment Entry Edition.

If other entitlements or metrics are used, the tag file placed by the installer should be replaced by the appropriate tag file as listed in Table 1 and as described below for copying tags for customer deployments. For example, if you use CPLEX Optimization Studio under the Client Device metric, you should use that tag to replace the Developer Edition tag.

As IBM ILOG CPLEX Optimization Studio components are embedded within the customer's application, there is no way for IBM to install a tag file with each deployed application. Consequently, customers who want ILMT to generate reports must copy tag files for deployment licenses, with each application that they deploy.

  1. Determine which deployment license(s) you need, and make sure that your company is properly entitled.

  2. Open the Tags.zip file that contains tag files for each deployment license metric. This file is in <INSTALL_PATH>/properties/version/ILMT.

    There may be additional zip files to open, depending on the product installer that you are using.

  3. Extract the tag file that corresponds to the deployment license.

  4. Copy the tag file onto each disk where your application is installed. It can be more convenient to store the tag file in the same directory as the execution file.

  5. Remove each deployment tag file from your disk (development environment).

Example:

  1. Open <INSTALL_PATH>/properties/version/ILMT/Tags.zip.

  2. If you have an application with entitlement for IBM ILOG CPLEX Optimizer Deployment Edition:

    1. Make sure that your company acquired appropriate PVUs for IBM ILOG CPLEX Optimizer Deployment Edition.

    2. Locate the IBM_ILOG_CPLEX_Optimizer_Deployment_Edition-<release>.swidtag file.

    3. Copy this file on each server where you install your application.

    4. Remove this file from your development disk (it should only be in the zip file).

  3. If you are deploying an application with entitlement for IBM ILOG CPLEX Optimizer Single User Edition:

    1. Locate the IBM_ILOG_CPLEX_Optimizer_Single_User_Edition-<release>.swidtag file.

    2. Copy this file onto each machine where you install your application.

    3. Remove this file from your development disk (it should only be in the zip file).

Logging license consumption

IBM ILOG CPLEX Optimization Studio is licensed under several license metrics for both development and deployment. The license metrics for IBM ILOG CPLEX Optimization Studio Developer Edition are usage-based. Usage can be logged to monitor compliance with license terms.

License consumption logging is activated by setting an environment variable that points to a configuration file. This file contains information on how logging occurs and what should be logged. Once logging is activated, consumption information is written to a database when CPLEX Studio is used. The consumption records are processed into Software License Metric Tag (SLMTag) files using a script provided with CPLEX Studio. These files have the extension .slmtag and are read periodically by the IBM License Metric Tool (ILMT) once it has been configured to scan for these files. You can generate reports that summarize usage. For more information on using ILMT, see the ILMT 9.0 Knowledge Center.

Configuring and enabling license consumption logging

To enable logging of license consumption, set the environment variable CPLEX_STUDIO_ILMT_CONFIG_FILE to the location of the configuration file. For example, in a Windows Command Prompt window, enter the command: 
set CPLEX_STUDIO_ILMT_CONFIG_FILE=c:\Users\JohnDoe\ILMTconfig.xml
The variable may also be set through the Windows Control Panel.
In a Bash shell on a Unix/Linux/OS X system, enter the command: 
CPLEX_STUDIO_ILMT_CONFIG_FILE=/home/johndoe/ILMTconfig.xml

The configuration file is written in XML.

This is an example of such a file for use on Windows:
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<CPLEX_Studio_ILMT_Configuration version="1.0">
  <LicenseMetric>FLOATING_USER</LicenseMetric>
  <Location>c:\ProgramData\IBM\CPLEX_Studio\ILMT</Location>
</CPLEX_Studio_ILMT_Configuration>
This is an example for a Unix-like operating system:
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<CPLEX_Studio_ILMT_Configuration version="1.0">   
  <LicenseMetric>FLOATING_USER</LicenseMetric>   
  <Location>/var/tmp/IBM/CPLEX_Studio/ILMT</Location>
</CPLEX_Studio_ILMT_Configuration>

A schema for the file and an example file are contained in the <install_dir>\properties\version\ILMT directory of your CPLEX Studio installation. There are default values for most elements as listed below. The LicenseMetric and Location elements are required.

If your product entitlement says "Floating User Single Session", you should specify FLOATING_USER in the LicenseMetric element. If your entitlement says "Authorized User", you should specify AUTHORIZED_USER in the LicenseMetric element. License consumption tracking is not provided for other entitlements.

The value of the Location element is a directory where the database and SLMTag files are written. You must create the directory before using it, and it must be writeable by all those using CPLEX Studio with this Location and InstanceID. The directory should also be readable by the ILMT tool. The directory with its database and SLMTag files should not be removed until the ILMT scan has completed.

The InstanceID can be used to track usage by product or at a more detailed level. The default InstanceID is the CPLEX Studio version, so usage will be reported by version. Usage can be tracked per project by using the project name as the InstanceID. For example, if your organization had a project on vendor selection, the InstanceID could be VendorSelection. For the AUTHORIZED_USER metric, usage by user can be tracked by setting a username as the InstanceID.

Table 2. Elements and default values for ILMT config file
Element Type Required Default Definition
LicenseMetric AUTHORIZED_USER or FLOATING_USER Yes (none) License metric under which the software is used.
Location String Yes (none) Fully qualified path name of folder/directory where SLMTag files and the SLMTag database are written.
InstanceID String No CPLEX_Studio_<version_number> Identifies software identity instance that provides license metric data. Will appear in ILMT reports. May only contain alphanumeric characters and characters from the set: ./_-:

Writing usage records in the database

Once consumption logging is activated, CPLEX Studio creates a usage database in the location specified by the Location element if it does not already exist. Each combination of InstanceID and LicenseMetric in a given location requires its own database. The database name starts with CPLEX_Studio and is followed by a hash of the InstanceID and LicenceMetric with the extension .db.

Records are written to the database for each usage of CPLEX Studio. Usage is defined as the period when an optimizer object exists. If optimizer objects are sequentially created and destroyed with significant gaps between them, there are records for multiple usage times. Multiple objects within a single process are counted as a single usage even if they are on separate threads. The record captures the period of time during which at least one object exists. Generating SLMTag files

CPLEX Studio provides a Python script to write SLMTag files from the records in the database. The script is located in the <install_dir>\properties\version\ILMT folder.
Note: This script is compatible with Python V2.6, and V2.7. After you apply the Python 2to3 conversion tool, this script is also compatible with Python V3.4 .

Once the SLMTag files are written, the records in the database are deleted. You can schedule regular execution of the script using operating system tools such as Task Scheduler on Windows or cron on Unix-like systems. Choose a frequency of execution that will provide your desired level of detail in the ILMT reports.

To generate an SLMTag file, use the command 
python slmtag.py [-c ilmt_config_file.xml]
If the configuration file is not specified in the command, the configuration file specified by the environment variable CPLEX_STUDIO_ILMT_CONFIG_FILE is used.

The XML format used for these files is standard across IBM products so that they can all be read by ILMT. Here is an example of a file written by CPLEX Studio:

<SchemaVersion>2.1.1</SchemaVersion>
<SoftwareIdentity>
  <Name>IBM ILOG CPLEX Optimization Studio Developer Editon</Name>   
  <PersistentId>85775252b3224d50adcfb6c1401b506e</PersistentId>
  <InstanceId>CPLEX_Studio_12.6.1</InstanceId> 
</SoftwareIdentity> 
<Metric logTime="2014-05-30T09:30:10+06:00">   
  <Type>FLOATING_USER</Type>   
  <Value>12</Value>
  <Period>     
    <StartTime>2014-05-29T09:30:10+06:00</StartTime>     
    <EndTime>2014-05-30T09:30:10+06:00</EndTime>   
  </Period>
</Metric> 
<Metric logTime="2014-05-31T09:30:10+06:00">
  <Type>FLOATING_USER</Type>   
  <Value>5</Value>   
  <Period>     
    <StartTime>2014-05-30T09:30:10+06:00</StartTime>     
    <EndTime>2014-05-31T09:30:10+06:00</EndTime>   
  </Period> 
</Metric>

Each invocation of slmtag.py will read the records in the database and write one metric element to summarize those records in the SLMTag file. The configuration file specifies which SLMTag file and database are used. The Value element inside the Metric element gives the peak usage for the period of time between the StartTime and EndTime values.

In the example above, the peak usage for the FLOATING_USER metric during the one day period from on May 29th to 30th was 12 and from May 30th to 31st was 5.

The SLMTag file grows over time and may be edited to remove older metric elements. Elements should be kept long enough to be scanned by ILMT; the scanning frequency is determined by the ILMT administrator but generally it should be sufficient to keep elements for a month.

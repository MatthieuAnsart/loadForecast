New and changed in OPL V6.3

This release contains some new language features, but has mainly concentrated on polishing existing OPL features.

  • Language

  • Database Management

  • Performance improvement

Language

New features

The following OPL functions are available to obtain values from constraints in a post-processing context:

  • dual

  • slack

  • reducedCost

They are documented in the Language Quick Reference.

Language changes

  • The names dual and slack are no longer accepted as identifiers as they are now function names.

  • OPL now raises errors for undefined external data (syntax =...) even when that data is not used by the model.

Database Management

OPL V6.3 uses DB Link V5.3 for the processing of Relational Database Management Systems (RDBMS).

Performance improvement

On AIX systems, OPL 6.3 is 15% faster than the previous release.

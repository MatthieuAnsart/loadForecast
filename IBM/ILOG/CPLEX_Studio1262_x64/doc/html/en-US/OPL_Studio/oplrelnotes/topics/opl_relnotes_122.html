Release notes for CPLEX Studio V12.2

Summarizes new and changed features in the CPLEX Studio IDE and the OPL language.

The product that was formerly OPL Studio is now called CPLEX Studio, and the IDE becomes the CPLEX Studio IDE. OPL, the Optimization Programming Language, remains unchanged.

  • CPLEX Studio IDE

  • OPL run time

CPLEX Studio IDE

Migration to Eclipse 3.4.2

The IDE is now based on Eclipse 3.4.2, to take advantage of the bug fixing feature of Eclipse and some of the latest Eclipse features.

Gantt views for scheduling problems

The results of scheduling problems can be viewed in the IDE, in a Gantt chart, or other type of chart. Refer to the following documents for details:

  • Viewing the results of scheduling problems in Getting Started with the IDE.

  • Getting Started with Scheduling in CPLEX Studio.

    • Viewing the results of sequencing problems in a Gantt chart

    • Viewing the results of state functions in a Gantt chart

    • Viewing the results of cumulative functions in the IDE

More memory for solving models

The CPLEX Studio IDE and the execution of OPL projects are launched in two different processes. This enables the IDE and the project execution to go up to the theoretical maximum heap size, making more memory available for solving large models. For more information refer to Executing projects in a separate process in Getting Started with the IDE.

Save and restore of results

After executing a model, you can save the results, keep a history of the saves and restore a saved version in the IDE.

See Saving and restoring results in Getting Started with the IDE.

CPLEX solution quality

CPLEX solution quality information is now available on the Solutions tab in the IDE. For more information, refer to The Output tabsin Getting Started with the IDE.

OPL run time

Increased quality

The quality of OPL run time has increased, largely due to intensive bug fixing. The following APARs (Authorized Program Analysis Report) have been closed.

  • RS00045 Memory access, extraction, and stack overflow exceptions with multi-threaded OPL applications using Java and .NET interfaces.

  • RS00054 When a multi-dimensional array is partially initialized, accessing its elements may result in unexpected errors.

  • RS00057 Variable bounds are never relaxed when relaxation preferences are specified for a constraint.

  • RS00059 Some CPLEX parameters are missing from cplex.IntParam and IloCplex.IntParam in .NET and Java OPL interfaces.

  • RS00064 CPLEX extraction errors with piecewise linear functions whose variable expression includes z/c where c is a constant

  • RS00067 Memory leak when ending OPL factory objects.

  • RS00087 A diff constraint such as x + constant != y behaves like x != y .

  • RS00094 Initializing tuples in ternary expressions may result in incorrectly inferred tuple types and errors using these tuples.

  • RS00124 Using the frac function inside a dexpr may result in incorrect output.

  • RS00126 Error when initializing a tuple from a subtuple.

  • RS00158 When a tuple field is a string set, its values cannot be accessed in a Java or .NET API.

  • RS00166 IloCP internal error: Cannot replay solution when using the scheduling function last.

  • RS00165 Allow changing the array size for Oracle OCI connections.

Multi-threading

A Java example is provided for high volume applications that process many OPL models in parallel. The example shows how to build a high-volume, client-server like architecture so that models can be solved faster in a multi-threaded environment.

The Java example, ConcurrentProcessing.java, shows OPL multi-threading capabilities in a financial application. It can be found in the installation directory:

<Install_dir>\opl\examples\opl_interfaces\java\ConcurrentProcessing\src\ConcurrentProcessing

The example is documented in High volume processing with multiple threads, in Language and Interfaces Examples.

Database

DB2 database support has been added for AIX platforms.

Support for OPL run time on Windows 7

Windows 7 is supported for OPL run time.

Transition based scheduling

CPLEX Studio V12.2 provides new integer expressions and constraints that allow for easier modeling of problems that involve constraints and costs related to transitions between activities executed on a unary resource. For instance, it is now possible to model transition costs or transition times that hold between immediate successors in a sequence. Furthermore, constraints between sequence variables, such as in permutation flow-shop problems, are now easier to handle. For more information on the transition based scheduling API, refer to:

  • The scheduling examples, in Transition based scheduling in the Language and Interfaces Examples.

  • The following OPL scheduling functions:

    startOfNext, startOfPrev, endOfNext, endOfPrev, lengthOfNext, lengthOfPrev, sizeOfNext, sizeOfPrev, typeOfNext, typeOfPrev.

    in the Language Quick Reference.

  • Sequencing of interval variables in the Language Reference Manual.

Incompatibilities

  • Modeling

    • An error occurs if an integer overflow is detected.

    • The order of precedence is fixed for the OPL ternary operator ?:.

    • An exception is raised when the transition distance extracted for a state function does not satisfy the triangular inequality.

  • Scripting

    • More type checking has been added to the scripting of OPL types. For example, an error occurs if you assign an int to a float, or a Date to an int.

    • A new API has been created for IloOplInputFile and IloOplOutputFile. These classes can be found in the IBM ILOG Script Reference Manual.

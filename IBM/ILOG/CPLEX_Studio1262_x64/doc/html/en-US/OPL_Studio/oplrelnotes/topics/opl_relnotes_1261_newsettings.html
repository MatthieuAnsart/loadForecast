New settings

New settings are available in the IDE settings editor, in the APIs, and in OPLScript.

The settings are provided for:

  • the linearization of binary variables in the case of QPs
  • random seed for CPLEX
  • deterministic time spent tuning
  • deterministic time spent probing
  • MIP local implied bound cuts
  • deterministic time before starting to polish a feasible solution
  • the export of external and internal data

The settings editor

  • The settings Linearization of binary variables for QPs and Random seed are included in the options for Mathematical Programming > General.

    • The qtolinind parameter switches on or off linearization of the quadratic terms in the objective function of a quadratic program (QP) or a mixed integer quadratic program (MIQP) during preprocessing.

      For more information, refer to the CPLEX document: linearization switch for QP, MIQP.

    • The randomseed parameter makes it possible for your application to manage the random seed that CPLEX uses in some of its internal operations. Variation in the random seed can increase diversity of results.

      For more information, refer to the CPLEX document: random seed

  • The setting Deterministic time spent tuning is included in the options for Mathematical Programming > Tune.

    The tuningdettilim parameter sets a time limit in deterministic ticks per model and per test set (that is, suite of models) applicable in tuning. For more information, refer to the CPLEX document: deterministic tuning time limit.

  • The setting Deterministic time spent probing is included in the options for Mathematical Programming > Mixed Integer Programming > Limits.

    The probedettime parameter limits the amount of time (expressed in deterministic ticks) spent probing. For more information, refer to the CPLEX document: deterministic time spent probing.

  • The setting MIP local implied bound cut generation switch is included in the options for Mathematical Programming > Mixed Integer Programming > Cuts.

    The localimplied parameter instructs CPLEX whether or not to generate locally valid implied bound cuts for the model. For more information, refer to the CPLEX document: MIP locally valid implied bound cuts switch.

  • The setting Deterministic time before starting to polish a feasible solution is included in the options for Mathematical Programming > Mixed Integer Programming > Solution polishing.

    The parameter polishafterdettime sets the amount of time, expressed in deterministic ticks, to spend during a normal mixed integer optimization, after which CPLEX starts to polish a feasible solution. For more information, refer to the CPLEX document: deterministic time before starting to polish a feasible solution.

  • The settings, Export external data and Export internal data are available in the options for Language > Run.

These parameters set the file names used by OPL to export external and internal data.

The APIs and OPLScript

  • In the Java and C++ class, IloOplSettings, and in the .NET class OplSettings:
    • the method setExportExternalData sets the file name used by OPL to export internal data
    • the method setExportInternalData sets the file name used by OPL to export external data.
  • In the OPLScript class, IloOplSettings, the methods are:
    • exportExternalData
    • exportInternalData

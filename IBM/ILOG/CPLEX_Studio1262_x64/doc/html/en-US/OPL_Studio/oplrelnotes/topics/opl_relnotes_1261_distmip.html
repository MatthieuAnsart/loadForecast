Using the CPLEX distributed MIP algorithm with OPL

You can use the CPLEX distributed MIP algorithm to solve a standalone OPL model from the IDE, from the OPL APIs, or from OPLScript.

Using distributed MIP from the OPL IDE

An OPL example, distMIP, illustrates the use of distributed MIP. This example can be found in your installation of CPLEX Optimization Studio. The default installation for Windows is:

C:\Program Files\IBM\ILOG\CPLEX_Studio<version>\opl\examples\opl\distMIP

To import the distMIP model into the IDE:

File > Import > Example > IBM ILOG OPL Examples > Sorted by Technique > Integer Programming > distMIP

The example contains a VMC file, process.vmc, and a settings file life.ops, both of which need to be modified.

The three run configurations illustrate the various ways in which the VMC file can be specified:

  • including it in a settings file
  • reading it with a CPLEX method
  • including it in an OPLScript block

A tutorial on this subject, Using the CPLEX distributed MIP algorithm with OPL, provides more detail.

Using distributed MIP from the APIs and OPLScript

Several examples named distMIP are available from the OPL example libraries to show how to use the distributed MIP.

These examples can be found in your installation directory, for example:

C:\Program Files\IBM\ILOG\CPLEX_Studio<version>\opl\examples\opl_interfaces

The methods readVMConfig, hasVMConfig, delVMConfig, and copyVMConfig are available in C++, Java, .NET (from the CPLEX C++ library, OPL jar and OPL .NET DLL), and in OPLScript.

IDE connection to CPLEX servers

From the IDE in CPLEX Studio, you can:

  • Connect to CPLEX servers
  • Execute single or multiple run configurations on a CPLEX server
  • Monitor the jobs running on a server
  • Restore the solution of a successful job from a server

The command Window > Preferences > CPLEX Servers opens the CPLEX Servers wizard, which enables you to add, remove, and edit a server.

For more information see IDE connection to CPLEX servers.

The IDE is available in new languages

The 12.5 CPLEX Studio IDE is also localized in Russian, Turkish, Polish and Brazilian Portuguese.

Multi-criteria objective

CP Optimizer can solve an OPL model with a multi-criteria objective, in the IDE and with the oplrun command.

New filtering behavior

The behavior of filtering has changed. If you initialize an object with an index that is out of bound, you no longer get an error message and the out of bound is ignored.

The following example contains a model with an index that is out of bound.

The .mod file:

tuple t
{
 int x;
 int s[1..3]; 
}
{t} se=...;
{t} a={i |i in se :1 == i.s[-1]};
execute
{
   writeln(a); 
}

The .dat file:

se={<1,[1,2,3]>,<3,[2,3,4]>};

The result of the instruction writeln(a) is an empty set.

Conflict management by CP Optimizer

CP Optimizer now supports conflict detection and conflict refiner control. These functions are no longer restricted to CPLEX.

To activate the conflict refiner, use the command oplrun with the option -conflict.

In the IDE, the conflict refiner control is automatic.

For more information, see Conflict refiner.

Solving models in batch mode and comparing KPIs

Three examples show how to use ILOG Script to solve models with multiple data sets and generate an HTML report to compare the resulting KPIs.

The batch project used to illustrate this feature can be found in your installation, for example:

C:\Program Files\IBM\ILOG\CPLEX_Studio<version>\opl\examples\opl\batch

The examples can be adapted for your own models and data sets, and you can also specify your own KPIs.

The HTML reports from the two advanced examples highlight the best and worst KPIs. The batchAdvancedCP example shows how to tune a specific CP model by changing the value of certain parameters, using the code engine.param.randomSeed. You use a random seed value for each run, and then compare the results.

See Solving in batch mode and comparing KPIs for details.

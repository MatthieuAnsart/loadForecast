Release notes for the IDE and OPL V12.6.2

These notes summarize new and improved features in the IDE and the OPL modeling language.

Cloud options in the IDE

Many of the OPL models have been adapted to run on the cloud. The models can be found in your installation of CPLEX Optimization Studio, for example:

C:\Program Files\IBM\ILOG\CPLEX_Studio1262\opl\examples\opl

The run configuration can contain zero or any number of plain .dat files (that is, .dat files with no Excel statement, or no DB Link statement). The code in a standalone .mod file must be cloud compliant (it does not require access to your disk).

Before you can run a model, you must register on a server. In the IDE, select Window > Preferences > OPL > DOcloud.

For more information, see Solving OPL models on the cloud.

New CPLEX parameter in the IDE

The CPLEX parameter CPX_PARAM_BQPCUTS controls the generation of Boolean Quadric Polytope (BQP) cuts when CPLEX solves a nonconvex quadratic program (QP) or a nonconvex mixed integer quadratic program (MIQP) to global optimality.

To see this parameter in a settings file, select Mathematical programming > Mixed Integer Programming > Cuts.

For more information, see the CPLEX topic Boolean Quadric Polytope cuts.

New SPSS model

A new SPSS project has been added to the distributed samples. It can be found in your installation of CPLEX Optimization Studio, for example:

C:\Program Files\IBM\ILOG\CPLEX_Studio1262\opl\examples\opl\SPSS\marketing

The project contains the model marketing_plan.mod, two data files, and an SPSS stream.

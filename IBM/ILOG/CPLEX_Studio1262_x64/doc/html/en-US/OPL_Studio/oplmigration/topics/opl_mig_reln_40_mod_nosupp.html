No longer supported

Lists the OPL 3.x features that are not supported in OPL 4.0.

Constraint programming

OPL Development Studio 4.0 does not support constraint programming. If you have used an earlier version of OPL Studio for constraint programming, please send email to oplcp@ilog.com for information about support for constraint programming in a future version of OPL Development Studio.

Hence, the following ILOG Solver and ILOG Scheduler features are not available:

  • predicates

  • higher order constraints (reification)

  • specialized constraints, such as alldifferent, distribute, atleast, atmost, etc.

  • search programming

  • all the ILOG Scheduler features such as Activity, DiscreteResource, precedes constraint, requires constraint, etc.

To develop constraint programming models, please use an earlier version of OPL Studio.

DBMapping

The syntax for DBmapping is no longer supported. You can easily write this instruction using the SQL SELECT syntax.

Data types

The following data types are not supported:

  • character

  • enum. Use {string} instead. (See Enumerated values for an example.)

Interactive input

The syntax for user interaction, like this:

int n << "Please, enter the value of n:";

is no longer supported.

Flat files

Data initialization using flat files like this:

int array[1..N] < "file.txt"

is no longer supported. See Flat files.

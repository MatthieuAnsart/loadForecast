The .NET API

Introduces the .NET namespaces and discusses integrating OPL with Microsoft Office.

The .NET interface offers basically the same functionality as the C++ one.

The .NET API is split into three namespaces:

  • The ILOG.OPL namespace contains the OPL control API, for loading and accessing models.

  • The ILOG.Concert namespace contains the Concert modeling API, for modifying models.

  • The ILOG.CPLEX namespace contains the CPLEX control API, for controlling the solving process.

This API is available for all the languages supported by the .NET platform. Examples are provided with OPL for C#, J#, and Visual Basic.

The .NET API is written as wrappers on the equivalent C++ libraries and offers the same functionality and performance. This means that there is no loss of performance when solving models whatever language you choose. However, the adaptation from C++ to .NET has some consequences on the way you can use either API. See the The .NET interface in the Integration Libraries User’s Manual for details.

Integrating OPL with Microsoft Office 2003

OPL 3.7 provided a COM component library that allowed users to use a small part of OPL API over COM enabled clients like Microsoft Office. Since then, Microsoft has been encouraging developers to migrate to the .NET platform.

While it is technically possible to create a COM wrapper around the .NET API for OPL, ILOG does not recommend this approach. Instead, ILOG recommends using Microsoft Visual Studio Tools for Office 2003. This allows you to extend Microsoft Office applications such as Microsoft Excel with OPL .NET API. This is demonstrated in the ExcelWarehouse example delivered in

<installation-directory>/examples/dotnet/ExcelWarehouse

See Integrating OPL with Excel using Visual Studio Tools for Office 2003 in the OPL 4.0 Integration Libraries User’s Manual.

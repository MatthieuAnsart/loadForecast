Introduction

Introduces the separation between model and data.

OPL 4.0 is based on a clearer separation between model and data. This allows you to start development with a simple data file and easily convert to external data sources later on. The facilities for creating and modifying data have been moved to data files (.dat), including read and write references to databases and spreadsheets. However, inline initializations are still allowed in model files:

int n=5;
Important:

In contrast to OPL 3, the order in which data files are given to OPL 4 is meaningful. Data needs to be defined in the order in which the elements are declared in the model file.

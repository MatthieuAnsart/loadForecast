Testing the default search

Explains how to test the default search on a migrated model.

About this task

To test your migrated model:

Procedure

  1. Run the model.

    If it finds good solutions quickly, then you do not need to write search phases.

    If it does not find good solutions, try the other search methods provided by ILOG CP Optimizer engine.

  2. Add a settings file (.ops) to your run configuration.

    See Adding a settings file in Getting Started with the IDE if you need help.

  3. Use one or more settings files to try the various search types available in the Constraint Programming/Search Control/General section of the settings editor.
  4. In particular, in Constraint Programming / Propagation Control, change the default inference level. If this doesn't give a better result, reset it.
  5. If none of them works well, click to revert to the default setting (Restart), then proceed to the Defining search phases stage.

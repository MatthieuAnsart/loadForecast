Converting an OPL 4.x non-CP model

Provides a procedure to convert a basic OPL 4.x model to an OPL 6.x model.

About this task

Here is a procedure to help users get started with converting an OPL 4.x model and become familiar with OPL 6.x features.

Procedure

To convert an OPL 4.x model:

  1. Make a copy of your model (.mod) and data (.dat) files, and load them into a new OPL 6.x project.

    See Creating a project in Getting Started with the IDE if you are not familiar with the OPL 6.x graphical user interface.

  2. If you had settings in your OPL 4.x project, add an empty settings (.ops) file to your OPL 6.x project and copy the settings you had in your previous project to this settings file.

    See Adding a settings file in Getting Started with the IDE if you are not familiar with creating a settings file.

  3. Test your model to make sure it runs “as is.”
  4. If it does not, comment out the offending sections (to pinpoint the statements where the errors lie), and then check the following sections of the OPL Release Notes to see if there were language changes since your OPL project was first created:

    • New and changed in OPL 5.0

    • New and changed in OPL 6.1.1

    • New and changed in OPL 6.2

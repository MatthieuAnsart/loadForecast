Disjunctive resources

Explains how to model disjunctive resources in OPL 6.x.

Disjunctive resources are those that can execute only one task at a time. In OPL 3.7, the special resource type UnaryResource was used. In OPL 6.x, the noOverlap constraint is used.

In OPL 3.7

  int nbOps = ...;
  int ProcessingTime[1..nbOps] = ...;
  Activity operations[i in 1..nbOps](ProcessingTime[i]);
  UnaryResource machine;
  solve {
    forall(i in 1..nbOps)
      operations[i] requires machine;
  };

In OPL 6.x

  using CP;
  int nbOps = ...;
  int ProcessingTime[1..nbOps] = ...;
  dvar interval operations[i in 1..nbOps] size ProcessingTime[i];
  constraints {
    noOverlap(operations);
  }

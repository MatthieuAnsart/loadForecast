New and changed in OPL 4.0

Presents the new features and the changes to OPL 4.0 affecting the language, the IDE, and the libraries.

  • Separation between data and model
    Describes how data/model separation in OPL expresses itself in databases, spreadsheets, flat files, and data blocks in model files.
  • The modeling language
    Describes the changes made to the syntax and types to improve the consistency of the modeling language.
  • Scripting
    In OPL 4.0, the OPL Script language is replaced by an implementation of ECMA-262 (also known as ECMAScript or JavaScript) named “ILOG Script”, with extensions for OPL. This section explains how the replacement of OPL Script by ILOG Script affects your way of writing script statements: abstract models, preprocessing, postprocessing, migration, setting CPLEX parameters, open arrays...
  • The IDE
    Describes new and modified features in the user interface.
  • The libraries
    Lists which classes are deprecated and which are now available via the ILOG Concert API, and gives an introductory description of the Java and .NET interfaces.

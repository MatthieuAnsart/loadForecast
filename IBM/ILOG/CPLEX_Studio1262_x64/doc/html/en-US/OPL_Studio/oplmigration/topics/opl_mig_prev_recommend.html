Recommended migration paths

OPL projects created in previous releases have specific migrations paths to version 6.x.

There are basically two paths for migrating your existing OPL projects into OPL 6.x.

Because of changes to the OPL language and/or syntax and the different ways that projects themselves were structured in different releases, these paths are dependent on the version used to create the original projects and the nature of the projects themselves.

  • Migrating your OPL 5.x projects — step by step procedures for how to use the OPL 6.x Import Wizard to migrate the OPL and ODM projects you created using OPL 5.x

  • Migrating your OPL 3.x and 4.x projects — a set of general guidelines for how to easily re-create your OPL 4.x and 3.x projects in the OPL 6.x IDE.

Migrating OPL 5.x projects

The OPL 6.x Import Wizard allows you to easily migrate your OPL 5.x projects and create the proper folder structure for them in OPL 6.x.

The Import Wizard also allows you to migrate your OPL-ODM projects. When you do this, your ODM-specific files, settings, data sources and, JAR files containing customizations are also migrated and placed in the same folder. Guidelines have been provided in the following section to help you with this process:

  • Importing OPL 5.x projects

Note:

One of the features of the Import Wizard is the ability to merge OPL projects as you migrate. That is, if you have separate OPL 5.x projects that share model, data or settings files, you can import them into the same folder and treat them not as separate projects but as one project with multiple run configurations. For technical reasons, this feature is not possible when importing OPL-ODM projects.

Migrating OPL 4.x and 3.x projects

Because of language and product changes between versions, projects created in OPL 3.x or OPL 4.x should be re-created as new projects in OPL 6.x. Guidelines have been provided in the following sections to help you with this process:

Compatibility

Lists the syntax that is maintained in OPL 4.0 for compatibility reasons.

The following syntax is maintained for compatibility reasons.

Arrays

For compatibility reasons, OPL 4 still support the [,] syntax for array declarations and expressions. See Arrays.

Delimiters

Also for compatibility reasons, quotes for string values and commas are optional in data files.

struct

The keyword struct is deprecated and will be removed in future releases. However, it is supported for compatibility reasons: using it brings up an error message, but your code remains valid. See tuple instead of struct.

Modulo

In OPL 4, the modulo operator is noted % but you can also use the keyword mod.

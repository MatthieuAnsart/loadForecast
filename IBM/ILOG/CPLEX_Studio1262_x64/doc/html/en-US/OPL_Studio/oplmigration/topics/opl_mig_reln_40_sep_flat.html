Flat files

Describes how to convert data from an OPL 3 flat text-file so that it can be handled by OPL 4

About this task

Data from text files is handled differently in OPL 4.

In OPL 3, you could initialize data by loading a text file, like this, for example:

aStruct myArray[anIndexer] < "flatFileForValuesOfMyArray.txt";

In OPL 4, this <"file" syntax is no longer supported. However, you can convert your existing OPL 3 flat text-file data by using the dump feature in OPL 3. Starting from the example above, do as follows.

Procedure

To convert an OPL 3 flat text-file data:

  1. Add at the end of the model: 
    cout << "myArray=" << myArray << ";" ;
  2. Solve the model.
  3. Execute the Dump active Model and Result command.
  4. Rename the generated file with a .dat extension.
  5. Load that file into your favorite editor and remove the model and result part, keeping only the data.

Results

This data file can now be used by OPL 3 and OPL 4.

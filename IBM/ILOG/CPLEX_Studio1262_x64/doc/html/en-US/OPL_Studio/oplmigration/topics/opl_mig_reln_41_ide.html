The IDE

The changes to the IDE affect project debugging and various parts of the main window.

Project debugging

  • Relaxation search: when a model yields no solution because it is infeasible, the Relaxations page of the Output window displays what variables and/or constraints you can relax. See Tutorial: Debugging Infeasible Models in the IDE User’s Manual.

  • Profiling: the Profiler page of the Output window displays in tabular form the time and memory consumed for each execution step to help you identify where you can improve your model. See Tutorial: Profiling the Execution of a Model in the IDE User’s Manual.

The Main window

Several new or changed functionalities reflect in the new default layout of the Main window:

  • The Workspace window now contains only the project tree and is therefore renamed to Projects.

  • The Model Browser panel becomes a separate window under the Projects panel.

  • Two new pages in the Output window: Relaxations and Profiler. See Project Debugging.

See The Main Window in the IDE User’s Manual for more information.

The Start Window

  • The number of recent files is customizable via the User Interface/Start panel of the General Settings dialog box.

Projects and models

  • Standalone models are still allowed but if you want to customize their options, you have to include them in a project.

  • The New Project dialog box enables you to create new projects in a more integrated way. See Creating a Project in the IDE User’s Manual.

  • Data files are now loaded by default in the project as soon as you add them. Just double-click on the data file name in the Projects window to display its contents in the Editing area.

  • In projects that include more than one data file, you can now change the order of data files from the Projects window.

  • The File menu now includes the Generate External Data File and Generate Calculated Data File commands. See Menu Commands and Equivalent Toolbar Buttons in the IDE User’s Manual.

  • The configuration file oplst.config is saved to Documents and Settings/username/Application Data/ILOG/OPL Studio IDE/4.1. The settings.xml file reflects the IDE general settings.

The Editing area

  • By default, line numbers appear in the text editor. However, you can hide them via the General Settings dialog box. See Editor Options in the IDE User’s Manual.

The Model Browser

  • The Model Browser now has a Property table which dynamically displays the solution values for the selected model element.

  • The former right-click command Open View is now called Open Value View and enables you to display the values of a model element. If you select a named constraint, the Property table shows its dual and slack values. See Using the Model Browser and Tutorial: Progress Chart in the IDE User’s Manual.

Project execution

  • The syntax error arrow is now red instead of blue. See How to Check for Syntax and Semantic Errorsin the IDE User’s Manual.

  • The solution notifier is no longer a window but a “balloon” in the Microsoft style.

    Figure 2. New notifier

  • It is no longer necessary to click Abort to stop execution while keeping the solutions displayed. At the end of execution, the IDE automatically returns to idle mode and the execution output remains displayed in the pages of the Output window. Click the Abort button only if you really want to stop execution before it is completed (in debug mode, for instance). See Executing the Project in the IDE User’s Manual.

Options

  • The Default Options dialog box is replaced by General Settings, which contains the Editor options formerly available as Project Options. See Customizing the IDE in the IDE User’s Manual.

  • The Apply button has been removed in the General Settings (formerly Default Options) and Project Options dialog boxes. To confirm a setting, just click OK.

  • The Read option panel has been added to the Mathematical Programming section of the Project Options dialog box. See Table 10.2 in the IDE User’s Manual.

  • In the OPL Language/General section of the Project Options dialog box:

    • The Big map threshold option has been renamed to Big map naming threshold for more clarity. See OPL Language: General Options in the IDE User’s Manual.

    • Two new options: Display relaxations and Collect profiling information. See Tutorial: Profiling the Execution of a Model Tutorial: Debugging Infeasible Models in the IDE User’s Manual.

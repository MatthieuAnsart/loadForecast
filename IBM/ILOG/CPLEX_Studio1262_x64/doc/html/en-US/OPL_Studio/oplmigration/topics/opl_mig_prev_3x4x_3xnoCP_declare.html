Declaration of data and variables

Lists the changes to make to data and variable declarations in an OPL 3.x model for compatibility OPL 4 and later

  1. Replace the keyword var with dvar. For example, write

    dvar float+ production[Products];

    instead of

    var float+ production[Products];

  2. Replace enum types by a set of strings {string}. See Enumerated values.

  3. Use the assignment operator = to declare ranges. For example, write

    range Workers = 1..nbWorkers;

    instead of

    range Workers 1..nbWorkers;

    Remember also that in OPL 4, a range is a value, no longer a type. See Ranges for details.

  4. Use the types float+ and int+ for decision variables only. See float+, int+ for details.

  5. No longer use grouped declarations. See Grouped declarations.

  6. Instead of relying on implicit type conversion, use the asSet method. For example, write

    {int} intSet = asSet(1..13);

    instead of

    {int} intSet = 1..13;

    See Conversion from a range to a set of integers for details.

Data block in model files

Describes how to handle data blocks in OPL 4.0

In OPL 3, you wrote:

data {
  nbMetals = 3;
  nbRaw = 2;
  costMetal = [22, 10, 13];
  costRaw = [6, 5];
};

because OPL 3 offered the possibility to include the equivalent of the data file in one or several data blocks at the end of the model file.

In OPL 4, this feature is no longer supported. You have to export the contents of data blocks (that is, what is enclosed between curly brackets) to one or several data files.

Also, the named tuple component syntax #<,># is not available for internal initialization in OPL 4.0. If you move the initialization to the .dat file, it works fine.

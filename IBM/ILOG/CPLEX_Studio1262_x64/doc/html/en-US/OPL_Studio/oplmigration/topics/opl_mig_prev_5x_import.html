Importing OPL 5.x projects

This section explains how to migrate your OPL 5.x projects into OPL 6.x

About this task

Once you have launched the OPL 6.x IDE, your next step will probably be to import one or more of your existing OPL 5.x projects. This section provides instructions for how to do this.

Note:

There is a conversion process that takes place during the importation of OPL 5.x projects into the OPL 6.x IDE. The first time you import a project, you use the Import > Migrate OPL 5.x projects menu command to load the project into your OPL Projects Navigator and convert it to the OPL 6.x format.

If you are importing the examples shipped with OPL 6.x, you use a different menu command, Import > Existing OPL 6.x projects. You also use this command to reload a project that you have previously converted and then deleted from your OPL Projects Navigator. See Working with projects in the OPL Projects Navigator for more information on this command.

In addition to the procedures detailed below, you can double-click on the files contained in an OPL 5.x project folder to start the migration of that project in the OPL IDE:

  • Double-clicking any OPL 5.x model (.mod), data (.dat), or settings (.ops) file will launch the Import Wizard discussed in the following procedures.

Procedure

With the OPL 6.x IDE running:

  1. In the main menu, choose File > Import.

    You can also right-click in the OPL Projects Navigator and choose Import to display a shortcut menu with the same options.

  2. Select Migrate OPL 5.x projects.

    The first screen of the Import Wizard is displayed:

    This screen can be used to import one or more OPL 5.x projects into OPL 6.x. The general procedure for doing this is to:

    • Select a root directory (this is the directory on your file system where your existing OPL project is located).

    • Select a project (or projects, if the root directory contains more than one project) to be imported.

    • Select a destination directory for your migrated OPL project.

    • Enter a name for the new folder or folders.

    • Indicate whether you want to import all files from the root directory when importing.

    The next steps walk you through each of the general procedures above, using the example of importing the C:\ILOG\OPL55\examples\opl\warehouse example from OPL 5.5.

  3. In the From directory field, enter the path name of the directory that contains the project(s) you want to import. You can type in the path name or use the Browse button to search for it.

    After you have selected a root directory, all of the OPL projects under that directory are listed in the Projects panel.

  4. Check the box of each of the projects you want to import. (In the example used in this procedure, there is only one, warehouse.prj.)
  5. In the Copy to directory field, enter the path name of the directory you want to import this project into. You can type in the path name or use the Browse button to search for it.
    Note:

    When you enter the name of the target project into the Into one new project field in the next step, that name will automatically be created as a folder name under the directory you specify in the Copy to directory field when you click the Finish button.

    For example, if your Copy to directory field specifies a path of C:\OPL_projects and your Into one new project field specifies a project name of warehouse, the new OPL 6.x project will be created in the C:\OPL_projects\warehouse folder.

    If the target directory you want to use for the Copy to directory field does not yet exist, you can use the Make new directory button on the popup window to create one.

  6. The Into one new project field is used to indicate whether you want to import all of the OPL projects you have checked in the Projects pane into one folder, or into separate subfolders for each project.

    • If you are importing a single project, enter the name of the project in the Into one new project field.

    • If you are importing more than one project and you want to merge projects (import all of the selected projects into one project folder), enter the folder name of the final project in the Into one new project field. The Import Wizard will migrate the projects to the new folder and create separate run configurations for each of them.

    • If you are importing more than one project and want to keep them separate, continue to step #7.

      Important:

      If you are importing multiple OPL-ODM 5.x projects (that is, projects that contain an ODM application), or ODM project folders that contain more than one sub-project in their top-level folders, you cannot use this merge feature.

      When you attempt to use the Into one new project option, an error message appears at the top of the window and the Finish button never becomes active.

      In this situation, you should use the Into a new project for every selected item option.

  7. If you are importing more than one project and you want to keep them as separate projects, click the Into a New Project for each selected item button. If you choose this option, each of the imported projects will automatically take the name of the original .prj file.
  8. Under the Options part of the window, in most cases you will check the Import all other files box to include any other project files. Leave it empty if you intend to re-create or discard any supplementary project files.

  9. When all required fields are completed, click Finish.

Migration from OPL 3.x (CP projects)

Provides a quick-reference list of the changes you need to make to an existing OPL 3.x model that uses Constraint Programming (CP) for it to be compatible with OPL 6.x.

  • Converting an OPL 3.x CP model
    Provides a procedure to convert an OPL 3.x CP model to an OPL 6.x model.
  • Data preprocessing
    Explains how lazy instantiation affects the migration of an OPL 3.x model to OPL 6.x.
  • Constraints, functions, and operators
    Describes the keywords affected by the changes of syntax to express constraint programming.
  • Declaring search phases
    Once you have ported the data declaration and the constraints of your OPL model, verify whether the default search yields a satisfying performance and result or whether you need to declare search phases.

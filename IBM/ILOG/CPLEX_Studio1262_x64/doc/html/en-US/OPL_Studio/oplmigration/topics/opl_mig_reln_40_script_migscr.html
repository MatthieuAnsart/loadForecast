Migrating script statements from OPL 3 to OPL 4

Uses the mulprod example to illustrate scripting in OPL 4.

In OPL 3

Scripting meant working with three kinds of files:

  • a model file, theModel.mod

  • a script file, theScript.osc

  • optional data files

In OPL 4

Model and data files still exist, but scripting is now integrated into the model file syntax. The statements affecting model management are included in the main function. So, you can choose to embed scripting into your model, or you can create a separate model file that provides scripting for other model files.

This section uses the mulprod example to illustrate scripting in OPL 4 versus OPL 3. This example shows how to solve multiple instances of a model. This is the OPL 4 version of the mulprod1.osc script in the OPL 3 distribution.

Migrating these script statements from OPL 3 to OPL 4 consists in:

  1. writing the OPL 4 version of the model mulprod.mod.

  2. declaring the main function:

    main {
}

  3. writing the counterpart of the OPL 3 script file in the body of the main function.

Background

Provides a summary of the design history of OPL.

The following sections cover the design history of OPL, to provide some background information on why migration of projects is necessary.

OPL Studio 3.x

ILOG OPL Studio 3.x supported both mathematical programming (MP) optimization based on the ILOG CPLEX solving engine and constraint programming (CP) based on ILOG Solver and ILOG Scheduler. The engine (CPLEX or Solver) was chosen automatically, whether the problem was linear or not. There was a sophisticated language in OPL 3.x with which to write search procedures.

ILOG OPL Development Studio 4.x - ODMS

April 2005: ILOG released ILOG OPL Development Studio 4.0, the foundation for ILOG Optimization Decision Management System (ODMS). The main theme for this release was to turn ILOG OPL Development Studio into a scalable optimization development tool that can be used for prototyping models, deploying models in production, and maintaining deployed applications. In this first step, only the CPLEX solving engine was supported.

Architectural changes introduced some changes into the language:

  • Only CPLEX was supported in this release. Quadratic programming was supported (QP, QCP, MIQP, MIQCP).

  • The scripting language used to chain models or manipulate data in a procedural way, OPL Script, was replaced by a standard scripting language, JavaScript.

  • A better separation between data and model for all data sources (flat files, Excel files, databases) led to some changes in the data declaration of a model.

  • Minor changes were introduced in the language, such as:

    • struct was renamed tuple.

    • var was renamed to dvar for decision variables, to avoid a confusion with var for JavaScript variables.

    • = when used in comparisons, was replaced by ==.

See New and changed in OPL 4.0, Issues handled in OPL 4.0.1, New and changed in OPL 4.1, and New and changed in OPL 4.2 in the Release Notes for a complete list of changes.

ILOG OPL Development Studio 5.0 - OPL-ODM integration

July 2006: ILOG integrated ILOG Optimization Decision Manager 2.0 with ILOG OPL Development Studio 5.0. ILOG Optimization Decision Manager is a specialized application development tool that helps users build and deploy decision support applications, putting the power of optimization into the hands of business people. In that release, ILOG OPL Development Studio also supported logical constraints as managed by the CPLEX solving engine.

See New and changed in OPL 5.0, New and changed in OPL 5.1, and New and changed in OPL 5.2 in the Release Notes for a complete list of changes.

ILOG OPL Development Studio 5.2 - new CP support

May 2007: ILOG introduced constraint programming back into OPL Development Studio. The main theme of this release was to make CP as easy to use as MP by providing a model and run development process while keeping good performance. To this end, a tunable default search has been developed. In this release, only “core constraint programming” was supported: arithmetic constraints, specialized constraints, and logical constraints on discrete decision variables. Detailed scheduling, that deals with time intervals and resources, was not supported yet.

A second wave of changes was introduced, compared to the OPL 3.x language:

  • If you are writing a CP model, you must start it with the statement

    using CP;

    If you are writing a CPLEX model, you can declare

    using CPLEX;

    but this is not mandatory since by default, OPL considers that the model uses CPLEX.

  • There is no search language. Instead, the default search can be tuned, when needed, by specifying search phases.

  • The set of constraints and their spelling has slightly evolved.

See New and changed in OPL 5.5 in the Release Notes for a complete list of changes.

ILOG OPL Development Studio 6.x - new IDE and CP Scheduling

May 2008: ILOG reintroduces CP Scheduling back into OPL Development Studio, along with an all-new graphical user interface in the OPL IDE, and numerous other improvements.

See New and changed in OPL 6.0 in the Release Notes for a complete list of changes.

Keywords and other name changes

Lists name and syntax changes in OPL 4.

Parent topic: The modeling language

dvar instead of var

Decision variables are noted with the keyword dvar to avoid confusion with the ILOG Script keyword var. Compare:

In OPL 3

You wrote:

var float+ production[Products];

In OPL 4

You write:

dvar float+ production[Products];

round instead of nearest

The function nearest has been renamed to round. Compare:

In OPL 3

You wrote:

float f1 = 0.6;
float f2 = nearest(f1);

In OPL 4

You write:

float f1 = 0.6;
float f2 = round(f1);

tuple instead of struct

The keyword struct has been renamed to tuple. Use this new keyword to declare a tuple schema. The keyword struct is deprecated and will be removed in a future release. Compare:

In OPL 3

You wrote:

struct Neighbors {
  int row;
  int col;
};

In OPL 4

You write:

tuple Neighbors {
  int row;
  int col;
}

The semi-colon ; after the final closing curly bracket } is no longer required.

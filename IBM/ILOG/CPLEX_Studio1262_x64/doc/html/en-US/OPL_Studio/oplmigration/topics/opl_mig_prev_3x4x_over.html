Overview and general migration procedure

A general discussion of the issues involved when re-creating your OPL 4.x and 3.x projects in the OPL 6.x IDE.

There have been many changes and improvements in the OPL language and syntax over the years, some of them so fundamental that it is not possible to create an Import Wizard that handles all eventualities.

For example, OPL 4.0 introduced a separation between the model and the data used for that model that was not possible in previous releases. Similarly, with each release there have been enhancements to the OPL language; some syntax elements have been deprecated and others have been newly created.

As a result, we recommend that you migrate the models that you created in OPL 3.x and 4.x by re-creating them in the OPL 6.x IDE.

These guidelines, plus the more detailed information in the following sections, will help you when re-creating your projects.

General procedure for migrating OPL 4.x and 3.x projects

  1. Create a new project in OPL 6.x, using the procedures detailed in the Creating a project section of Getting Started with the OPL IDE.

  2. Import (or paste) your existing model (.mod into the OPL editor and, with the assistance of the syntax checker, update them to comply with OPL 6.x syntax.

    The information in the following sections will help you to pinpoint the things you need to change, for non-CP models, CP models, and CP Scheduling models.

  3. If your existing project contains a data (.dat) file, import (or paste) it into the OPL editor and, again with the assistance of the syntax checker, update it to OPL 6.x levels.

  4. Following the instructions in the Executing a project sections of Getting Started with the OPL IDE manual, create or re-create run configurations and/or settings (.ops) files for your project.

  5. Save the project in your workspace.

Guidelines for how to handle the different types of OPL projects and the issues that may arise when re-creating them are covered in the following sections:

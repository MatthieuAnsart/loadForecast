Activities and interval variables

Explains the use of interval variables as a replacement for the OPL 3.7 Activity type.

In OPL 3.7 the Activity type was used to represent an operation being carried out between two time points. In OPL 6.x, the interval variable is used in this role.

For example, an activity that takes 10 time units to execute, which cannot start before 0 and cannot end after 100, would be created as:

In OPL 3.7

  Activity a(10);
  solve {
    0 <= a.start;
    a.end <= 100;
  };

In OPL 6.x

  using CP;
  dvar interval a in 0..100 size 10;

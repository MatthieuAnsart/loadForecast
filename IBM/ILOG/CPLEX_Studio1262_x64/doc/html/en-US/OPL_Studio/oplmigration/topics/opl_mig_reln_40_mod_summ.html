Keywords

Table 1 summarizes the changes that affect OPL keywords.

Table 1. Keywords
Old New Comments
var dvar To avoid confusion with the ILOG Script keyword var
struct tuple Use this new keyword to declare a tuple schema. See Compatibility for details.

Operators

Table 2 summarizes the changes that affect OPL operators. The operator syntax is closer to languages like C++ and Java.

Table 2. Operators
Operator OPL 3 OPL 4
Equality = ==
Different from <> !=
Logical AND & &&
Logical OR (disjunction) \/ ||
Assignment of constraints : =
Integer division / div
Modulo mod %

Data Types

Table 3 summarizes the changes that affect data types.

Table 3. Data Types
Types OPL 3 OPL 4
character Supported No longer supported
enumerated lists enum No longer supported. Use set of strings instead
float+, int+ No restriction Available for decision variables only
ranges Used as types and/or values Used only as values

Declarations

Table 4 summarizes the changes in the way you must declare data.

Table 4. Declarations
What? OPL 3 OPL 4 Comments
Arrays [i,j] [i][j] The old syntax is deprecated but still supported for compatibility reasons. See Compatibility below for details.
Grouped Declarations Comma-separated declarations for identifiers of the same type Single-identifier declarations only  

Functions

Table 5 summarizes the changes that affect functions.

Table 5. Functions
What OPL 3 OPL 4 Comments
Function nearest round The function was just renamed.
procedural function call on strings and sets Methods and/or functions Functions

Write

last(S)

instead of

Syntax

Table 6 summarizes the changes to the syntax.

Table 6. Syntax
Syntax element OPL 3 OPL 4 Comments
conditional constraint posting

if ... then ... endif

if ... then ... else ... endif

if (...) { ... }

if (...) { ... } else { ... }

 Supported only outside forall constructs.
literal strings: \ \\ The end of line within a string literal needs to be escaped with \
flat file initialization < "file.txt"   No longer supported
interactive input int n << "value ?";   No longer supported
objectives No delimiter end with ; Objectives must be terminated by a semi-colon.
constraint range   Range expressions can have no more than 3 operands.  
domain range Implicit conversion from an integer range to an integer set No support of implicit type conversions

Use the asSet method explicitly:

{int} intSet = asSet(1..13);

aggregate piecewise   Supports only one piece

Model templates

Describes how to use the model templates feature in the IDE.

About this task

Model templates are part of the code completion feature. The template options are colored blue, to distinguish them from other options.

  • Templates (snippets of code), either predefined and supplied with CPLEX® Studio or created by you, can be selected from a list and inserted in their entirety.

  • They can provide an easy way to insert a complete structure, including placeholders for variables that can then be replaced with the variables by tabbing between them and typing or using the Dot (.) syntax to access variables and other elements from your model.

  • Templates can be context sensitive, allowing different completions for different parts of the edited file. For example, entering OPL code vs. entering ILOG® Script code.

  • Templates can be simple constructions, such as:

    using ${engine}

    They can be more complex, with line feeds, for example:

    execute ${name} {
}

    They can include dvar arrays inside the model (the number of dimensions being automatically correct). For example:

    x[${i}][${j}]

  • A number of useful templates are supplied with CPLEX Studio, but you can add your own as you find uses for them.

  • All templates can be edited using the OPL Preferences editor. New templates can be easily created, based on code that is frequently reused in your work.

  • Working with model templates
    Model templates can be selected from the code completion popup window whenever it is open.
  • Editing or creating model templates
    Predefined templates are supplied with CPLEX Studio, but you can edit them or add your own templates.

Tooltips and arrows

Tooltips appear on most buttons, and arrows appear on some.

When you move the pointer over a button in any toolbar, a tooltip displays a short description of it. For example, the Pause button, which becomes active once a solve has started.

Note that some buttons are associated with a small arrow pointing downwards, such as the Run button , Debug button , and Browse button . Click that arrow to select the element to which the button action must apply, or to access other functions.

Tooltips describe adequately the functions of the standard toolbar (Save, Save all, ...) Here, we add more detail for the:

  • Execution toolbar

  • Debug view toolbar

  • OPL Projects Navigator view toolbar

  • Problem browser view toolbar

  • Output Area toolbars

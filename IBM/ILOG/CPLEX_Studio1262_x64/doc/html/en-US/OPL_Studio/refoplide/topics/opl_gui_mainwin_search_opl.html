OPL search

The OPL search in the IDE editor is aware of OPL syntax and model structure.

To open the OPL Search from the menu bar, select Search > OPL Search, or click the search button and select OPL Search.

This opens a search dialog box with three tabs, showing the OPL Search uppermost. In the example below, the search criteria are: all OPL elements named "skills" (not case sensitive) found in OPL declarations.

Type Search for a name of a tuple
Field Search for a field in a tuple
Element Search for other OPL elements (data, variable)
Declaration The item is in an OPL declaration.
References A declared item is referred to in the part of the model that uses the declarations.

The search function operates on all the projects open in the workspace (project browser). It does not return every instance of the string, but only real OPL elements. For example, a search on "day" would return only OPL elements declared as "day", or referenced in models. It would not return "today", weekday, "Monday", as an ordinary search would do. The OPL Search can be called "intelligent" as it aware of the OPL syntax and model structure.

The results of the OPL search are displayed in the Search tab of the console.

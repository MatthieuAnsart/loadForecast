Supported databases

OPL models support several databases.

On UNIX platforms, you may need to rebuild the libraries for a database connection. All the necessary files are located in:

<Install_dir>\opl\lib

where <Install_dir> is the directory in which you have installed CPLEX® Optimization Studio.

Go to the shared directory and build the required driver with a command such as:

make -f makefile.drv libdbora11.so

The table below lists the databases to which you can connect your OPL model to read and write data.

Table 1. Supported databases
Platform Database
Windows 32 bits Oracle 10g and 11g, DB2®, SQL Server 2005, SQL Server 2008 R2

Drivers: ODBC, OLE DB
Windows 64 bits Oracle 10g and 11g, DB2, SQL Server 2008 R2

Drivers: ODBC, OLE DB
Linux 32 bits and 64 bits Oracle 10g and 11g
AIX® 32 bits and 64 bits Oracle 10g and 11g, DB2
UltraSPARC 32 bits Oracle 10g and 11g, DB2
UltraSPARC 64 bits Oracle 10g and 11g
Notes:
  • OLE DB is used to connect to the SQL Server databases.
  • On UNIX platforms, if you encounter problems using Rogue Wave DB Link with the OPL Java™ API, try setting the environment variable LD_PRELOAD (or LD_PRELOAD_32, LD_PRELOAD_64 for Solaris) to libdbkernel.so

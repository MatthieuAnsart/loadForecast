Search control/Limits

Describes the general options that enable you to control the limits in the search process.

Click Constraint Programming>Search control>Limits on the left, as shown in the following screen capture.

Figure 1. CP options: Search control/Limits

The table below describes the Search Control/Limits options.

Table 1. CP Search control/Limits options 
Name Description Default Value
Branch limit Limits the number of branches that are created before terminating a search. A branch is a decision made at a choice point in the search. A typical node is made up of two branches. For example: x == value and x != value . A branch is only counted a the moment a decision is executed, not when the two branches of the choice point are decided. A branch is counted even if the decision leads to an inconsistency (failure). The possible values of this parameter range from 0 to IloIntMax . A value of IloIntMax does not set any limit. The value IloIntMax is the default value. 2100000000
Fail limit Limits the number of failures that can occur before terminating the search. The possible values of this parameter range from 0 to IloIntMax . A value of IloIntMax does not set any limit. The value IloIntMax is the default value. 2100000000
Solution limit Limits the number of feasible solutions that are found before terminating a search. The possible values range from 0 to IloIntMax . A value of IloIntMax does not set any limit. The value IloIntMax is the default value. 2100000000
Time limit Limits the CPU time spent solving before terminating a search. The possible values of this parameter range from 0 to IloInfinity . A value of IloInfinity does not set any limit. The value IloInfinity is the default value. 1e+075

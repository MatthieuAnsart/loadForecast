Search control/General options

Describes the general options that enable you to control the search process in constraint programming models.

Click Constraint Programming>Search control>General on the left, as shown here.

Figure 1. CP options: Search control/General

In the table below, Search control/General options are ordered alphabetically.

Table 1. CP Search control/General options 
Name Description Default Value
Dynamic probing Controls probing carried out during search. Probing can be useful on some problems as it can make stronger inferences on combinations of constraints. Possible values for this parameter are On (dynamic probing is activated with a constant strength), Auto (dynamic probing is activated and its strength is adjusted adaptively) and Off (dynamic probing is deactivated). Dynamic probing only has an effect when using the Restart search type on problems without interval variables. Auto
Dynamic probing strength

Controls the effort which is dedicated to dynamic probing. It is expressed as a factor of the total search effort: changing this parameter has no effect unless Dynamic probing parameter is set to Auto or On.

When Dynamic probing is set to On, the probing strength is held constant througout the search process. When Dynamic probing is set to Auto, the probing strength starts off at the specified value and is thereafter adjusted automatically.

Possible values for this parameter range from 0.001 to 10000. A value of 1.0 indicates that dynamic probing will consume a roughly equal amount of effort as the rest of the search. The default value of this parameter is 0.03, meaning that around 3% of total search time is dedicated to dynamic probing.

0.03
Number of search points in MultiPoint Controls the number of (possibly partial) solutions manipulated by the multipoint search algorithm. A larger value than 30 diversifies the search, with possible improvement in solution quality at the expense of a longer run time. A smaller value intensifies the search, resulting in faster convergence at the expense of solution quality. Note that memory consumption increases proportionally to this parameter, for each search point must store each decision variable domain. 30
Number of workers

Sets the number of workers to run in parallel to solve your model. The default value auto corresponds to the number of CPUs available.

If the number of workers is set to n (with n > 1), the CP Optimizer engine creates n workers, each in their own thread, that will work together to solve the problem. The emphasis of these workers is to find better partial solutions and not to speed up the proof of optimality.

There is no limit to the maximum number of workers. Note that using multiple workers requires more memory than using a single worker.

 Auto
Random seed The search uses some randomization in some strategies. This parameter sets the seed of the random generator used by these strategies. Possible values range from 0 to IloIntMax . 0
Restart fail limit When the Search control/General options is set to Restart, a depth-first search is restarted after a certain number of failures. This parameter controls the number of failures that must occur before restarting search. Possible values range from 0 to IloIntMax . This value can increase after each restart: see the parameter Search control/General options. 100
Restart growth factor When the Search control/General options is set to Restart, a depth-first search is restarted after a certain number of failures. This parameter controls the increase of this number between restarts. If the last fail limit was f after a restart, for the next run, the new fail limit will be f times the value of this parameter. Possible values of this parameter range from 1.0 to IloNumMax . The initial fail limit can be controlled with the parameter Search control/General options. 1.05e+000
Search type

Determines the type of search that is applied when solving a problem.

  • Auto: the appropriate search type is chosen automatically (usually Restart).

  • Depth first: a regular depth-first search.

  • Restart: a depth-first search that restarts from time to time.

  • Multi point: a method that combines a set of (possibly partial) solutions.

See also Search types.

 Auto
Temporal relaxation This advanced setting can be used to control the strength of usage of a temporal relaxation internal to the CP Optimizer engine. The possible values are 0, 1, and 2, with 2 as the default, meaning the strongest usage of the relaxation. For some models, using the strongest level becomes inefficient; you may deactivate the use of the temporal relaxation entirely using value 0 or partially using value 1. 2
Time mode This parameter defines how time is measured by the CP Optimizer engine, the two possible values being Elapsed time and CPU time. CP Optimizer uses time for both display purposes and for limiting the search via the Time limit setting in the Limits pane. Note that when multiple processors are available and the Number of workers is set to greater than one, the CPU time can be greater than the elapsed time by a factor up to the number of workers. CPU time

Search control/Tolerances

Describes the general options that enable you to control tolerances in the search process.

Click Constraint Programming>Search control>Tolerances in the left pane, as shown in the following screen capture.

Figure 1. CP options: Search control/Tolerances

The table below describes the Search control/Tolerances options.

Table 1. CP Search Control/Tolerances Options 
Name Description Default Value
Optimality Tolerance

Sets an absolute tolerance on the objective value for optimization models. This means that when the CP Optimizer engine reports an optimal solution found, then there is no solution which improves the objective by more than the value of this parameter.

This parameter is used in conjunction with Search control/Tolerances. The optimality of a solution is proven if either of the two parameters' criteria is fulfilled.

 1e-015
Relative optimality tolerance Sets a relative tolerance on the objective value for optimization models. This means that when the CP Optimizer engine reports an optimal solution found, then there is no solution which improves the objective by more than “the absolute value of the objective x the value of this parameter”. 1e-004

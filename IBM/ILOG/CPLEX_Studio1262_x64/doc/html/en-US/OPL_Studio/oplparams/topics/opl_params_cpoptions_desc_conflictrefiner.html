Conflict refiner

CP Optimizer uses a conflict refiner to manage conflicts and identify the causes of infeasibility.

Given an infeasible model, the conflict refiner can identify conflicting constraints and variable domains within the model to help you identify the causes of the infeasibility. In this context, a conflict is a subset of the constraints and variable domains of the model that are mutually contradictory. The conflict refiner first examines the full infeasible model to identify portions of the conflict that it can remove. By this process of refinement, the conflict refiner arrives at a minimal conflict. A minimal conflict is usually smaller than the full infeasible model and thus makes infeasibility analysis easier.

Note that the CP Optimizer conflict refiner behaves similarly to the CPLEX conflict refiner with a similar API.

For more information on the conflict refiner, please refer to the concept Conflict refiner in the CP Optimizer C++ API Reference Manual.

interval

OPL keyword (CP, scheduling) for interval variables.

Purpose

OPL keyword (CP, scheduling) for representing interval variables.

context Model files (.mod)

Syntax

dvar interval a [optional[(IsOptional)]]
                [in StartMin..EndMax]
                [size SZ | SZMin .. SZMax]
                [intensity F]

Where:

int IsOptional, StartMin, EndMax, SZ, SZMin, SZMax;
stepFunction F;
-maxint/2 + 1 <= StartMin <= maxint/2 -1
-maxint/2 + 1 <= EndMax <= maxint/2 -1
0 <= SZ <= maxint/2 -1
0 <= SZMin <= maxint/2 -1
0 <= SZMax <= maxint/2 -1

Description

An interval variable represents an interval of time during which something happens (for example, a task occurs, an activity is carried out) and whose position in time is an unknown of the scheduling problem. An interval is characterized by a start value, an end value, a size and an intensity. The length of an interval is its end time minus its start time.

An important additional feature of interval variables is the fact that they can be optional; that is, one can decide not to consider them in the solution schedule. Also span and alternative can be used to expand the usage of intervals. Although interval is a CP keyword it is still accepted as a CPLEX identifier.

Example

An interval variable with unconstrained size (in [0,maxint]):

dvar interval a;

An optional interval variable with size 10 that, when present, must start after value 20 and end before value 1000:

dvar interval a optional in 20..1000 size 10;

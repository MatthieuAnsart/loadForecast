inverse

OPL function to compute the inverse of values of arrays of integer decision variables.

context Model files (.mod) - Not allowed in a CPLEX® constraint block.
type 
boolean (1 if the constraint is true, 0 otherwise)

Syntax

inverse(int[],int[])
inverse(dvar int[],dvar int[])

Description

This constraint takes two arrays of integer variables that must be indexed by an integer and be one-dimensional. The following constraint:

inverse(x, y);

implies that for any value i of the indexer of y, we have 

x[y[i]] = i

and similarly for the indexer of x.

This function also works for integer arrays outside constraint blocks both in constraint- and mathematical programming models.

You can use this function within IBM® ILOG® Script statements by specifying the OPL namespace:

( Opl.xxx() )
Note: This constraint cannot be used in a meta-constraint.

Example

using CP;

int n = 5;

range R = 1..n;
range S = 11..10+n;
dvar int f[S] in R;
dvar int g[R] in S;


subject to {
 
    inverse(all[R](i in R) g[i],
            all[S](j in S) f[j]);
}

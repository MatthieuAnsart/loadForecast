and

OPL aggregate to express cumulative constraints.

Purpose

OPL aggregate constraint. An equivalent of forall to express cumulative constraints.

context Model files (.mod) - Not allowed in a CPLEX constraint block
type 
boolean (1 if the constraint is true, 0 otherwise)

Syntax

AggregateExpression: "sum" '(' Qualifiers ')' Expression
                   | "min" '(' Qualifiers ')' Expression
                   | "max" '(' Qualifiers ')' Expression
                   | "prod" '(' Qualifiers ')' Expression
                   | "or" '(' Qualifiers ')' Expression
                   | "and" '(' Qualifiers ')' Expression
                   | "union" '(' Qualifiers ')' Expression
                   | "inter" '(' Qualifiers ')' Expression
                   | AllExpression

Description

This constraint is equivalent to the forall keyword but enables you to express cumulative constraints in a more compact form by using a logical AND to aggregate several constraints into a single one.

Behavior on empty sets:

and(empty set)=true

Example

using CP;

int n = 5;

range R = 1..n;
dvar int x[R] in R;

maximize sum(i in R)x[i];

subject to {
  and(i in 1..5:i % 2==1) x[i]==1;
  and(i in 1..5:i % 2==0) x[i]==2;
}

SPSSRead

Reads data from an SPSS stream into an OPL tuple set, via SPSS Modeler.

Context

Data files (.dat)

Syntax

<element> from SPSSRead(spss, "node_name");
where <element> is an OPL tuple set and node_name is an SPSS node.

Description

Once you have established a connection to SPSS Modeler using the keyword SPSSConnection, you can read data from an SPSS stream into an OPL tuple set using SPSSRead.

OPL tuple sets are the only type of data that can be read. If data types do not match, an error message, containing an OPL tuple definition, is displayed. This definition can be copied and pasted.

Note:

  • OPL cannot process SPSS termination nodes; it processes only operation nodes. You must, therefore, modify SPSS data streams so that OPL can use them. In the example below, we insert an operation node, Filter, before the terminal result table, and edit the filter node. OPL reads data from the Filter node. For more information, see Adapting the SPSS stream for CPLEX Studio in Preparatory steps.

  • OPL cannot process the characters $, &, #, %, -, ., or an empty string, and automatically replaces them with an underscore (_). A warning message is displayed in the IDE. So, for example, $S-campaign-1 is renamed S_campaign_1. (You can also rename the characters manually, in the SPSS Modeler. See Adapting the SPSS stream for CPLEX Studio.)

Example

SPSSConnection spss("pm_selflearn.str","");
Table from SPSSRead(spss,"Filter");

DBConnection

OPL keyword for connection to a database

Purpose

OPL keyword for connection to a database (external initialization of data).

context Data files (.dat)

Syntax

Connection: "DBConnection" Id '(' String String ')'
          | "SheetConnection" Id '(' String ')'

Description

There are basically two ways of initializing data in OPL:

  • Internal Initialization consists in initializing the data at the same time as it is declared. Inline initializations can contain expressions to initialize data items, such as 

    int a[1..5] = [b+1, b+2, b+3, b+4, b+5];

  • External Initialization consists in specifying initialization subsequently as an OPL statement in a separate data file (.dat). This includes reading from a database.

In the context of database initialization, you use the keyword DBConnection to connect to one or several databases, as shown in the following example. It is possible to connect to several databases within the same model.

Here are two examples declaring a connection. In both examples, the connection db should be viewed as a handle on the database.

Example 1

DBConnection db("odbc","oilDB/user/passwd");

This example uses the ODBC data source oilDB declared by the system to connect to the database. The string passed as first argument indicates that you want to connect to a database source managed by ODBC. The string passed as second argument is the connection string; often this is the database or data source name, user and password.

The recommended format to use for the second string is as follows:

"<data source name>/[user]/[password]"

where <data source name> is the identifier you typed in the field Data Source Name when linking the database to ODBC. In this example, this identifier is oilDB. The fields user and password may be omitted, but note that the slash signs (/) are mandatory.

Example 2

DBConnection db("access","oilDB.mdb");

The string passed as first argument indicates that you want to connect to an Access database. The string passed as second argument designated the Access database file. This is a helper implementation actually based on ODBC and you do not need to specify the full path name. Path names are resolved relatively to the directory of the current .dat file.

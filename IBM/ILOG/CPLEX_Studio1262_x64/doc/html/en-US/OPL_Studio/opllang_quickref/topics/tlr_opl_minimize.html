minimize

OPL keyword for the objective function

Purpose

OPL keyword to express the objective function.

context Model files (.mod)

Syntax

Objective: . "minimize" Expression
         | . "maximize" Expression

Description

OPL instructions are responsible for stating problem constraints and, in optimization problems, the objective function. Constraints are stated in OPL by an optimization instruction: maximize or minimize.

Note:

  1. Optimization instructions require an objective function of type integer or float.

  2. You cannot insert an execute block between a maximize or minimize statement and constraint definitions. If there are no constraints for an objective definition, use an empty one like “subject to {}” instead.

Example

{string} Products = { "Car", "Truck"};
{string} Resources = { "Oil", "Wood"};

dvar float+ inside[Products];
dvar float+ outside[Products];

float insideCost[Products]= [100,200];
float outsideCost[Products]= [400,300];
float consumption[Products][Resources]=[[20,10],[5,40]];
float capacity[Resources]=[10,20];
float demand[Products]= [40,30];


minimize sum(p in Products) (insideCost[p]*inside[p] + outsideCost[p]*outside[p]);
subject to { 
  forall(r in Resources) 
    sum(p in Products) consumption[p,r] * inside[p] <= capacity[r]; 
  forall(p in Products) inside[p] + outside[p] >= demand[p]; 
}

Solving OPL models on the cloud

IBM Decision Optimization on Cloud enables OPL models to be solved on the cloud.

Models available for the cloud

Some of the OPL models can be run on the cloud from the IDE. In the New Example wizard, most of the models are "cloud ready". The run configuration can contain zero or any number of plain .dat files (that is, a .dat file with no Excel statement, or no DB Link statement). The code in a standalone .mod file must be cloud compliant (it does not require access to your disk).

Choosing a server

Before you can run a model, you must add server credentials. In the IDE, select Window > Preferences > OPL > DOcloud and add your Base URL and API Key.

If you do not have server credentials, click the DOcloud link in the Preferences window to obtain them.

If you are not registered on Docloud, you first need to register. Then, click the Developer tab and select Get API key and base URL.

Copy the base URL and generate an API key. Copy these into the IDE.

Running a model

If you select Cloud mode in a settings file, and run a model in your local IDE, all non compliant cloud features will raise an error. This setting helps you develop a model for the cloud on your local machine by checking the local solve more thoroughly. When you run a model with the DOcloud option, this setting detects OPL statements that are not supported on the cloud although they are supported on premises.

To run a model on the cloud, select the run configuration and right click to display the menu. Select the option Run this on DOcloud.

Monitoring jobs in the IDE

The DOcloud tab in the IDE displays the status of an executed run configuration. Only a run configuration containing a tuple set as a post processing structure will provide a report.

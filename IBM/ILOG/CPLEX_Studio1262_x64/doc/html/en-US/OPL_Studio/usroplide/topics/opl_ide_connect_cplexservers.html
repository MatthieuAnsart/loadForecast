IDE connection to CPLEX servers

You can connect to CPLEX servers from the IDE in CPLEX Studio.

About this task

The command Window > Preferences > OPL > CPLEX Servers opens the CPLEX Servers window, which enables you to add, remove, and edit a server. The screen capture below shows the local server selected, after clicking the button Add Local Server.

Procedure

  1. To add a new server, click the Add button and enter the name and location of the server. Then click OK. For example:

    The server name and location are validated. If an invalid server name, or location, is entered, a yellow triangle appears. When you hover over the triangle, a tooltip explains why the entry is invalid. The connection is not tested, only the validity of the address in terms of syntax is tested.

    You must also enter the login and password for the new CPLEX server.

  2. To set a server as the default, check the empty box to the left of the server name.

    Alternatively, choose a server from the dropdown menu in the main menu bar.

  3. To remove or edit a server, select the server name and the appropriate buttons become active.

  4. You can also export or import an XML file containing a list of servers. This list may be provided by an administrator, or created automatically if you press the Export button.

    When you click Export, the currently displayed server names and locations are saved in an XML file.

    Note: You are advised not to edit this XML file manually.
  5. When you press the Import button, the list of servers in the XML file replaces the list in the CPLEX servers window.

Executing OPL run configurations on a CPLEX server

About this task

You can execute one, or several, run configurations from the same OPL project on a CPLEX server. When you execute a run configuration on a CPLEX server, an application is packaged and commands are sent to the server to solve the selected run configuration(s) and return the results.

Procedure

  1. Right-click a run configuration, select Run this on, then choose a server from the list.
  2. You can select several run configurations and execute them on the server as a batch task.

Monitoring jobs in the IDE

About this task

You can display information about the jobs running on the servers.

Procedure

Select Window > Show View > CPLEX Servers. The IDE console shows the servers that are running, with their status. If a remote server is used, the job may take a while, and the status is 'Fetching'.

By default, the view shows only the default server. To view all the servers, click the toggle button "Display only default server".

When the job has finished, you see it displayed in the list of jobs. In our example, we ran a configuration of the cutstock problem.

You also see jobs that are not finished. As soon as you run a job on a server the hierarchy of information is updated.

The console monitors what is currently running on the server and the CPLEX Servers tab displays the following hierarchy of information:

  • the server name
    • the applications that are running on each server
      • the jobs that are running in each application

In our example, this corresponds to:

  • Localhost
    • the OPL project "cutstock"
      • the run configuration "Fixed number of patterns"

A run configuration can be executed several times, but this creates a new job each time. However, the application is not redeployed.

If you hover the mouse cursor over the buttons at the top of the CPLEX Servers tab, the tool tips will display their respective monitoring functions.

Restoring a solution from a server

Procedure

Select a successful job in the CPLEX Servers view and right-click. Select Import results from the menu. The menu is not available on failed jobs, or jobs that are still running.

If several OPL projects are open, you are asked to select the project for which you want to import the results.

Results

The solution is displayed in the Problem browser and the Solutions tab. You will also find information in the Engine log and the Profiler.

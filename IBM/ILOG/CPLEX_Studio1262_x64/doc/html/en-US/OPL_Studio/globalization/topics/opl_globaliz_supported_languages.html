Supported languages and features

The CPLEX Studio IDE is localized in Brazilian Portuguese, Chinese, French, German, Japanese, Korean, Polish, Russian, Spanish, and Turkish.

CPLEX Studio is "globalization ready", that is, it can be used in non-English contexts. However, this notion sometimes covers several different aspects of a product, so here we describe the supported languages and features and those that are Not supported. The installer is localized, but the documentation and error messages are not.

Asian languages

The CPLEX Studio IDE is localized in Chinese, Japanese, and Korean. Provided that the Chinese, Japanese, or Korean operating system is installed on the computer and that the locale matches, menus, buttons, and other GUI elements appear in the corresponding language.

  • Multibyte string data is supported throughout the process of modeling and creating applications: OPL model files (.mod) and text-based data files (.dat), as well as databases, can contain multibyte strings that can be used in your applications.

  • You can edit multibyte string data in the CPLEX Studio IDE.

    When string data defined in a .dat file contains multibyte strings, it is mandatory to include these strings between double quotation marks. In the following example, the monobyte string kluski may optionally be included in double quotation marks, whereas these marks are mandatory for the Japanese multibyte string.

Note:

Multibyte characters do not display in the IDE editor if your machine is not running on an operating system that supports such characters.

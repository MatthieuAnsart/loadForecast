Using callbacks

These routines invoke callbacks, implement callbacks, manipulate the branch and bound tree, access callback information. See also callbacks for mixed integer programs (MIPs).

  • CPXXgetcallbackinfo and CPXgetcallbackinfo
    The routine CPXXgetcallbackinfo/CPXgetcallbackinfo accesses information about the current optimization process from within a user-written callback function.
  • CPXXgetlpcallbackfunc and CPXgetlpcallbackfunc
    The routine CPXXgetlpcallbackfunc/CPXgetlpcallbackfunc accesses the user-written callback routine to be called after each iteration during the optimization of a continuous problem (LP, QP, or QCP), and also periodically during the CPLEX presolve algorithm.
  • CPXXgetnetcallbackfunc and CPXgetnetcallbackfunc
    The CPXXgetnetcallbackfunc/CPXgetnetcallbackfunc accesses the user-written callback routine to be called each time a log message is issued during the optimization of a network problem.
  • CPXXgettuningcallbackfunc and CPXgettuningcallbackfunc
    The routine CPXXgettuningcallbackfunc/CPXgettuningcallbackfunc accesses the user-written callback routine to be called before each trial run during the tuning process.
  • CPXXsetlpcallbackfunc and CPXsetlpcallbackfunc
    The routine CPXXsetlpcallbackfunc/CPXsetlpcallbackfunc modifies the user-written callback routine to be called after each iteration during the optimization of a linear program, and also periodically during the CPLEX presolve algorithm.
  • CPXXsetnetcallbackfunc and CPXsetnetcallbackfunc
    The routine CPXXsetnetcallbackfunc/CPXsetnetcallbackfunc sets the user-written callback routine to be called each time a log message is issued during the optimization of a network program.
  • CPXXsettuningcallbackfunc and CPXsettuningcallbackfunc
    The routine CPXXsettuningcallbackfunc/CPXsettuningcallbackfunc modifies the user-written callback routine to be called before each trial run during the tuning process.

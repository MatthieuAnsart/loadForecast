CPXXgetprotected and CPXgetprotected

The routine CPXXgetprotected/CPXgetprotected accesses the set of variables that cannot be aggregated out.

int CPXXgetprotected( CPXCENVptr env, CPXCLPptr lp, CPXDIM * cnt_p, CPXDIM * indices, CPXDIM pspace, CPXDIM * surplus_p )

int CPXgetprotected( CPXCENVptr env, CPXCLPptr lp, int * cnt_p, int * indices, int pspace, int * surplus_p )

Description

Warning:

This is an advanced routine. Advanced routines typically demand a thorough understanding of the algorithms used by CPLEX. Thus they incur a higher risk of incorrect behavior in your application, behavior that can be difficult to debug. Therefore, the team encourages you to consider carefully whether you can accomplish the same task by means of other Callable Library routines instead.

The routine CPXXgetprotected/CPXgetprotected accesses the set of variables that cannot be aggregated out.

Note: If the value of pspace is 0, the negative of the value of surplus_p returned specifies the length needed for array indices.

Arguments

Name Description
env A pointer to the CPLEX environment, as returned by CPXXopenCPLEX/CPXopenCPLEX.
lp A pointer to a CPLEX LP problem object, as returned by CPXXcreateprob/CPXcreateprob.
cnt_p A pointer to an integer to contain the number of protected variables returned, that is, the true length of the array indices.
indices The array to contain the indices of the protected variables.
pspace An integer specifying the length of the array indices.
surplus_p A pointer to an integer to contain the difference between pspace and the number of entries in indices. A nonnegative value of surplus_p specifies that the length of the arrays was sufficient. A negative value specifies that the length was insufficient and that the routine could not complete its task. In that case, the routine CPXXgetprotected/CPXgetprotected returns the value CPXERR_NEGATIVE_SURPLUS, and the value of surplus_p specifies the amount of insufficient space in the arrays.

Return

The routine returns 0 (zero) if successful and nonzero if an error occurs. The value CPXERR_NEGATIVE_SURPLUS specifies that insufficient space was available in the array indices to hold the protected variable indices.

Example

status = CPXgetprotected (env, lp, &protectcnt,
		   protectind, 10, &surplus);

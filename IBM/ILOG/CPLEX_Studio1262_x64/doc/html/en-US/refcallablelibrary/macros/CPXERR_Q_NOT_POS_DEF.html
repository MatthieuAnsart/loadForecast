CPXERR_Q_NOT_POS_DEF

5002 Q in '%s' is not positive semi-definite

#define CPXERR_Q_NOT_POS_DEF 5002

The Q matrix associated with the quadratic objective or with a quadratic constraint must be positive semi-definite (for minimizations). Check the appropriate quadratic term(s). If only the objective is quadratic, you can use the solution target parameter to solve models either to local optimality or to global optimality. For more detail about this choice, see optimality target in the manual Parameters of CPLEX.

This error also arises if a quadratic constraint is not convex. A convex quadratic constraint is of the form: 
 x'Qx + p'x + c <= 0
Quadratic constraints in which the relation is greater than or equal to (that is, >=) and quadratic constraints in which the relation is equality (that is, ==) are not convex. Consequently, greater-than-or-equal-to quadratic constraints can raise this error. Likewise, quadratic equality constraints can raise this error.

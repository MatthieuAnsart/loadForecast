CPXXrefinemipstartconflict and CPXrefinemipstartconflict

The routine CPXXrefinemipstartconflict/CPXrefinemipstartconflict refines a conflict in order to determine why a given MIP start is not feasible.

int CPXXrefinemipstartconflict( CPXCENVptr env, CPXLPptr lp, int mipstartindex, CPXDIM * confnumrows_p, CPXDIM * confnumcols_p )

int CPXrefinemipstartconflict( CPXCENVptr env, CPXLPptr lp, int mipstartindex, int * confnumrows_p, int * confnumcols_p )

Description

The routine CPXXrefinemipstartconflict/CPXrefinemipstartconflict refines a conflict in order to determine why a given MIP start is not feasible. In other words, this routine identifies a minimal conflict for the infeasibility of the linear constraints and bounds in a model inferred from the pairs of variables and values of a MIP start.

In order to analyze the infeasibility of a MIP start specifying pairs of variables and values in model with quadratic constraints, indicator constraints, or special ordered sets, as well as linear constraints and bounds, use the routine CPXXrefinemipstartconflictext/CPXrefinemipstartconflictext.

The given MIP start need not be complete; that is, it is not necessary to specify all the variables of the model.

When this routine returns, the value in confnumrows_p specifies the number of constraints participating in the conflict, and the value in confnumcols_p specifies the number of variables participating in the conflict.

Use the routine CPXXgetconflict/CPXgetconflict to determine which constraints and variables participate in the conflict. Use the routine CPXXclpwrite/CPXclpwrite to write the conflict to a file.

This conflict is a submodel of the original model with the property that CPLEX cannot generate a solution from the chosen MIP start using the given level of effort and that removal of any constraint or bound in the conflict invalidates that property.

The parameters CPX_PARAM_CUTUP, CPX_PARAM_CUTLO, CPX_PARAM_OBJULIM, CPX_PARAM_OBJLLIM do not influence this routine. If you want to study infeasibilities introduced by those parameters, consider adding an objective function constraint to your model to enforce their effect before you invoke this routine.

When the MIP start was added to the current model, an effort level may have been associated with it to specify to CPLEX how much effort to expend in transforming the MIP start into a feasible solution. This routine respects effort levels except level 1 (one): check feasibility. It does not check feasibility.

Arguments

Name Description
env A pointer to the CPLEX environment as returned by the routine CPXXopenCPLEX/CPXopenCPLEX.
lp A pointer to a CPLEX problem object as returned by CPXXcreateprob/CPXcreateprob.
mipstartindex The index of the MIP start among all the MIP starts associated with the problem.
confnumrows_p A pointer to an integer where the number of linear constraints in the conflict is returned.
confnumcols_p A pointer to an integer where the number of variable bounds in the conflict is returned.

Return

The routine returns 0 (zero) if successful and nonzero if an error occurs.

Routines in the CPLEX Callable Library (C API)

These routines are available in the CPLEX Callable Library (C API).

  • CPXXaddchannel and CPXaddchannel
    The routine CPXXaddchannel/CPXaddchannel instantiates a new channel object.
  • CPXXaddcols and CPXaddcols
    The routine CPXXaddcols/CPXaddcols adds columns to a specified CPLEX problem object.
  • CPXXaddfpdest and CPXaddfpdest
    The routine CPXXaddfpdest/CPXaddfpdest adds a file to the list of message destinations for a channel.
  • CPXXaddfuncdest and CPXaddfuncdest
    The routine CPXXaddfuncdest/CPXaddfuncdest adds a function msgfunction to the message destination list for a channel.
  • CPXXaddindconstr and CPXaddindconstr
    The routine CPXXaddindconstr/CPXaddindconstr adds an indicator constraint to the specified problem object.
  • CPXXaddmipstarts and CPXaddmipstarts
    The routine CPXXaddmipstarts/CPXaddmipstarts adds multiple MIP starts to a CPLEX problem object of type CPXPROB_MILP, CPXPROB_MIQP, or CPXPROB_MIQCP.
  • CPXXaddqconstr and CPXaddqconstr
    The routine CPXXaddqconstr/CPXaddqconstr adds a quadratic constraint to a specified CPLEX problem object.
  • CPXXaddrows and CPXaddrows
    The routine CPXXaddrows/CPXaddrows adds constraints to a specified CPLEX problem object.
  • CPXXaddsolnpooldivfilter and CPXaddsolnpooldivfilter
    The routine CPXXaddsolnpooldivfilter/CPXaddsolnpooldivfilter adds a new diversity filter to the solution pool.
  • CPXXaddsolnpoolrngfilter and CPXaddsolnpoolrngfilter
    Adds a new range filter to the solution pool.
  • CPXXaddsos and CPXaddsos
    The routine CPXXaddsos/CPXaddsos adds information about a special ordered set (SOS) to a problem object of type CPXPROB_MILP, CPXPROB_MIQP, or CPXPROB_MIQCP.
  • CPXXbaropt and CPXbaropt
    The routine CPXXbaropt/CPXbaropt may be used to find a solution to a linear program (LP), quadratic program (QP), or quadratically constrained program (QCP) by means of the barrier algorithm at any time after the problem is created by a call to CPXXcreateprob/CPXcreateprob.
  • CPXXboundsa and CPXboundsa
    The routine CPXXboundsa/CPXboundsa accesses upper and lower sensitivity ranges for lower and upper variable bounds for a specified range of variable indices.
  • CPXXchgbds and CPXchgbds
    The routine CPXXchgbds/CPXchgbds changes the lower or upper bounds on a set of variables of a problem.
  • CPXXchgcoef and CPXchgcoef
    The routine CPXXchgcoef/CPXchgcoef changes a single coefficient in the constraint matrix, linear objective coefficients, righthand side, or ranges of a CPLEX problem object.
  • CPXXchgcoeflist and CPXchgcoeflist
    The routine CPXXchgcoeflist/CPXchgcoeflist changes a list of matrix coefficients of a CPLEX problem object.
  • CPXXchgcolname and CPXchgcolname
    The routine CPXXchgcolname/CPXchgcolname changes the names of variables in a CPLEX problem object.
  • CPXXchgctype and CPXchgctype
    The routine CPXXchgctype/CPXchgctype changes the types of a set of variables of a CPLEX problem object.
  • CPXXchgmipstarts and CPXchgmipstarts
    The routine CPXXchgmipstarts/CPXchgmipstarts modifies or extends multiple MIP starts.
  • CPXXchgname and CPXchgname
    The routine CPXXchgname/CPXchgname changes the name of a constraint or the name of a variable in a CPLEX problem object.
  • CPXXchgobj and CPXchgobj
    The routine CPXXchgobj/CPXchgobj changes the linear objective coefficients of a set of variables in a CPLEX problem object.
  • CPXXchgobjoffset and CPXchgobjoffset
    The routine CPXXchgobjoffset/CPXchgobjoffset changes the constant offset of the objective function for a problem.
  • CPXXchgobjsen and CPXchgobjsen
    The routine CPXXchgobjsen/CPXchgobjsen changes the sense of the optimization for a problem, to maximization or minimization.
  • CPXXchgprobname and CPXchgprobname
    The routine CPXXchgprobname/CPXchgprobname changes the name of the current problem.
  • CPXXchgprobtype and CPXchgprobtype
    The routine CPXXchgprobtype/CPXchgprobtype changes the current problem to a related problem.
  • CPXXchgprobtypesolnpool and CPXchgprobtypesolnpool
    The routine CPXXchgprobtypesolnpool/CPXchgprobtypesolnpool changes the current problem, if it is a mixed integer problem, to a related fixed problem using a solution from the solution pool.
  • CPXXchgqpcoef and CPXchgqpcoef
    The routine CPXXchgqpcoef/CPXchgqpcoef changes the coefficient in the quadratic objective of a quadratic problem (QP) corresponding to the variable pair (i,j) to the value newvalue.
  • CPXXchgrhs and CPXchgrhs
    The routine CPXXchgrhs/CPXchgrhs changes the righthand side coefficients of a set of linear constraints in the CPLEX problem object.
  • CPXXchgrngval and CPXchgrngval
    The routine CPXXchgrngval/CPXchgrngval changes the range coefficients of a set of linear constraints in the CPLEX problem object.
  • CPXXchgrowname and CPXchgrowname
    This routine changes the names of linear constraints in a CPLEX problem object.
  • CPXXchgsense and CPXchgsense
    The routine CPXXchgsense/CPXchgsense changes the sense of a set of linear constraints of a CPLEX problem object.
  • CPXXcleanup and CPXcleanup
    The routine CPXXcleanup/CPXcleanup changes to zero any problem coefficients that are smaller in magnitude than the tolerance specified in the argument eps.
  • CPXXcloneprob and CPXcloneprob
    The routine CPXXcloneprob/CPXcloneprob can be used to create a new CPLEX problem object and copy all the problem data from an existing problem object to it.
  • CPXXcloseCPLEX and CPXcloseCPLEX
    This routine frees all of the data structures associated with CPLEX.
  • CPXXclpwrite and CPXclpwrite
    After CPXXrefineconflict/CPXrefineconflict or CPXXrefineconflictext/CPXrefineconflictext has been invoked on an infeasible problem to identify a minimal set of constraints that are in conflict, this routine will write an LP format file containing the identified conflict.
  • CPXXcompletelp and CPXcompletelp
    The routine CPXXcompletelp/CPXcompletelp instructs CPLEX to flush its modification cache.
  • CPXXcopybase and CPXcopybase
    The routine CPXXcopybase/CPXcopybase copies a basis into a CPLEX problem object.
  • CPXXcopyctype and CPXcopyctype
    The routine CPXXcopyctype/CPXcopyctype can be used to copy variable type information into a given problem.
  • CPXXcopylp and CPXcopylp
    The routine CPXXcopylp/CPXcopylp copies data that define an LP problem to a CPLEX problem object.
  • CPXXcopylpwnames and CPXcopylpwnames
    The routine CPXXcopylpwnames/CPXcopylpwnames copies LP data into a CPLEX problem object in the same way as the routine CPXXcopylp/CPXcopylp, but CPXXcopylpwnames/CPXcopylpwnames accepts additional arguments to specify the names of constraints and variables in the CPLEX problem object.
  • CPXXcopynettolp and CPXcopynettolp
    The routine CPXXcopynettolp/CPXcopynettolp copies a network problem stored in a network problem object to a CPLEX problem object (as an LP).
  • CPXXcopyobjname and CPXcopyobjname
    The routine CPXXcopyobjname/CPXcopyobjname copies a name for the objective function into a CPLEX problem object.
  • CPXXcopyorder and CPXcopyorder
    The routine CPXXcopyorder/CPXcopyorder copies a priority order to a CPLEX problem object of type CPXPROB_MILP, CPXPROB_MIQP, or CPXPROB_MIQCP.
  • CPXXcopypartialbase and CPXcopypartialbase
    The routine CPXXcopypartialbase/CPXcopypartialbase copies a partial basis into an LP problem object.
  • CPXXcopyqpsep and CPXcopyqpsep
    The routine CPXXcopyqpsep/CPXcopyqpsep is used to copy the quadratic objective matrix Q for a separable QP problem.
  • CPXXcopyquad and CPXcopyquad
    The routine CPXXcopyquad/CPXcopyquad is used to copy a quadratic objective matrix Q when Q is not diagonal.
  • CPXXcopysos and CPXcopysos
    The routine CPXXcopysos/CPXcopysos copies special ordered set (SOS) information to a problem object of type CPXPROB_MILP, CPXPROB_MIQP, or CPXPROB_MIQCP.
  • CPXXcopystart and CPXcopystart
    The routine CPXXcopystart/CPXcopystart provides starting information for use in a subsequent call to a simplex optimization routine (CPXXlpopt/CPXlpopt with CPX_PARAM_LPMETHOD or CPX_PARAM_QPMETHOD set to CPX_ALG_PRIMAL or CPX_ALG_DUAL, CPXXdualopt/CPXdualopt, CPXXprimopt/CPXprimopt, or CPXXhybnetopt/CPXhybnetopt).
  • CPXXcreateprob and CPXcreateprob
    The routine CPXXcreateprob/CPXcreateprob creates a CPLEX problem object in the CPLEX environment.
  • CPXXdelchannel and CPXdelchannel
    The routine CPXXdelchannel/CPXdelchannel flushes all message destinations for a channel, clears the message destination list, and frees the memory allocated to the channel.
  • CPXXdelcols and CPXdelcols
    The routine CPXXdelcols/CPXdelcols deletes all the columns in a specified range.
  • CPXXdelfpdest and CPXdelfpdest
    The routine CPXXdelfpdest/CPXdelfpdest removes a file from the list of message destinations for a channel.
  • CPXXdelfuncdest and CPXdelfuncdest
    The routine CPXXdelfuncdest/CPXdelfuncdest removes the function msgfunction from the list of message destinations associated with a channel.
  • CPXXdelindconstrs and CPXdelindconstrs
    The routine CPXXdelindconstrs/CPXdelindconstrs deletes a range of indicator constraints.
  • CPXXdelmipstarts and CPXdelmipstarts
    The routine CPXXdelmipstarts/CPXdelmipstarts deletes a range MIP starts.
  • CPXXdelnames and CPXdelnames
    The routine CPXXdelnames/CPXdelnames removes all names that have been previously assigned to model entities, such as rows, columns, quadratic constraints, special ordered sets (SOSs), or indicator constraints.
  • CPXXdelqconstrs and CPXdelqconstrs
    The routine CPXXdelqconstrs/CPXdelqconstrs deletes a range of quadratic constraints.
  • CPXXdelrows and CPXdelrows
    The routine CPXXdelrows/CPXdelrows deletes a range of rows.
  • CPXXdelsetcols and CPXdelsetcols
    The routine CPXXdelsetcols/CPXdelsetcols deletes a set of columns from a CPLEX problem object.
  • CPXXdelsetmipstarts and CPXdelsetmipstarts
    The routine CPXXdelsetmipstarts/CPXdelsetmipstarts deletes a set of MIP starts.
  • CPXXdelsetrows and CPXdelsetrows
    The routine CPXXdelsetrows/CPXdelsetrows deletes a set of rows.
  • CPXXdelsetsolnpoolfilters and CPXdelsetsolnpoolfilters
    The routine CPXXdelsetsolnpoolfilters/CPXdelsetsolnpoolfilters deletes filters from the problem object specified by the argument lp.
  • CPXXdelsetsolnpoolsolns and CPXdelsetsolnpoolsolns
    The routine CPXXdelsetsolnpoolsolns/CPXdelsetsolnpoolsolns deletes solutions from the solution pool of the problem object specified by the argument lp.
  • CPXXdelsetsos and CPXdelsetsos
    The routine CPXXdelsetsos/CPXdelsetsos deletes a group of special ordered sets (SOSs) from a CPLEX problem object.
  • CPXXdelsolnpoolfilters and CPXdelsolnpoolfilters
    The routine CPXXdelsolnpoolfilters/CPXdelsolnpoolfilters deletes filters from the the problem object specified by the argument lp.
  • CPXXdelsolnpoolsolns and CPXdelsolnpoolsolns
    The routine CPXXdelsolnpoolsolns/CPXdelsolnpoolsolns deletes a range of solutions from the solution pool.
  • CPXXdelsos and CPXdelsos
    The routine CPXXdelsos/CPXdelsos deletes a group of special ordered sets (SOSs) from a CPLEX problem object.
  • CPXXdeserializercreate and CPXdeserializercreate
    Creates a de-serializer that de-serializes data from a block of memory.
  • CPXXdeserializerdestroy and CPXdeserializerdestroy
    Destroys a de-serializer that was created by CPXXdeserializercreate/CPXdeserializercreate.
  • CPXXdeserializerleft and CPXdeserializerleft
    Accesses the number of bytes left in a de-serializer.
  • CPXXdisconnectchannel and CPXdisconnectchannel
    The routine CPXXdisconnectchannel/CPXdisconnectchannel flushes all message destinations associated with a channel and clears the corresponding message destination list.
  • CPXXdperwrite and CPXdperwrite
    When solving degenerate linear programs with the dual simplex method, CPLEX may initiate a perturbation of the objective function of the problem in order to improve performance.
  • CPXXdualopt and CPXdualopt
    The routine CPXXdualopt/CPXdualopt may be used at any time after a linear program has been created via a call to CPXXcreateprob/CPXcreateprob to find a solution to that problem using the dual simplex algorithm.
  • CPXXdualwrite and CPXdualwrite
    The routine CPXXdualwrite/CPXdualwrite writes a dual formulation of the current CPLEX problem object.
  • CPXXembwrite and CPXembwrite
    The routine CPXXembwrite/CPXembwrite writes out the network embedded in the selected problem object.
  • CPXXfclose and CPXfclose
    The routine CPXXfclose/CPXfclose closes files that are used in conjunction with the routines CPXXaddfpdest/CPXaddfpdest, CPXXdelfpdest/CPXdelfpdest, and CPXXsetlogfile/CPXsetlogfile.
  • CPXXfeasopt and CPXfeasopt
    The routine CPXXfeasopt/CPXfeasopt computes a minimum-cost relaxation of the righthand side values of constraints or bounds on variables in order to make an infeasible problem feasible.
  • CPXXfeasoptext and CPXfeasoptext
    The routine CPXXfeasoptext/CPXfeasoptext extends CPXXfeasopt/CPXfeasopt in several ways.
  • CPXXfltwrite and CPXfltwrite
    The routine CPXXfltwrite/CPXfltwrite writes filters from the selected problem object to a file in FLT format.
  • CPXXflushchannel and CPXflushchannel
    The routine CPXXflushchannel/CPXflushchannel flushes (outputs and clears the buffers of) all message destinations for a channel.
  • CPXXflushstdchannels and CPXflushstdchannels
    The routine CPXXflushstdchannels/CPXflushstdchannels flushes the output buffers of the four standard channels cpxresults, cpxwarning, cpxerror, and cpxlog.
  • CPXXfopen and CPXfopen
    The routine CPXXfopen/CPXfopen opens files to be used in conjunction with the routines CPXaddfpdest, CPXXdelfpdest/CPXdelfpdest and CPXXsetlogfile/CPXsetlogfile.
  • CPXXfputs and CPXfputs
    The routine CPXXfputs/CPXfputs can be used to write output to a file opened with CPXXfopen/CPXfopen.
  • CPXXfreeprob and CPXfreeprob
    The routine CPXXfreeprob/CPXfreeprob removes the specified CPLEX problem object from the CPLEX environment and frees the associated memory used internally by CPLEX.
  • CPXXgetax and CPXgetax
    The routine CPXXgetax/CPXgetax accesses row activity levels for a range of linear constraints.
  • CPXXgetbaritcnt and CPXgetbaritcnt
    The routine CPXXgetbaritcnt/CPXgetbaritcnt accesses the total number of Barrier iterations to solve an LP problem.
  • CPXXgetbase and CPXgetbase
    The routine CPXXgetbase/CPXgetbase accesses the basis resident in a CPLEX problem object.
  • CPXXgetbestobjval and CPXgetbestobjval
    The routine CPXXgetbestobjval/CPXgetbestobjval accesses the currently best known bound of all the remaining open nodes in a branch-and-cut tree.
  • CPXXgetcallbackinfo and CPXgetcallbackinfo
    The routine CPXXgetcallbackinfo/CPXgetcallbackinfo accesses information about the current optimization process from within a user-written callback function.
  • CPXXgetchannels and CPXgetchannels
    The routine CPXXgetchannels/CPXgetchannels obtains pointers to the four default channels created when you call the routine CPXXopenCPLEX/CPXopenCPLEX.
  • CPXXgetchgparam and CPXgetchgparam
    The routine CPXXgetchgparam/CPXgetchgparam returns an arrary of parameter numbers (unique identifiers) for parameters which are not set at their default values.
  • CPXXgetcoef and CPXgetcoef
    The routine CPXXgetcoef/CPXgetcoef accesses a single constraint matrix coefficient of a CPLEX problem object.
  • CPXXgetcolindex and CPXgetcolindex
    The routine CPXXgetcolindex/CPXgetcolindex searches for the index number of the specified column in a CPLEX problem object.
  • CPXXgetcolinfeas and CPXgetcolinfeas
    The routine CPXXgetcolinfeas/CPXgetcolinfeas computes the infeasibility of a given solution for a range of variables.
  • CPXXgetcolname and CPXgetcolname
    The routine CPXgetcolname accesses a range of column names or, equivalently, the variable names of a CPLEX problem object.
  • CPXXgetcols and CPXgetcols
    The routine CPXXgetcols/CPXgetcols accesses a range of columns of the constraint matrix of a CPLEX problem object.
  • CPXXgetconflict and CPXgetconflict
    This routine returns the linear constraints and variables belonging to a conflict previously computed by the routine CPXXrefineconflict/CPXrefineconflict.
  • CPXXgetconflictext and CPXgetconflictext
    For an infeasible problem, if the infeasibility has been analysed by CPXXrefineconflictext/CPXrefineconflictext, this routine accesses information about the conflict computed by it.
  • CPXXgetcrossdexchcnt and CPXgetcrossdexchcnt
    The routine CPXXgetcrossdexchcnt/CPXgetcrossdexchcnt accesses the number of dual exchange iterations in the crossover method.
  • CPXXgetcrossdpushcnt and CPXgetcrossdpushcnt
    The routine CPXXgetcrossdpushcnt/CPXgetcrossdpushcnt accesses the number of dual push iterations in the crossover method.
  • CPXXgetcrosspexchcnt and CPXgetcrosspexchcnt
    The routine CPXXgetcrosspexchcnt/CPXgetcrosspexchcnt accesses the number of primal exchange iterations in the crossover method.
  • CPXXgetcrossppushcnt and CPXgetcrossppushcnt
    The routine CPXXgetcrossppushcnt/CPXgetcrossppushcnt accesses the number of primal push iterations in the crossover method.
  • CPXXgetctype and CPXgetctype
    The routine CPXXgetctype/CPXgetctype accesses the types for a range of variables in a problem object.
  • CPXXgetcutoff and CPXgetcutoff
    The routine CPXXgetcutoff/CPXgetcutoff accesses the MIP cutoff value being used during mixed integer optimization.
  • CPXXgetdblparam and CPXgetdblparam
    The routine CPXXgetdblparam/CPXgetdblparam obtains the current value of a CPLEX parameter of type double.
  • CPXXgetdblquality and CPXgetdblquality
    The routine CPXXgetdblquality/CPXgetdblquality accesses double-valued information about the quality of the current solution of a problem.
  • CPXXgetdettime and CPXgetdettime
    This routine returns a deterministic time stamp.
  • CPXXgetdj and CPXgetdj
    The routine CPXXgetdj/CPXgetdj accesses the reduced costs for a range of the variables of a linear or quadratic program.
  • CPXXgetdsbcnt and CPXgetdsbcnt
    The routine CPXXgetdsbcnt/CPXgetdsbcnt accesses the number of dual super-basic variables in the current solution.
  • CPXXgeterrorstring and CPXgeterrorstring
    The routine CPXXgeterrorstring/CPXgeterrorstring returns an error message string corresponding to an error code.
  • CPXXgetgrad and CPXgetgrad
    The routine CPXXgetgrad/CPXgetgrad can be used, after an LP has been solved and a basis is available, to access information useful for different types of post-solution analysis.
  • CPXXgetindconstr and CPXgetindconstr
    The routine CPXXgetindconstr/CPXgetindconstr accesses a specified indicator constraint on the variables of a CPLEX problem object.
  • CPXXgetindconstrindex and CPXgetindconstrindex
    The routine CPXXgetindconstrindex/CPXgetindconstrindex searches for the index number of the specified indicator constraint in a CPLEX problem object.
  • CPXXgetindconstrinfeas and CPXgetindconstrinfeas
    The routine CPXXgetindconstrinfeas/CPXgetindconstrinfeas computes the infeasibility of a given solution for a range of indicator constraints.
  • CPXXgetindconstrname and CPXgetindconstrname
    The routine CPXXgetindconstrname/CPXgetindconstrname accesses the name of a specified indicator constraint of a CPLEX problem object.
  • CPXXgetindconstrslack and CPXgetindconstrslack
    The routine CPXXgetindconstrslack/CPXgetindconstrslack accesses the slack values for a range of indicator constraints.
  • CPXXgetinfocallbackfunc and CPXgetinfocallbackfunc
    The routine CPXXgetinfocallbackfunc/CPXgetinfocallbackfunc accesses the user-written callback routine to be called regularly during the optimization of a mixed integer program (MIP).
  • CPXXgetintparam and CPXgetintparam
    The routine CPXXgetintparam/CPXgetintparam obtains the current value of a CPLEX parameter of type CPXINT.
  • CPXXgetintquality and CPXgetintquality
    The routine CPXXgetintquality/CPXgetintquality accesses integer-valued information about the quality of the current solution of a problem.
  • CPXXgetitcnt and CPXgetitcnt
    The routine CPXXgetitcnt/CPXgetitcnt accesses the total number of simplex iterations to solve an LP problem, or the number of crossover iterations in the case that the barrier optimizer is used.
  • CPXXgetlb and CPXgetlb
    The routine CPXXgetlb/CPXgetlb accesses a range of lower bounds on the variables of a CPLEX problem object.
  • CPXXgetlogfile and CPXgetlogfile
    The routine CPXXgetlogfile/CPXgetlogfile accesses the log file to which messages from all four CPLEX-defined channels are written.
  • CPXXgetlongparam and CPXgetlongparam
    The routine CPXXgetlongparam/CPXgetlongparam obtains the current value of a CPLEX parameter of type CPXLONG.
  • CPXXgetlpcallbackfunc and CPXgetlpcallbackfunc
    The routine CPXXgetlpcallbackfunc/CPXgetlpcallbackfunc accesses the user-written callback routine to be called after each iteration during the optimization of a continuous problem (LP, QP, or QCP), and also periodically during the CPLEX presolve algorithm.
  • CPXXgetmethod and CPXgetmethod
    The routine CPXXgetmethod/CPXgetmethod returns an integer specifying the solution algorithm used to solve the resident LP, QP, or QCP problem.
  • CPXXgetmipcallbackfunc and CPXgetmipcallbackfunc

  • CPXXgetmipitcnt and CPXgetmipitcnt

  • CPXXgetmiprelgap and CPXgetmiprelgap

  • CPXXgetmipstartindex and CPXgetmipstartindex

  • CPXXgetmipstartname and CPXgetmipstartname

  • CPXXgetmipstarts and CPXgetmipstarts

  • CPXXgetnodecnt and CPXgetnodecnt

  • CPXXgetnumsemicont and CPXgetnumsemicont
    The routine CPXXgetnumsemicont/CPXgetnumsemicont accesses the number of semi-continuous variables in a CPLEX problem object.
  • CPXXgetnumsemiint and CPXgetnumsemiint

  • CPXXgetnumsos and CPXgetnumsos

  • CPXXgetobj and CPXgetobj

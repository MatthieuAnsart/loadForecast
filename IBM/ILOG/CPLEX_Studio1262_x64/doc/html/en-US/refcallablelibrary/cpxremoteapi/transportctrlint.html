CPXXtransportctrlint and CPXtransportctrlint

This routine performs transport-specific control operations. It allows control of low-level transport options for the remote object similar to the ioctl system call that allows device-specific operations on Unix systems.

void CPXXtransportctrl( CPXCENVptr env, int ctrl, CPXINT * data_p )

void CPXtransportctrl( CPXCENVptr env, int ctrl, CPXINT * data_p )

Description

This routine performs low-level, transport-specific control operations on the remote object transport instance associated with env. The routine is similar to the ioctl function that is well known on Unix systems to control low-level, device-specific options.

The following table specifies the various control operations that are supported by this routine.
Table 1. Supported operations for CPXXtransportctrlint and CPXtransportctrlint
Transport ctrl Operation
process 0x3000 Stores the process ID of the child process in *data_p.
MPI 0x1000 Stores the rank of the remote process in *data_p.
TCP/IP 0x2000 Sets the timeout for socket-receive operations to *data_p. The timeout is specified in milliseconds; negative values revert to the system default value.
0x2001 Stores the timeout for socket-receive operations in *data_p. The timeout is specified in milliseconds; a negative value specifies that the timeout is set to the system default value.
0x2002 Sets the timeout for socket-send operations to *data_p. The timeout is specified in milliseconds; negative values revert to the system default value.
0x2003 Stores the timeout for socket-send operations in *data_p. The timeout is specified in milliseconds; a negative value specifies that the timeout is set to the system default value.

Arguments

Name Description
env A pointer to a remote object CPLEX environment, as returned by CPXXopenCPLEXremote or CPXopenCPLEXremote.
ctrl An integer that specifies which control operation to perform. (See the table for a list of supported operations.)
data_p Input/output argument that holds or receives data, depending on the operation selected by ctrl.

Return

The routine returns 0 (zero) on success and an error code otherwise.

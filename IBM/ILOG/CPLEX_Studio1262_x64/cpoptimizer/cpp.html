Using CP Optimizer with Microsoft Visual C++

This file describes how to use Microsoft Visual Studio to:

  • build and run the C++ examples delivered with CP Optimizer (see javadotnet.html for the Java and C# examples).
  • create a C++ project and link the target with CP Optimizer.

CP Optimizer Installation Folder

Throughout this document, the CP Optimizer installation folder is referred to as <CPOPTDIR>, the Concert Technology installation folder is referred to as <CONCERTDIR>, and the CPLEX installation folder is referred to as <CPLEXDIR>,

For example, if CP Optimizer has been installed in the folder C:\Program Files\IBM\ILOG\CPLEX_Studio_1262\cpoptimizer (the default location), references to <CPOPTDIR>\include represent C:\Program Files\IBM\ILOG\CPLEX_Studio_1262\cpoptimizer \include. The default for Concert Technology is C:\Program Files\IBM\ILOG\CPLEX_Studio_1262\concert and the default for CPLEX is C:\Program Files\IBM\ILOG\CPLEX_Studio_1262\cplex.

Libraries

The CP Optimizer libraries are delivered in the multi-threaded DLL format (linkable with msvcrt.lib) and multi-threaded DLL debug format (linkable with msvcrtd.lib) for Visual Studio 2010, 2012 and 2013. All CP Optimizer libraries on windows use the standard template library (STL) and are compiled using the namespace std.

Building and Running CP Optimizer Examples

The C++ CP Optimizer examples have all been gathered in one project for each port. The instructions below use the x86_windows_vs2012 port, but the same instructions apply for other Windows ports. The related solution file for this port is is <CPOPTDIR>\examples\x86_windows_vs2012\stat_mda\examples.sln. Be aware that the order of the instructions below is important.

  1. Start Microsoft Visual Studio 2012.
  2. From the File menu, choose Open Solution.
    The Open Solution dialog box appears.
    The default selection in the Files Of Type drop-down list is     Solution Files (.sln).
    • Select the folder <CPOPTDIR>\examples\x86_windows_vs2012\stat_mda.
    • Select the examples file and click Open.
  3. To build only one example (for instance, ppp):
    • Select the ppp project in the Solution Explorer window.
    • From the Build menu, choose Build ppp.
      Wait for the completion of the building process.
  4. To build all of the examples:
    • From the Build menu, choose Build Solution
      Wait for the completion of the building process.
  5. To run an example (for instance, ppp):
    • Open a command prompt window by running the Visual Studio 2012 Command Prompt.
      In the window Visual Studio 2012 Command prompt:
      • Type <CPOPTDIR>\examples\x86_windows_vs2012\stat_mda\ppp.
      • The result is then displayed.

Building Your Own Project which Links with CP Optimizer

Note:
The information below applies to the 32-bit Visual Studio 2012 port, but the general information applies also to the other ports.

Let's assume that you want to build a target named test.exe and have:

  • a source file named test.cpp which uses Concert Technology;
  • a folder where this file is located and which, for the sake of simplicity, we'll refer to as <MYAPPDIR>.

One way to achieve that is to create a Solution named test.sln as described here. Be aware that the order of instructions is important.

  1. Start Microsoft Visual Studio 2012.
  2. The first step is to build the test.sln solution.
    From the File menu, select New->, and then Project....

The New Project dialog box appears.

    • In the Project Types pane, select Visual C++ Projects.
    • In the Templates pane, select the Win32 Project icon.
    • Fill in the project name (test).
    • If necessary, correct the location of the project (to <MYAPPDIR>)
    • Click OK

When the Win32 Application Wizard appears...

    • Click on Application Settings.
    • Select Console Application as Application type.
    • Make sure that Empty project is checked in Additional Options.
    • Click Finish.

This creates a solution, test, with a single project, test. You can see the contents of the solution by selecting Solution Explorer in the View menu.

  1. Now you must add your source file to the project. From the Project menu, choose Add Existing Item...
    • Move to the folder <MYAPPDIR> and select test.cpp.
    • Click Open.
  2. Next, you have to set some options so that the project knows where to find the CP Optimizer and Concert include files and the CP Optimizer and Concert libraries.

From the Project menu, choose test Properties.
The test Property Pages dialog box appears.

In the Configuration drop-down list, select Release.

Select C/C++ in the Configuration Properties tree.

    • Select General:
      • In the Additional Include Directories field, add the directories:
        • <CPOPTDIR>\include.
        • <CONCERTDIR>\include.
      • For Debug Information Format, choose Line Numbers Only (/Zd).
      • Choose No for Detect 64-bit Portability Issues.
    • Select Preprocessor:
      • Add IL_STD to the Preprocessor Definitions field. This defines the macro IL_STD which is needed to use the STL.
    • Select Code Generation:
      • Set Runtime Library to Multi-threaded (/MD).

Select Linker in the Configuration Properties tree.

    • Select Input and then select Additional Dependencies. Add the files:
      • wsock32.lib
      • <CPOPTDIR>\lib\x86_windows_vs2012\stat_mda\cp.lib
      • <CONCERTDIR>\lib\x86_windows_vs2012\stat_mda\concert.lib
      • <CPLEXDIR>\lib\x86_windows_vs2012\stat_mda\cplex1262.lib
      • <CPLEXDIR>\lib\x86_windows_vs2012\stat_mda\ilocplex.lib

Click OK to close the test Property Pages dialog box.

  1. Next, you have to set the default project configuration.

From the Build menu, select Configuration Manager...

    • Select Release in the Active Solution Configuration drop-down list.
    • Click Close.
  1. Finally, to build the project, from the Build menu, select Build Solution

After completion of the compiling and linking process, the target is created. The full path of the test.exe is <MYAPPDIR>\test\Release\test.exe. Note that, if the installer has not set the PATH variables to CPLEX Optimizer Studio DLLs, the CPLEX DLL needs to be added to the default path variable (PATH) before running any executable. It is located in <CPLEXDIR>\bin\x86_win32 for 32 bits applications and in <CPLEXDIR>\bin\x86_win64 for 64 bits applications.

Remark:

From the CP Optimizer point of view, the only difference between the Win32 Release and Win32 Debug targets is:

  • the NDEBUG macro is defined for the Win32 Release target.
  • the NDEBUG macro is not defined for the Win32 Debug target.

This is why we have suggested using Release in the test.sln example, even though it is not the default proposed by Visual C++ .NET. Refer to the Visual C++ Reference Manual for full information on Release and Debug configurations.

The interaction of the NDEBUG macro and the CP Optimizer inline member functions is documented in the Concepts section of the CP Optimizer Reference Manual.
